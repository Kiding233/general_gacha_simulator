## 文本节点连线关系（设计要点）

### 核心输入→输出链路

```
可用资源 → 资源获取函数 → 行动空间
              ↓
         抽卡策略（决定资源投入序列）
              ↓
         （广义）卡池机制 → 单抽信息向量
              ↓
         目标卡集合 → 广义出率 → 统计量
              ↓
         风险阈值α → 整体成功概率
```

### GDR → 决策链路

```
广义出率 → 抽出全部目标卡（二值）→ 整体成功概率（依赖α）
              ↓
         成功概率是否高于阈值 → 确定初始方案
              ↓
         资源亏损 → 加权满意度
              ↓
         资源剩余 → 条件风险价值(CVaR)
```

### 逆向问题链路

```
给定成功概率 → 后退法减少目标卡
              ↓
         给定成功概率 → 需要多少额外资源
              ↓
         条件剩余分布 → 可用资源估计
```

### 策略比较链路

```
策略参数 → 多种策略 → 加权计数方法 → 福利水平比较
              ↓
         策略不敏感GDR识别 → 退化问题
              ↓
         统计功效 → 差异显著性
```

## 已实现功能详解

### 前进法/后退法（2026-05-12 重写）

**正确实现原理**：每次目标集合变化都必须重新运行完整模拟。

```
前进法：
  从空集合 {} 开始
  ↓
  按 desire_weights 降序排序目标卡
  ↓
  逐个添加目标卡，每次添加都重新运行 num_simulations 次完整模拟
  ↓
  计算新目标集合的成功率
  ↓
  当成功率 ≥ success_threshold 时停止
  ↓
  返回最终目标集合

后退法：
  从完整集合 {A,B,C,...} 开始
  ↓
  按 miss_cost_weights 升序排序目标卡
  ↓
  逐个尝试移除目标卡，每次移除都重新运行 num_simulations 次完整模拟
  ↓
  确保成功率始终 ≥ success_threshold
  ↓
  记录最后一次成功移除后的状态
  ↓
  返回最终目标集合
```

**关键设计**：策略（`_SmartStrategy`）直接依赖目标卡集合，策略不同则抽卡行为不同，必须重新模拟。

### VaR/CVaR 分析（2026-05-12 改进）

- 改为原生表格输出（不是 matplotlib 图片）
- 表格类型通过 ResultUnit.set_table() 渲染
- 导出时自动识别类型：图片导出 PNG，表格导出 CSV

### JASP 风格分析面板（Phase2 重构）

- 右侧改为可滚动结果区域
- 每个分析结果成为独立的 ResultUnit 卡片（标题+内容+导出按钮）
- 支持三种内容类型：chart / table / text
- 增量分析：缓存已完成的分析结果，条件不变时跳过重算
