## 一、理论架构概述

### 1.1 核心理念

理论参考.svg 建立了一套以**蒙特卡洛模拟**为核心方法的、用于分析和规划多卡池抽卡行为的理论框架。其出发点是对传统"单卡池概率学"的八个具体缺陷的反思（按图中连线关系）：

1. 对指导具体的抽卡实践作用有限
2. 难以看出决定抽多个卡池时成功的整体概率
3. 假设难以满足/不符合实际
4. 聚焦于长期问题，忽略短期问题
5. 较少地做敏感性/稳健性分析
6. 不能自选保守程度，期望太激进，保底太保守
7. 难以适应多样的现实情况和复杂的现实需要
8. （仍需）兼容单卡概率学的计算

该框架的替代方案是：**完整建模卡池机制 → 蒙特卡洛模拟 → 广义出率(GDR)评估 → 统计决策**。

### 1.2 整体架构：以「基础关系」为核心的概念网络

SVG 的实际结构并非线性管线，而是以 **「基础关系」** 节点（id=626）为中心的**概念网络**。基础关系包含 8 个核心概念，互相关联：

> **SVG 连线揭示的原始关系：**
> - 可用资源 → {抽卡策略, 行动空间}
> - 行动空间 → {抽卡策略}（行动空间定义了可选行动的范围）
> - 抽卡策略 → {资源投入序列}（策略决定具体的资源投入方案）
> - 资源投入序列 → {抽卡结果}（每次行动生成一次抽卡结果）
> - （广义）卡池机制 → 资源投入序列→抽卡结果 这条连线本身（**卡池机制决定了投入如何转化为结果**：基础概率分布 × 保底概率修正算子，将一次抽卡行动映射为具体的出卡结果）
> - 抽卡结果 → {广义出率, 单抽信息向量, 行动空间, 可用资源}（结果消耗资源、触发保底状态更新、改变后续行动空间）
> - （广义）卡池机制 → {抽卡策略, 行动空间}（机制参数影响策略可行性和行动选择）
> - 广义出率 → {整体成功概率, 抽卡策略, 统计量}

核心链路在具体实现中可以理解为（注意卡池机制不是链路中的一个步骤，而是作用于「投入→结果」这条边的函数）：

```
可用资源 + 资源获取函数（时间驱动）
    → 行动空间（抽卡/等待/兑换/商店/氪金）
    → 抽卡策略（决定资源投入序列，可以是事先决定或动态决定的）
    → 资源投入序列 ──[每次行动]──→ 抽卡结果
                         ↑
                （广义）卡池机制
              基础概率分布 × 保底概率修正算子
              （决定投入如何生成每个抽卡结果）
                         ↓
    → 单抽信息向量（每抽结果的完整编码）
    → 目标卡集合（从全卡ID空间中选定的兴趣集）
    → 广义出率 GDR（全部结果→实数的函数族）
    → 统计量（期望/分位数/分布/CVaR）
    → 风险阈值α下的整体成功概率
    → 决策
```

### 1.3 两套假设体系

SVG 中区分了两层假设：

**（A）基本假设（2条）**——建模的前提条件：
- **有哪些池子**：为此要么有提前公布的信息，要么有千里眼，要么假设稳定的版本周期（对应长期问题）
- **有确定的卡池机制**：为此厂商不能暗改概率 / 不假设公示概率不可信

**（B）一般卡池额外满足的假设（10条）**——当前实现的简化约定：
- 抽卡不推进现实时间
- 卡池-卡绑定（每张卡唯一对应一个池子类别）
- 基础概率分布不随时间变化
- 同一卡池不同抽使用同一种资源
- 单抽资源消耗固定
- 最小行动单位为抽一次获取确定数量张卡
- 不考虑资源约束时，行动空间不由上一次行动决定
- 「一个」池子只占据一个连续的现实时间区间
- 池子结束后就永远结束
- 在做出抽卡行动时，所有不消耗抽卡资源能获取的资源都已获取（零资源消耗零时间增量的行动不存在）

### 1.4 GDR（广义出率）的精确语义

这是理论框架中**最核心的概念**，SVG 中有最长的详细描述（data-details 约 1500 字）。其精确定义为：

> 广义出率是一类函数，其**参数为某一次抽卡尝试的全部结果**，值域为实数。**约定更高的值总是意味着更好的结果**，这样就可以统一地讨论如何最大化广义出率。

**理论基础的随机过程定义**：
- 在行动时间 t，样本被映射为这一时间点/行动后的「单抽信息向量」
- 我们关心 t 及 t 之前的所有单抽信息向量（即全部抽卡结果）
- 由于不方便直接研究多个向量，定义 GDR 为：**t 及 t 之前的所有单抽信息向量到实数的函数**
- 两个函数的复合（随机过程 + GDR）就是一个新的随机过程，在 t_max（抽卡结束后）等价于一个随机变量
- 使用频率派概率定义：成功概率 = GDR 水平 ≥ c 的次数 / 总模拟次数（在 n→∞ 时的极限）

**GDR 与成功判断的关系**：
- GDR 提供了一次抽卡是否成功的判断标准
- 具体的成功概率**必须依赖 GDR 的选择来进行解释**
- 整体成功概率定义为：给定 GDR 水平 c，P（本次抽卡的 GDR 水平 ≥ c）

**GDR 的构造方式**：
1. **加权计数方法**：「计数并加权求和是将不定维数的数据压缩为一维数据的最普通和最常见的方法」
2. **比率型**：某种出卡数 / 实际资源消耗（关注效率）；或其倒数（每卡资源消耗）
3. **二值函数族**：输出 {0,1}，如「是否抽出全部目标卡」
4. **联合构造**：min(f₁-a, f₂-b, …)，支持多条件「且」逻辑
5. **保证型**：GDR₁ ≥ c₁（二值）× GDR₂（其他函数），扩展到多个条件

**GDR 选择的任意性和主观性**：
- 广义出率的具体选择具有相当大的任意性，取决于个体看重的方面
- 当一个函数被看作 GDR 时，意味着你认为更高的取值代表更好的结果
- 有些统计量不宜作为优化目标（因为给定卡池机制没有可靠的方法能优化它）
- **策略不敏感的 GDR**（退化问题）：几乎只由卡池机制和总抽数投入决定，策略几乎不改变其水平，因此不宜作为优化目标，但需要实际验证

### 1.5 行动空间的关键建模决策

行动空间的描述包含了几个重要的建模决策（data-details 约 600 字）：

1. **时间是特殊资源**：等待是一种消耗时间获取资源的行动——对计划资源投入函数在 (等待前, 等待后] 区间内求和得到资源增量
2. **商店是广义卡池**：每种可兑换资源视为一个独特的卡池，获取对应产出的概率是 100%。商店随时间更新 = 行动空间随时间更新。购买次数限制 = 独特约束条件或特殊资源
3. **行动空间的定义方式**：先定义无约束行动空间，然后施加对应约束
4. **在只考察抽卡相关问题时可忽略无关行动**

### 1.6 抽卡策略的语义

> 「行动的生成方法，可以是事先决定或动态决定的。好的抽卡策略应该使得某项关心的参数最大化。」

- **在哪些池子投入抽数** → 由目标卡选择决定
- **以何种方式投入** → 由关心的广义出率决定
- 在特定假设下，「每个池子抽多少抽」就可以代表一种具体策略（行动的组合）
- 策略的参数化：如以资源投入序列表示的策略

### 1.8 过程分析的事件定义（两层分类体系）

SVG 的事件分类是一个**两层体系**：单抽层（每次抽到稀有卡时的具体情况）和池子层（整个池子的最终结果）。两层相对独立，但池子层某种意义上包含单抽层——单抽层的稀有卡事件嵌套在池子层的决策过程中。连线上的文本标注（是/否、全部/部分）定义了分支条件。

> **⚠ 未决问题**：两层之间的确切关系尚未完全理清。当一个池子出两个及以上 SSR 时，单抽层的多次稀有卡事件如何汇总为池子层的一个终态标签，需要进一步设计。

#### 第一层：单抽层 —— 出了什么卡？（稀有卡事件分类）

> 入口：**「（某一抽）出了稀有卡」** → 「出了什么卡」→ 四条路径，每条经「是否触发保底」（是/否）分叉，共 8 种标签：

```
（某一抽）出了稀有卡
  └─ 出了什么卡？
      ├─ "当期up"目标卡 ─── 是否触发保底？
      │   ├─ 是 → 保底[i]出
      │   └─ 否 → 提前出
      │
      ├─ 非"当期up"目标卡 ─── 是否触发保底？
      │   ├─ 是 → 保底[i]歪得目标卡
      │   └─ 否 → 提前歪得目标卡
      │
      ├─ "当期up"非目标卡 ─── 是否触发保底？
      │   │   （注：需要有一个以上up才存在此分支）
      │   ├─ 是 → 保底[i]歪非目标up
      │   └─ 否 → 提前出非目标up
      │
      └─ 普通稀有卡非目标卡 ─── 是否触发保底？
          ├─ 是 → 保底歪常驻
          └─ 否 → 提前出常驻
```

> **关于单抽层是否假定「抽的是包含目标卡的池子」**：否。四条路径中，「当期up目标卡」仅当池子的up包含目标卡时才可能产出，其余三条（歪得目标卡、出非目标up、出常驻）在任何池子中均可能发生。标签中的「目标卡」参照的是用户设定的整体目标集合，而非仅限当前池子的up——因此单抽层也适用于 up 不含目标卡的池子（只是「当期up目标卡」路径永远不会触发）。

#### 第二层：池子层 —— 这个池子的整体结果？

> 入口：**「该池子"当期up"是否包含未抽出的目标卡」**（是/否）→ 四个主分支均经「是否投入非免费抽」↔「是否抽出当期up」两层判断，共 12 种终态：

```
该池子"当期up"是否包含未抽出的目标卡？
│
├─【是：up 包含未抽出的目标卡】
│  │
│  ├─ 是否投入非免费抽？──【是】
│  │  └─ 是否抽出"当期up"的目标卡？
│  │     ├─【是】→ 全部or部分？
│  │     │   ├─ 部分 → 出部分卡
│  │     │   └─ 全部 → 出全部卡
│  │     └─【否】→ 抽了没出
│  │
│  └─ 是否投入非免费抽？──【否】
│     └─ 是否抽出"当期up"的目标卡？
│        ├─【是】→ 全部or部分？
│        │   ├─ 部分 → 免费抽出部分卡
│        │   └─ 全部 → 免费抽出全部卡
│        └─【否】→ 跳过
│
└─【否：up 不包含未抽出的目标卡】
   │
   ├─ 是否投入非免费抽？──【是】
   │  └─ 是否抽出"当期up"？
   │     ├─【是】→ 全部or部分？
   │     │   ├─ 部分 → 计划外抽卡出部分up
   │     │   └─ 全部 → 计划外抽卡出全部up
   │     └─【否】→ 浪费资源
   │
   └─ 是否投入非免费抽？──【否】
      └─ 是否抽出"当期up"？
         ├─【是】→ 全部or部分？
         │   ├─ 部分 → 计划外免费抽出部分up
         │   └─ 全部 → 计划外免费抽出全部up
         └─【否】→ 忽略
```

> **池子层 12 种终态**：出全部卡、出部分卡、免费抽出全部卡、免费抽出部分卡、计划外抽卡出部分up、计划外抽卡出全部up、浪费资源、计划外免费抽出部分up、计划外免费抽出全部up、跳过、抽了没出、忽略。

> **细化方向**：SVG 中「是否超出需求」节点连接在「计划外抽卡出…」附近（标注：是→浪费资源），提示可以按此进一步细化事件划分，但该节点不直接挂在池子层决策树上，属于可选的扩展维度。

#### 代码实现与SVG的对照

**（B）代码当前实现（`process_trace.py` 的 `infer_events`）**——按「保底触发方式 + 是否投入抽数」简化为 5 种：

| 事件类型 | 定义 | 对应 SVG 两层标签 |
|----------|------|-------------------|
| **pity_hit** | 通过保底机制获得目标卡 | 单抽层·保底[i]出 + 池子层·出全部/部分卡 |
| **early_hit** | 在保底触发前提前获得目标卡 | 单抽层·提前出 + 池子层·出全部/部分卡 |
| **miss** | 投入了抽数但未获得目标卡 | 池子层·抽了没出 |
| **skip** | 该池有目标卡但选择不投入 | 池子层·跳过 |
| **ignore** | 该池无目标卡，自然不投入 | 池子层·忽略 |

**关键差异**：

| SVG 有、代码无的标签 | 说明 |
|----------------------|------|
| 保底歪得目标卡 / 提前歪得目标卡 | 歪到了非当期up但仍是目标卡——当前 pity_hit/early_hit 合并了此情况 |
| 保底歪非目标up / 提前出非目标up | 出了当期up但不是目标——当前未单独区分 |
| 保底歪常驻 / 提前出常驻 | 出了常驻卡——当前未单独区分 |
| 免费抽出全部/部分卡 | 仅用免费资源即达成目标——当前归入 skip/ignore |
| 计划外抽卡出全部/部分up | 投入了非免费资源抽取不含目标的池子——当前归入 ignore |
| 浪费资源 | 超出需求后继续抽出（连线标注：忽略/浪费资源）——当前未追踪 |
| 兑换 | 通过兑换获得目标卡——P2 计划中待实现 |

**当前实现状态**：代码实现了简化的 5 种机制维度事件，覆盖了最常见的场景。待扩展的方向包括：
- P2 计划：兑换池/资源池事件（exchange/no_exchange/resource_draw/resource_ignore）
- 单抽层细化：区分「保底出目标卡」与「保底歪得目标卡」，区分「歪非目标up」与「歪常驻」
- 池子层细化：区分「计划内」与「计划外」抽卡，「免费抽出」与「跳过」的细分
- 两层交叉分析：单抽层 8 种 × 池子层 12 种 → 理论上 96 种组合，实际可通过维度规约降维

### 1.9 短期问题与长期问题的对比

SVG 将问题按时间范围分为两类，两类在卡池确定性、目标确定性、GDR 取值性质、研究目的上均有本质差异：

| 维度 | 短期问题 | 长期问题 |
|------|----------|----------|
| **卡池** | 近期的少数几个确定的卡池 | 以特定规则生成的一年以上时间跨度的卡池序列 |
| **目标卡** | 有确定的目标卡序列/抽取意愿 | 无确定的目标卡序列/目标卡抽取意愿 |
| **GDR 取值** | 可能离散（卡池少、结果空间小） | 近似连续（卡池多、结果空间大） |
| **研究目的** | 研究具体情境下的问题 | 研究一般情境下的问题 |

其他相关的对比维度：

| 维度 | 一端 | 另一端 |
|------|------|--------|
| **问题方向** | 正问题：给定条件求 GDR 分布 | 逆问题：给定统计量反推参数 |
| **分析性质** | 描述统计（发生了什么） | 规范分析/优化（应该怎么做） |
| **策略预设** | 预设恒定策略 | 动态变化/不预设策略 |
| **时间建模** | 行动时间（正整数，单位为一次行动） | 现实时间（单位为天，连续→可无限细分） |
| **资源分类** | 免费可再生 / 免费不可再生 / 付费不可再生 / 付费可再生 | — |

### 1.10 分析方法的层级结构

基于 SVG 的大型容器节点，分析方法按以下层级组织：

```
基于模拟结果的统计分析
├── 描述统计（资源投入分布 / 剩余分布 / 广义出率分布 / 每池分布）
├── 极端情形分析（最差情形 / 最好情形 / VaR / CVaR / 条件分布）
├── 过程/轨迹/事件分析（事件×成败交叉统计 / 转移矩阵）
├── 脆弱性分析（以池子和成功/失败为条件的条件资源剩余分布）
├── 最差后期影响（连续池子推进 / 期望可抽池子数）
```

### 1.11 复刻问题的理论框架

SVG 将「复刻问题」作为一个独立的大型主题（容器节点 id=627，1130×731），包含以下子议题：

| 子议题 | 说明 | 状态 |
|--------|------|------|
| 复刻间隔 | 同一张卡两次出现的时间间隔的建模 | ✅ 已支持（`rerun_of` 标签） |
| 「早抽早享受」和晚出的代价 | 早抽到可以早使用 vs 晚抽到可能错过的权衡 | ⚪ 理论问题 |
| 复刻是一个策略问题 | 不是简单的是否抽取，而是涉及时间偏好的策略选择 | ⚪ 理论问题 |
| 潜在价值亏损＆没抽造成的遗憾（按时间累加） | 未拥有卡的时间段内累积的「遗憾」价值 | 🟡 需带时间惩罚的GDR（A24） |
| 每个样本随时间变化的广义出率水平 | GDR 在时间轴上的动态变化 | 🟡 需 GDR 时间序列支持 |
| 单卡价值随时间缩水？ | 卡的「价值」是否随版本更迭而下降 | ⚪ 理论问题 |
| 抽取意愿随时间缩水？提升？ | 对一张卡的渴望程度是否随时间变化 | ⚪ 见动态抽卡意愿建模（E9） |

**核心洞察**：SVG 指出「复刻是一个策略问题」——如果一张卡会在未来复刻，那么「现在抽」和「等复刻」是两个不同的决策，其优劣取决于时间偏好、资源机会成本和风险承受能力。这引出「等歪行不行」（见条目 D15）。

### 1.12 单卡价值的建模维度

SVG 将「单卡价值的建模」（容器节点 id=636，935×950）作为一个独立子树，列出了影响单卡价值的多个维度：

| 维度 | 说明 |
|------|------|
| **重复抽取价值** | 同一张卡抽到第N次的价值通常递减（命座/潜能系统） |
| **常数价值** | 最简单模型：每张卡有固定不变的价值 |
| **随时间衰减** | 卡的「强度」随版本更迭而下降 |
| **以新卡推出为时间单位的随机游走** | 价值在新卡推出时发生跳跃变化 |
| **随拥有的卡/是否拥有这张卡变化** | 条件价值：拥有某张卡后，相关卡的价值可能改变（配队协同） |
| **基于社区共识 vs 个人偏好** | 价值的两个来源：客观强度评价 vs 主观喜爱 |
| **随环境变化** | 版本更新、新卡推出等外部因素改变卡的价值 |
| **现实分析中单卡价值都是条件单卡价值** | 价值永远是有条件的，不存在绝对价值 |

**GDR 设计启示**：当稀有卡出率高但平均价值低时，需要构建特别的 GDR 来反映真实收益（而非简单计数）。价值和数量可以相加，但价值的单位需要是「相当于多抽几张卡」。
