# 外部研究报告对 GachaStat 项目的启发分析

> 分析日期：2026-05-26
> 分析范围：两份外部研究报告——「风险决策理论在抽卡机制建模中的应用」与「玩家社区抽卡策略比较方法论」
> 目标：识别两份报告中对当前项目有直接启发价值的理论框架、方法论和具体改进点

---

## 摘要

两份报告分别来自行为经济学和计算社会科学视角：
- **报告一（风险决策理论）**：构建了从单池 MDP 到并发资源反馈网络的四层数学模型，引入前景理论（CPT）和后悔理论（RT）作为行为校准层
- **报告二（策略比较方法论）**：建立了六维策略评估体系，提出反事实遗憾分析、帕累托前沿、行为 Agent 对抗赛等具体比较方法

两份报告对 GachaStat 项目的启发可分为三个层次：**立即可落地的改进**（策略比较面板重构、风险评估维度补全）、**需要新增模块的中期改进**（保底基数资产化、行为经济学模块）、**长期理论参考**（MDP 最优策略、资源产出矩阵）。

## 第一部分：报告一（风险决策理论）核心框架与启发

### 1.1 四层模型总览

报告一提出了从简单到复杂的递进模型：

```
Layer 1: 单池 MDP（最优停止问题）
    ↓  保底机制引入状态记忆性
Layer 2: 跨期 DP（保底继承 → 基数资产化）
    ↓  垫刀转化为跨期期权
Layer 3: 并发网络（多池子 + 资源反馈）
    ↓  资源产出矩阵 O_m 建模闭环经济
Layer 4: 行为校准（CPT + RT）
    ↓  从数学最优到真实行为
```

### 1.2 启发一：保底基数资产化 ★★★

**报告原文**（§4.2）：保底计数器不是沉没成本，而是对未来池子的**期权投资**。

**当前项目状态**：
- `StrategyContext` 已传入 `future_schedules`（[strategy.py:23](gacha_simulator/core/strategy.py#L23)），但**所有 7 种策略均未使用该字段**
- 策略决策完全基于当前池状态，不知道未来还有什么池子
- `pity_reserve` 只看当前池保底概率是否达阈值，不比较「继续抽」vs「保留基数到下一池」

**可落地的改进**：

在 `StrategyContext` 中增加基数资产估值方法：

```
A_m(i) = c × (E[T_m | 0] − E[T_m | i])
```

即：已投入 i 抽所节省的期望抽取次数 × 单抽成本。对于线性递增保底曲线（70 抽后每抽 +6% 直至 90 抽 = 100%）：
- E[T | 0] ≈ 62.5 抽
- E[T | 50] ≈ 18.3 抽
- A(50) ≈ 44.2c —— 带着 50 抽基数进下一池，相当于节省了 44.2 抽

**具体做法**：
1. 在 `StrategyContext` 或 `PityEngine` 中新增 `compute_carryover_value(pool_id, pity_count)` 方法
2. 修改 `SmartStrategy` 和 `PityReserveStrategy`，在退出决策时纳入基数期权价值：
   - 若「当前池继续抽的边际期望」<「保留基数到下一池的期权价值」→ 退出

### 1.3 启发二：前景理论（CPT）行为校准 ★★

**报告原文**（§2.2, §6.1）：CPT 通过三个修正解释了抽卡中的「非理性」行为：
- **参照点依赖**：投入 80 抽未出 → 编码为损失，而非中性成本
- **损失厌恶**（λ ≈ 2.25）：亏 100 钻的心理痛苦 ≈ 赚 225 钻的心理愉悦
- **概率权重扭曲**：高估 0.6% 基础出货率（驱动「单抽出奇迹」），低估中高概率区间（软保底区过度乐观）

**当前项目状态**：
- 所有 GDR 指标、SuccessChecker、策略决策均基于客观期望值
- 无任何行为偏差建模

**可落地的改进**：

新增 `core/behavioral.py` 模块，提供：

```python
@dataclass
class BehavioralParams:
    loss_aversion: float = 2.25       # λ
    diminishing_sensitivity: float = 0.88  # α/β
    probability_weighting: str = 'tk92'

def cpt_value(delta: float, params: BehavioralParams) -> float:
    """前景理论价值函数 v(Δx)"""
    if delta >= 0:
        return delta ** params.diminishing_sensitivity
    else:
        return -params.loss_aversion * ((-delta) ** params.diminishing_sensitivity)

def probability_weight(p: float, params: BehavioralParams) -> float:
    """Tversky-Kahneman (1992) 概率权重函数"""
    ...

def cpt_utility(final_resources, initial_resources, params) -> float:
    """对一次模拟结果计算 CPT 效用"""
    delta = final_resources - initial_resources
    return cpt_value(delta, params)
```

然后新增两个行为 GDR：
- `cpt_utility`：用 CPT 价值函数替代线性净收益，捕获损失厌恶和边际敏感度递减
- `perceived_success_probability`：用概率权重函数 w(p) 扭曲后的「感知成功率」，解释为何玩家在低概率区过度乐观

### 1.4 启发三：后悔理论（RT）与跨池选择 ★

**报告原文**（§2.3, §6.2）：后悔 = 实际结果 − 「如果当初选择其他方案会怎样」的反事实结果。继承机制创造了三类独特后悔源：垫刀利用后悔、跨池分配后悔、并发时机后悔。

**当前项目状态**：
- 完全缺失

**可落地的改进**：

这一改进依赖**同种子配对设计**（所有策略在同一随机路径上运行），与项目中其他模块的独立种子架构有冲突。建议作为比较分析面板的可选分析模式（见报告二启发），而非策略系统的内置功能。

### 1.5 启发四：MDP/DP 最优策略 ★

**报告原文**（§3-5）：单池 MDP 可精确求解最优停止边界；跨期 DP 可求解全局资源分配。

**当前项目状态**：
- 7 种策略均为贪婪/启发式，不满足贝尔曼最优性（已在[全模块理论审查](全模块理论严谨性系统性审查.md)中确认）

**实际考量**：
- 状态空间爆炸（§5.2：M=3 时 |S| ≈ 5.8×10¹³）使精确 DP 不可行
- 报告提出的滚动时域启发式（H=2~3）是可行的折中方案，但每次配置变更都需重新计算投入-效用表
- **建议**：不追求实时的 MDP 求解器，而是将单池最优停止边界作为**标杆**——离线计算后，测量各启发式策略的效率损失

### 1.6 启发五：资源产出矩阵 ★

**报告原文**（§5.3）：定义 O_m 矩阵，第 l 行第 k 列 = 池子 m 的第 l 种结果产出资源 k 的数量。

**当前项目状态**：
- 项目的 Gain 系统（每日资源收入）和 Bonus 系统（第 N 次/超出 N 张额外资源）已部分覆盖资源产出
- 但产出是确定性的（条件满足必定产出），不区分概率
- 策略层面完全不感知「抽 A 池能产出 B 池资源」的反馈回路

**建议**：这是长期方向，依赖 Gain/Bonus 系统与策略系统的深度整合。当前不优先实施。

## 第二部分：报告二（策略比较方法论）核心框架与启发

### 2.1 六维评估体系 vs 当前策略比较

报告二提出策略比较应在六个维度上评估：

| 维度 | 报告二的指标 | 当前项目覆盖 | 差距 |
|------|------------|-------------|------|
| **数学效率** | 期望净收益、出货率、保底触发率 | 平均抽数、成功率、平均目标获得 | 缺净收益、出货率 |
| **风险特征** | 方差、最大回撤、CVaR₅%、出货间隔 | **完全缺失** | 无任何风险指标 |
| **鲁棒性** | 概率敏感度、价值敏感度、资源扰动 | **完全缺失** | 无压力测试 |
| **行为适配度** | 认知负荷、后悔期望、心理账户冲突 | **完全缺失** | 无行为建模 |
| **跨期适应性** | 价值重估弹性、资源再配置速度 | **完全缺失** | 无动态适应性测试 |
| **社会维度** | 晒卡防御力、可辩护性 | **完全缺失** | 无社交维度 |

**当前策略比较面板**（[strategy_comparison_panel.py](gacha_simulator/gui/strategy_comparison_panel.py)）仅覆盖数学效率维度的 5 个朴素指标（平均抽数、保底触发、目标获得、成功率、剩余资源），且指标选择缺乏理论依据。

### 2.2 启发六：反事实遗憾分析 ★★★

**报告原文**（§4.2）：对每条模拟路径，计算「如果采用其他策略会怎样」。

这是报告二最具原创性的方法论贡献。核心公式：

```
Path_Regret_i = max_{π' ≠ π_i} (V_path^{π'} − V_path^{π_i})
```

**实现前提**：需要在**同一随机路径**上运行多个策略，即所有策略共用同一组随机种子序列。

**当前项目状态**：
- 策略比较面板虽使用相同种子集（`seed=0` 起始，`[0, 1, ..., N-1]`），但各策略**独立运行**各自的模拟批次，并非在同一条随机路径上比较
- 每个策略的 N 次模拟各自产生 N 条独立路径，不存在「路径 i 上策略 A vs 策略 B」的配对

**可落地的改进**：

修改模拟引擎，支持「路径锁定模式」：
- 生成 N 组随机种子序列
- 对每个策略，在**完全相同的 N 组种子上**运行模拟
- 输出为每条路径上各策略的配对结果，计算后悔分布

**实际约束**：这一功能要求一次性运行所有选定策略（不能跨批次积累），与 ResultStore 的独立种子架构有张力。建议作为比较分析面板的一个可选模式（「配对比较模式」vs「分布比较模式」）。

### 2.3 启发七：帕累托前沿分析 ★★

**报告原文**（§4.3）：以期望收益为 X 轴、风险为 Y 轴、认知负荷为气泡大小，绘制策略散点图，标识帕累托前沿和被支配策略。

**当前项目状态**：
- 退路搜索面板（`retreat_search_panel.py`）已有 Pareto 前沿概念（资源-目标权衡），但策略比较面板完全没有
- 策略比较仅按成功率单维降序排列，无法展示多目标权衡

**可落地的改进**：

在比较分析面板中新增「帕累托散点图」：
- X 轴：期望收益（所有目标卡的价值加权获得率）
- Y 轴：风险（CVaR₅% 或收益方差）
- 气泡大小：策略的认知复杂度（规则数/分支数）
- 颜色：策略类型
- 前沿连线：标识非支配策略集

### 2.4 启发八：六大原型策略映射 ★★

报告二归纳了玩家社区的六大原型策略，映射到当前项目的 7 种策略：

| 社区原型 | 规则 | 当前对应 | 匹配度 |
|----------|------|----------|--------|
| 全囤型 | 只抽必抽池，其他完全跳过 | `NoDrawStrategy`（P7 新增，内部策略） | 部分——NoDraw 完全不动，全囤型是选择性全押 |
| 全垫型 | 每个池子垫到软保底前停手 | `fixed_count`（每池固定次数） | 部分——缺「保底前停手」的阈值感知 |
| 阈值型 | 「X 抽出就停」的硬规则 | `pity_reserve`（保底概率 ≥ 阈值才抽） | 近似——但逻辑不同：社区阈值是「抽 N 次不出就停」，`pity_reserve` 是「概率高才抽」 |
| 价值型 | 按社区评测的 T 度表分配资源 | `smart`（按需追卡） | 良好——`smart` 策略根据目标卡需求决定 |
| 反馈型 | 优先抽能产出其他池子资源的池子 | **无** | **缺失** |
| 情感型 | XP 抽卡，无视数值 | `target_hunting`（指定池追卡） | 近似 |

**关键缺口——反馈型策略**：当前没有任何策略利用「抽 A 池产出 B 池资源」的反馈信息。`StrategyContext` 不暴露资源产出信息。这是唯一一个在当前策略体系中没有对应物的社区原型。

### 2.5 启发九：行为 Agent 对抗赛 ★

**报告原文**（§4.4）：用参数化的 CPT/RT Agent 执行策略，测量「执行衰减率」——同一策略在理性 Agent 和行为 Agent 下的表现差距。

**可落地的改进**：

当行为经济学模块（`behavioral.py`）实现后，可在比较分析面板中新增「行为模式」选项：
- 选择一种 Agent 类型（理性/损失厌恶/后悔厌恶/冲动）
- 用该 Agent 的 CPT/RT 效用函数重新评估所有数据集的「感知收益」
- 输出策略-Agent 匹配矩阵：哪些策略对哪些玩家类型最友好

### 2.6 启发十：鲁棒性压力测试 ★★

**报告原文**（§3.3）：对每种策略在扰动后的环境集合中重新评估，计算鲁棒性得分。

**与前文的关联**：这正是项目中未实现的「敏感度分析」Tab 的核心内容。报告二给出了具体可操作的测试框架：

| 扰动类型 | 测试方式 | 含义 |
|----------|----------|------|
| 概率敏感度 | 保底概率曲线 ±10% | 对官方暗改的抗性 |
| 价值敏感度 | 目标卡价值 ±20% | 对评测误差的抗性 |
| 资源扰动 | 初始资源随机波动 | 对肝度/氪度差异的适应性 |
| 序列扰动 | 池子开放顺序打乱 | 对规划外新池的适应性 |

**可落地的改进**：

作为比较分析面板的「压力测试模式」：
1. 用户选择 1 个基准数据集 + 扰动参数
2. 系统自动生成扰动环境变体
3. 在每个变体环境中重跑策略
4. 输出鲁棒性得分矩阵 + 最脆弱环节标注

## 第三部分：两报告交叉分析

### 3.1 共同指向的改进

两份报告从不同角度指向了同一个核心缺失：**策略比较缺乏多维度的系统化评估框架**。

| 报告一贡献 | 报告二贡献 | 交汇点 |
|-----------|-----------|--------|
| CPT 效用函数 | 行为适配度维度 | 行为经济学模块 |
| 保底基数资产化 | 跨期适应性维度 | 策略需感知跨期价值 |
| 后悔理论 | 反事实遗憾分析 | 同种子配对比较 |
| 资源产出矩阵 | 反馈型策略 | 策略需感知资源反馈 |

### 3.2 相互补充的改进

| 报告一独有 | 报告二独有 |
|-----------|-----------|
| 单池 MDP 最优停止 | 帕累托前沿可视化 |
| 跨期贝尔曼方程 | 六维评估体系 |
| 概率权重函数 w(p) | 社区六大原型策略 |
| 行为偏差校准（λ, α, β） | 策略-Agent 匹配矩阵 |

### 3.3 潜在的矛盾

1. **同种子 vs 独立种子**：
   - 报告二的反事实遗憾分析**要求**同种子配对设计
   - 报告一的 MDP 框架**不关心**种子设计，关注的是策略函数本身的性质
   - 在项目架构中（ResultStore 跨批次独立存储），独立种子是更自然的选择；反事实遗憾作为可选模式

2. **理论最优 vs 行为适配**：
   - 报告一给出了 MDP 最优解，隐含「存在唯一最优策略」
   - 报告二强调「没有唯一最优，只有帕累托前沿上的适配选择」
   - 报告中已有调和方案（行为 Agent 对抗赛），可同时展示理性排名和行为排名

## 结论

两份报告为 GachaStat 项目提供了两个互补的理论基础：报告一提供了「如何建模抽卡决策的数学本质」，报告二提供了「如何系统化地比较不同策略」。两者共同揭示的核心缺失是：**项目当前的策略比较缺乏多维度的系统化评估框架**。

最紧迫的改进是**将策略比较从一维（成功率排序）升级为多维评估体系**（效率 + 风险 + 鲁棒性 + 随机占优），这与已经设计的「比较分析面板重构」计划方向完全一致。两份报告为该计划提供了具体的分析方法论支撑。
