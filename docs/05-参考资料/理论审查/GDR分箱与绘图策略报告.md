# GDR 分箱与绘图策略报告

> 基于离散性分类体系，对全部 17 种 GDR 逐一给出分箱和绘图建议。
> 编写日期：2026-05-25

---

## 一、分类体系回顾

| 类型 | 支撑集 | 特征 |
|------|--------|------|
| **A. 有限支撑** | 固定有限集 | 取值空间不随 N 扩大 |
| **B. 无限格点** | {a + k·Δ}，可能叠加有限种偏移 | 范围随 N 扩大，步长 Δ 恒定 |
| **C. 渐近稠密** | 有理比值 p/q，p,q 随 N 增长 | 格点间距 → 0 |

B 类细分为 B₁（格点稀疏，梳状可辨）和 B₂（格点密集，梳状不可辨）。当前框架无 D 类（真连续）——所有输入均为离散量，日收入只能产生有限种偏移，无法形成密度。

---

## 二、当前绑图实践概述（`analysis_panel.py`）

**`_hist_params` 方法（line 1801-1826）**：
1. 唯一样本 ≤1 → 单 bin
2. `_is_discrete` 判定为整数 → `range(lo, hi+1)` 逐整数分箱，`density=False`
3. IQR/span < 0.05（集中分布）→ 50~200 固定 bin 数
4. 其他 → Freedman-Diaconis 规则

**`_is_discrete` 方法（line 1784-1787）**：仅检测「所有值是否接近整数」（`abs(v − round(v)) < 1e-9`）。

**已知不足**：
- 只识别步长=1 的整数离散，不识别步长≠1（如 cost=160）或非整数格点（如 0, 0.33, 0.67, 1.0）
- 无分箱起点偏移逻辑——始终从 `lo` 开始
- 山脊线图（line 1229-1230）统一用 FD 分箱，不考虑离散结构
- 单 GDR 分布图不叠加 KDE，仅条件分布图（line 443）用 `density=True` 直方图

## 三、逐 GDR 分析

### A 类：有限支撑

#### 1. `all_targets` — 抽出全部目标卡

| 维度 | 分析 |
|------|------|
| **类型** | A（二元 {0, 1}） |
| **值域** | {0.0, 1.0} |
| **Δ** | 1.0（唯一非零间距） |
| **分箱起点** | 不适用——直接按值统计 |
| **bin 宽度** | 不需要分箱 |
| **绘图** | 饼图 或 直接标注「成功率 = P(1)」的数值 + 条形图 |
| **KDE** | 不适用 |
| **特殊值** | 无 |

**当前做法**：`_hist_params` 中 `_unique_count ≤ 1` 时走单 bin 路径——但实际通常有 2 个值（0 和 1），`_is_discrete` 返回 True 走 `range(0, 2)` → bins=[0,1,2]，正确。✅

#### 9. `resource_consumed` — 资源消耗

| 维度 | 分析 |
|------|------|
| **类型** | B₁（cost 的整数倍，Δ=cost_per_draw，通常 160） |
| **值域** | {0, 160, 320, …} |

#### 10. `resource_remaining` — 资源剩余

| 维度 | 分析 |
|------|------|
| **类型** | B₁（通常情况）。格点步长 Δ=cost_per_draw，叠加 K 种日收入偏移（K=池排期时间点数，通常 ≤10） |
| **值域** | [0, Q₀ + max_income]，浮点 |
| **Δ** | cost_per_draw（典型 160） |
| **分箱起点** | R₀ = median(r mod cost)。若 std(residuals) < 0.3×cost 则偏移对齐 bin 边界到 R₀+k×cost；否则直接用 `floor(r/cost)` 分箱。**日收入多偏移自动被 floor 吸收** |
| **bin 宽度** | cost_per_draw（若 B₁）；FD 规则（若 B₂） |
| **绘图** | B₁：直方图(bin=cost) + ECDF。B₂：直方图(FD) + ECDF。均不用 KDE |
| **KDE** | ❌ 不适用（离散格点） |
| **特殊值** | `use_draw_units` 模式下除以 cost_per_draw 转为「剩余抽数」，变为 Δ=1 的 B₁ |

**当前做法**：`_is_discrete` 返回 False（浮点数），走 FD 分箱。**不合理**——忽略了 cost_per_draw 的自然格点结构，bin 边界随机切割梳齿。山脊线图同样走 FD。

**建议修正**：新增「步长检测」——若 `std(r mod cost) < 0.3×cost`，识别为步长=cost 的格点分布，按格点分箱。否则回退 FD。

## 五、改进建议优先级

### 高优先级（当前做法有实质性错误）

1. **A 类 GDR（#2~4）有限格点检测**：在 `_is_discrete` 之外新增「有限有理格点」检测——若 `len(unique(samples)) ≤ 20` 且所有值为 k/T 形式，走条形图而非直方图。

2. **`resource_consumed`（#9）步长检测**：`_is_discrete` 不能只检测 Δ=1，需检测任意固定步长。若 `std(r mod Δ) < 0.3Δ`，识别实际步长并按 Δ 分箱。

3. **`resource_remaining`（#10）格点结构**：同上，检测 cost_per_draw 步长，按格点分箱。日收入偏移自动被 `floor(r/cost)` 吸收。

4. **`resource_per_card`（#12）inf 处理**：分箱前排除 inf，图表中标注 inf 占比。

### 中优先级（改善但不紧急）

5. **C 类 GDR 叠加 KDE**：为 `resource_efficiency`、`weapon_character_ratio`、`draw_conversion_efficiency` 等 C 类 GDR 的直方图叠加 KDE 曲线。

6. **分箱起点偏移**：B₁ 类检测 R₀ = median(r mod Δ)，bin 边界对齐到 R₀+k·Δ。

### 低优先级（锦上添花）

7. **`all_targets` 专用展示**：二元 GDR 用成功率数字代替直方图。
