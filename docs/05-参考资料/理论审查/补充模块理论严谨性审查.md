## 一、最好/最差情形分析（分析面板中的 risk_best_case / risk_worst_case）

**涉及文件**：`distribution.py`（BestCaseAnalysis/WorstCaseAnalysis）、`risk_analysis.py`（RiskAnalyzer）、`analysis_panel.py`（risk_best_case/risk_worst_case 分析类型）

> **注**：`worst_impact.py` 中的 `WorstImpactAnalyzer`（最差影响分析引擎 + 连续虚拟池模拟）属于独立模块，已在[原审查 §2.8](全模块理论严谨性系统性审查.md)中覆盖。本节仅审查分析面板中 `risk_best_case` / `risk_worst_case` 所使用的分布层和风险分析层。

**功能概述**：两个分析类型为任何 GDR 指标（或自定义分布）提供尾部统计：
1. **分布层**（`distribution.py`）：`WorstCaseAnalysis`——VaR/CVaR + 条件尾部子分布；`BestCaseAnalysis`——上分位数 + 条件顶部子分布
2. **风险分析层**（`risk_analysis.py`）：`RiskAnalyzer` 将分布层方法封装为面向 GDR 指标的报告生成器（`worst_case_analysis()` / `best_case_analysis()` / `full_report()`）

### 1.1 合格项

- **CVaR（条件风险价值）的实现**：`EmpiricalDistribution.cvar(alpha)` 取排序样本底部 α 尾部的条件均值。CVaR 是 Rockafellar & Uryasev (2000) 引入的一致性风险度量（满足次可加性、正齐次性、单调性、平移不变性），是当前金融风险管理的监管标准（巴塞尔协议 III）。

- **条件分布的语义正确性**：`WorstCaseAnalysis.conditional_tail()` 返回底部 α 分位数以下的子分布，`BestCaseAnalysis.conditional_top()` 返回顶部 α 分位数以上的子分布。这两个操作在语义上等价于「以分布尾部为条件」——对理解「失败时损失多少」「成功时获得多少」有直接的决策价值。

### 1.2 理论缺陷

**缺陷 B1：BestCaseAnalysis 缺乏理论基础与对称性论证（中等）**

`BestCaseAnalysis` 简单地取 `quantile(1-alpha)` 作为「最好情形」分位数。然而，风险度量文献中不存在「最佳情形风险度量」这一概念——Stone (1973) 和 Fishburn (1977) 的风险定义仅关注损失侧。分布的上尾（高收益侧）的统计性质与下尾（高损失侧）不对称：

- 下尾由 VaR/CVaR 的数学理论支撑（Artzner et al., 1999; Rockafellar & Uryasev, 2000）
- 上尾缺乏对偶的公理化框架——「最好情形」分位数只是上分位数的另一个名称

当前实现没有文档化这一不对称性，用户可能误以为 BestCaseAnalysis 具有与 WorstCaseAnalysis 同等的理论基础。建议在 UI 中标注为「乐观情形」而非「最好情形」，并补充说明其估计方差在相同 α 水平下与最差情形不对称（取决于分布的偏度）。

**缺陷 B2：经验分位数在尾部的不稳定性未被量化（严重）**

`WorstCaseAnalysis.cvar(alpha=0.05)` 和 `BestCaseAnalysis.upper_quantile(alpha=0.05)` 直接使用排序样本的经验分位数，在 n=1000 时尾部仅使用 50 个样本点。经验分位数的渐近方差为：

```
Var(quantile(p)) ≈ p(1-p) / (n · f(F⁻¹(p))²)
```

其中 f 是密度函数。在尾部（p 接近 0 或 1），f(F⁻¹(p)) 通常较小（分布密度在尾部递减），方差远大于中位数。对于 α=0.01（n=1000 时仅 10 个样本），经验分位数的估计误差可达分布的 10-20%。

当前代码对此完全没有处理——无置信区间、无 bootstrap 标准误、无极端值理论（EVT）的尾部外推。用户看到 `CVaR(0.05) = 7500` 时，完全不知道这个估计的精度（可能是 7500 ± 2000）。

**缺陷 B3：CVaR 的离散 ceil 偏差（已在原审查中记录为缺陷 1，此处仅确认）**

`int(math.ceil(alpha * self._n))` 导致尾部比例系统性膨胀。原审查已将此项列为中等缺陷。

**缺陷 B4：空分布的静默返回（已在原审查中记录为缺陷 2，此处仅确认）**

`n=0` 时所有方法返回 0.0 而非 NaN。下游代码无法区分「分布均值为 0」和「分布为空」。

**缺陷 B5：RiskAnalyzer 的 full_report 是同步阻塞的（低等）**

`full_report()` 在 GUI 线程中同步计算所有分布的 VaR、CVaR、最差/最好情形，对大规模数据集（n>5000）可能产生可感知的 UI 卡顿。但该函数不在主流 GUI 路径中使用（主流路径使用 QThread Worker），实际影响有限。

**缺陷 B6：BestCaseAnalysis 的条件顶部分布缺乏样本量下限（低等）**

`conditional_top_alpha(alpha=0.05)` 在 n<20 时返回空分布（0 个样本满足 top α 条件）。当 n=1000 时仅 50 个样本，在长尾分布中这些样本的分布极不稳定。没有文档或代码守卫提示用户样本量不足。

### 1.3 文献对照

| 概念 | 文献 | 一致性 |
|------|------|--------|
| VaR / CVaR 尾部风险度量 | Artzner et al. (1999), *Mathematical Finance*; Rockafellar & Uryasev (2000), *Journal of Risk* | ✅ VaR/CVaR 定义正确 |

## 二、转变分析

**涉及文件**：`per_pool_analysis.py`（TransitionMatrix/compute_transition_matrices_from_flags/compute_transition_flags_from_gdr）、`streaming.py`（DrawSequenceExtractor/WorkerLocalExtractor 中的 transition_flags 预计算）、`analysis_panel.py`（transition_analysis 可视化）、`process_analysis.py`（AA/BB/AB/BA 交叉分析中的相关部分）

**功能概述**：对相邻卡池间的成功/失败状态转移进行建模。核心数据结构为 2x2 转移矩阵 `TransitionMatrix`（成功→成功、成功→失败、失败→成功、失败→失败），通过计数比例直接估计。三种可选的成功判定标准（all_targets / any_ssr / per_pool_target）和两种范围模式（cumulative / single_pool）。

### 2.1 合格项

- **2x2 马尔可夫转移矩阵的形式**：以离散时间、有限状态马尔可夫链建模池间状态转移在概念上是正确的。Anderson & Goodman (1957) 证明了转移计数的 MLE 是转移概率的一致估计量。当前实现使用简单比例计数 `P(成功|之前成功) = N_ss / N_s`，这是转移概率的 MLE。

- **纯函数设计**：`compute_transition_matrices_from_flags(success_flags_per_sim, pool_ids_ordered)` 接受布尔矩阵、返回 `TransitionMatrix` 列表，无外部依赖、无副作用。这种设计使其易于测试、易于替换底层成功判定逻辑、易于并行化。

- **流式预计算**：`DrawSequenceExtractor` 和 `WorkerLocalExtractor` 在模拟过程中同步构建 `transition_flags`（布尔成功/失败矩阵），避免了对完整模拟历史的二次遍历。这在工程上是正确的——减少了分析阶段的计算和内存开销。

- **三种成功标准的灵活性**：`all_targets`（严格累积目标达成）、`any_ssr`（宽松的至少一个 SSR）、`per_pool_target`（每池独立目标达成）覆盖了从宽松到严格的连续谱，允许用户按需选择分析粒度。

### 2.2 理论缺陷

**缺陷 T1：转移概率的简单比例估计无不确定性量化（严重）**

当前实现将转移概率报告为单一数值，无标准误、无置信区间、无假设检验：

```python
success_to_success = ss / s_before if s_before > 0 else 0
```

对于一个 2x2 转移矩阵，每行是一个二项比例（给定之前状态，之后状态的条件概率）。二项比例的 MLE 的标准误为 `sqrt(p(1-p)/n)`。在典型模拟规模下（n=1000）：

- 若 `P(成功|之前失败) = 0.3`，则 SE ≈ 0.015，95% CI ≈ [0.27, 0.33]
- 若 `P(成功|之前成功) = 0.8`，则 SE ≈ 0.013，95% CI ≈ [0.77, 0.83]
- 极端情况（`P=0.95`，`n_before=200`）：SE ≈ 0.015，但正态近似在边界处失效

更严重的是，转移矩阵各行构成一个**多项分布**（行内概率和为 1）——一个单元格的方差必然伴随其他单元格的补偿变化。当前的不相关比例估计忽略了这一约束。

**文献支撑**：Craig & Sendi (2002, *Health Economics*) 推荐 Bootstrap 方法估计转移矩阵置信区间；Möstel, Pfeuffer & Fischer (2020, *Computational Statistics*) 系统比较了多项同时置信域（Sison-Glaz, Goodman, Quesenberry-Hurst）和 Bootstrap 方法。

**缺陷 T2：零分母静默归零（中等）**

```python
success_to_success = ss / s_before if s_before > 0 else 0
```

当 `s_before = 0`（之前无成功）时，代码返回 0.0。但除零是一种数据缺失——我们不知道「如果之前成功会怎样」，而不是「之前成功后必然失败」。静默返回 0.0 会误导用户相信 `P(成功|之前成功) = 0%`，而非「数据不足以估计」。

类似地，若所有模拟之前都成功（`s_before = n_sims`），`fs / (n_sims - s_before)` 除零返回 0，但「所有人之前都成功」本身就意味着某种 ceiling effect——可能之前池太容易，或者用户资源极其充裕。

**缺陷 T3：一阶马尔可夫假设可能不成立（严重）**

当前转移分析假设池间状态转移满足马尔可夫性质：下一池的成功/失败仅依赖于上一池的状态。在抽卡游戏中，这一假设在两个维度上可能不成立：

1. **保底状态记忆**：保底计数器积累跨越多池。如果 B 池的保底计数器从 A 池继承了 70 抽（80 抽触发软保底），则 A→B 的转移概率不仅取决于 A 的成败，还取决于 A 结束时的保底计数器值。一阶成败空间无法捕捉这一信息。

2. **资源消耗路径依赖**：A 池消耗的资源量（而非仅成败二元状态）决定了 B 池可用的预算。如果 A 池「险胜」（消耗了 80% 资源）vs「轻松获胜」（消耗了 30% 资源），B 池的成功概率截然不同，但当前模型将它们等同为 `before=True`。

实践中，这一假设的违反程度取决于池子数量、资源充裕度和保底继承规则的激进程度。建议至少补充一个**信息性诊断**：计算「给定之前状态」的子群的资源分布均值/方差，若差异显著（如 30%+），则为用户标注马尔可夫假设可能被违反。

**缺陷 T4：多池转移的多重比较问题（中等）**

对于 K 个有顺序的池，生成 K 个转移矩阵（每个池间转移一个）。每个转移矩阵有 4 个概率值。如果用户检查所有 K×4 个值并关注「显著的」变化（如 P(成功|失败) 在某个转移处突然下降），这就构成了多重比较——检验了 K 个假设但未进行任何校正。

对于典型的 5-10 个池子，Bonferroni 校正会显著降低检出真实差异的功效，但完全不校正又会产生假阳性风险。建议在转变分析面板中标注「此为探索性分析，非确认性检验」，并在未来版本中实现 FDR 校正。

**缺陷 T5：事件-成败转变分析与池-池转变分析的重复信息与冗余（低等）**

`process_analysis.py` 中的 AB 分析（事件→成败）计算 `P(成功|事件模式)`，这与池-池转变分析在概念上有重叠——「事件」维度中的 `pity_hit`/`early_hit`/`miss` 等信息本身就编码了池子内部的过程，而池-池转变仅看最终的成败二元结果。两种分析向用户呈现不同粒度的信息，但 UI 层面没有说明两者的关系（是互补还是替代），用户可能因信息重复而感到困惑。

### 2.4 改进方向：在不膨胀状态空间的前提下补充转移强度信息

当前二元状态（成功/失败）的一个关键局限（见缺陷 T3）是丢失了转移的「强度」——「险胜」和「轻松获胜」被等同对待。在不引入分箱/KDE 膨胀状态空间的前提下，可在 `TransitionMatrix` 中附加两个连续度量：

- **之前成功组的平均资源消耗**：`mean({资源消耗_i | before_i=True})`
- **之前失败组的平均资源消耗**：`mean({资源消耗_i | before_i=False})`

这两个值的差异直接量化了「成败的资源代价差异」。若差异小（如 <10%），说明成败主要由运气驱动，转移矩阵的马尔可夫解释可信；若差异大（如 >50%），说明资源消耗路径依赖严重，为马尔可夫假设违反提供定量证据。此方案不增加状态空间（仍为 2×2），计算成本可以忽略（两次均值聚合），且直接利用了模拟过程中已有的资源消耗数据。

## 三、最少资源与最多目标卡分析

**涉及文件**：`retreat_search.py`（RetreatSearchEngine）、`resource_search_panel.py`（ResourceSearchWorker/二分搜索）、`retreat_search_panel.py`（方案搜索 GUI）、`forward_backward.py`（数据类）、`retreat_config.py`（截断配置构建器）、`vulnerability.py`（脆弱性分析——退路分析的输入）

**功能概述**：三个搜索算法回答「退路」问题——
1. **最少额外资源**（search_min_resource）：给定目标卡集合，需要多少额外资源才能在截断后的剩余卡池中达到给定的成功率阈值？使用翻倍上界 + 二分搜索。
2. **最多目标卡**（search_max_targets）：零额外资源下，最多能保留多少目标卡？使用贪婪后退法（按错失代价权重升序逐一移除）。
3. **Pareto 前沿**（search_pareto）：生成额外资源 vs 保留目标卡数量的权衡曲线。对每个子集调用 search_min_resource。

### 3.1 合格项

- **截断配置构建的正确性**：`RetreatConfigBuilder.build()` 正确地将原始配置截断到指定池之后，偏移天数、重置初始资源和保底计数器。这是「退路分析」的基础——分析「从这一刻起」的场景而非全时间线。

- **翻倍上界的合理性**：在未知所需资源量的情况下，从初始猜测开始指数扩展上界是标准做法。对于成本=160 的池，初始搜索 `50 * 160 = 8000` → 翻倍至 51200 抽的最坏路径覆盖了绝大多数场景。

- **贪婪后退法的方向正确性**：按错失代价权重升序（最不重要优先移除）是合理的方向——优先牺牲「代价最小」的目标卡。这一思路与运筹学中的贪心覆盖算法思路一致（Chvátal, 1979）。

### 3.2 理论缺陷

**缺陷 R1：二分搜索作用于蒙特卡洛噪声目标函数——收敛无保证（严重）**

`search_min_resource` 的核心是一个标准二分搜索：

```
while r_hi - r_lo > epsilon:
    r_mid = (r_lo + r_hi) / 2
    prob = simulate(r_mid)   # 500 次蒙特卡洛模拟 → 带噪声的 p̂
    if prob >= threshold:
        r_hi = r_mid
    else:
        r_lo = r_mid
```

问题：`prob` 是一个随机估计，标准差约为 `sqrt(p(1-p)/500)`。当真实成功率接近阈值 p* 时，SE≈0.022（p*=0.95），二分决策约有 22% 的概率**错误选择搜索方向**。

更精确的分析（Waeber, Frazier & Henderson, 2013, *Operations Research*）：在随机目标函数的二分搜索中，单步决策错误概率 q 导致最终估计的期望绝对误差至少为 `O(q · 区间宽度)`，而非确定性的 `O(区间宽度 / 2^K)`（K 为迭代次数）。换言之，噪声将二分搜索的对数收敛率退化为线性。

实际后果：
- 同一数据集的两轮搜索可能返回差异显著的最小资源估计（如 12000 vs 14500）
- 当 `epsilon = cost_per_draw = 160` 时，噪声误差（~±500）远超精度目标

**文献支撑**：Waeber et al. (2013) 提出了带随机目标函数的二分搜索的修正方案——在每步多次采样以减少决策错误概率；Bechhofer, Santner & Goldsman (1995, 专著) 提出了序贯选择程序（如 Paulson 程序、KN 程序）来保证在随机目标函数下以预设概率选择正确决策。

**推荐改进方案：序贯概率比检验（SPRT）替代固定 N 采样**。当前每步固定运行 500 次模拟后做出二分决策。更高效的做法是序贯采样——持续模拟直到有足够的统计证据做出决策：

```
while r_hi - r_lo > epsilon:
    r_mid = (r_lo + r_hi) / 2
    # 序贯采样：继续模拟直到能做出置信决策
    while not decided:
        run_one_simulation(r_mid)
        p_hat = current_success_rate
        # SPRT: 若 p_hat 的置信区间完全高于或低于 threshold，则决策
        if ci_lower(p_hat, n) > threshold → r_hi = r_mid (prob 显著 > threshold)
        if ci_upper(p_hat, n) < threshold → r_lo = r_mid (prob 显著 < threshold)
```

优势：
- 远离阈值的点（如 p≈0.3 或 p≈0.99）只需很少的模拟即可做出决策（可能 <100 次）
- 靠近阈值的点自动分配更多模拟（可达 2000+ 次），确保决策可靠性
- 总体模拟次数通常比固定 N=500 的方案减少 40-60%（Waeber et al., 2013）
- 提供了自然的不确定性量化——每步决策附带置信水平

此方案不改变二分搜索的整体结构，仅替换内层的固定 N 采样循环。可作为一个低风险的增量改进。

**缺陷 R2：贪婪后退法不是真正的 Pareto 前沿——声称有误导性（严重）**

`search_max_targets` 和 `search_pareto` 都被描述为「Pareto 最优」——在 UI 中称为「Pareto 前沿」搜索模式。但实际上：

1. **search_max_targets** 仅探索 N+1 个点（全量 → 按顺序移除一张 → 再移除一张 → ... → 空集），而非 2^N 个可能的卡子集。真正的 Pareto 前沿在目标卡数量 K×资源 R 的二维空间中可能包含多达 N 个非支配点，而贪婪路径可能完全错过。

2. **移除顺序是固定的**（按错失代价权重升序），不重新考虑已移除的卡。在某些场景中，移除两张低权重卡可能不如移除一张高权重卡（如果高权重卡后面对应的池消耗极高资源）——但算法不会探索这种可能性。

3. **不执行支配性过滤**：即使探索到的点中，`search_pareto` 也不检查一个点是否被另一个点严格支配（如点 B 卡更多 AND 资源更少）。被支配点仍被作为「前沿」呈现。

这一问题的性质与原审查中的缺陷 24（退路搜索的 Pareto 定义不严格）一致，但此处是通过搜索算法构造前沿，而非从已有数据中提取。

**文献标准**：真正的多目标 Pareto 前沿求解需要（Miettinen, 1999, *Nonlinear Multiobjective Optimization*）：
- 探索全部可行解空间（或证明缩小空间的等价性）
- 应用支配性过滤：点 A 支配点 B 当且仅当 A 在所有目标上 ≥ B 且至少在一个目标上 > B
- 至少保证非支配集是真实 Pareto 前沿的 ε-近似

当前算法不满足以上任何一条。建议在 UI 中将「Pareto 前沿」改为「资源-目标权衡曲线（贪心近似）」，并添加支配性过滤。

**缺陷 R3：翻倍上界搜索可能失败且无回退机制（中等，已在原审查中记录为缺陷 25）**

从 50 抽开始翻倍 10 次 → 最坏搜索范围为 50×2^10 = 51200 抽。若实际所需超过此值（例如极端低保底概率 + 极高目标要求），`r_hi` 停留在 51200 抽，但实际需求可能是 80000 抽。二分搜索在无效区间上运行，返回的点声称 `概率 ≈ threshold` 但实际不满足。

`resource_search_panel.py` 中的独立实现翻倍 15 次（50×2^15 ≈ 1,638,400 抽），上限更高但仍可能被突破。两个实现的翻倍次数不一致（10 vs 15），且均无理论推导的最坏情况保证。

**缺陷 R4：每步独立模拟不共享信息——效率极低（中等）**

`search_pareto` 对每个目标子集调用一次 `search_min_resource`，后者在二分搜索的每一步运行 500 次模拟。对于 N=5 张目标卡，最坏情况：
- 6 个子集（完整 + 移除 1/2/3/4/5 张后）× 平均 8 步二分 × 500 模拟 = **至少 24,000 次模拟**
- 所有模拟独立运行，不利用之前的采样信息

更高效的方案是采用**随机克里金（Stochastic Kriging）**或**贝叶斯优化**——将模拟结果视为对黑箱函数 `p(resource, target_set)` 的噪声观测，使用高斯过程代理模型预测未探索区域的概率，指导下一探索点的选择（Frazier, 2018, *Operations Research*）。这种方法可在数倍的样本量下达到相同精度。

**缺陷 R5：错失代价权重是启发式序数度量——缺乏基数解释（中等）**

`search_max_targets` 按错失代价权重升序移除目标卡。但错失代价权重由用户主观输入（默认 1.0），既无理论校准也无基数含义：
- 移除 `miss_cost=1.0` 的卡 A vs 移除 `miss_cost=1.5` 的卡 B——移除顺序是确定的（A 先于 B），但系数 1.5 的「增量」无实际含义
- 如果用户将所有权重设为相同的默认值，算法按卡 ID 字母序移除——这完全是任意的

在多目标决策理论中，权重应由 Swing Weighting（von Winterfeldt & Edwards, 1986）或 Analytic Hierarchy Process（Saaty, 1980）等方法系统引出。就当前目的而言，至少应在 UI 中标注权重仅影响**移除顺序**而非绝对价值。

**缺陷 R6：epsilon 精度设置缺乏统计依据（低等）**

`precision_draws=1` → `epsilon = cost_per_draw * 1`。对于 160 成本的单抽，这意味着算法在资源区间 < 160 时停止搜索。但蒙特卡洛噪声的标准差通常远大于此（在 p=0.5 时约 0.022×总资源值）。以精度 160 搜索而噪声水平为 ±500 是不一致的——类似于用游标卡尺测量一块橡皮泥的宽度。

## 四、每池GDR分析

**涉及文件**：`process_trace.py`（PoolEvent/SampleTrace/infer_events/compute_pool_gdr_cumulative/compute_pool_gdr_single_pool）、`process_analysis.py`（compute_aa/compute_bb/compute_ab/compute_ba）、`per_pool_analysis.py`（PoolSnapshot/CumulativeSnapshot/各类汇总函数）、`gdr.py`（per_pool_draw_rate/target_card_draws/GDRContext）

**功能概述**：将全局 GDR 分析分解到每个卡池级别，计算：
1. **每池事件推断**：7 种事件类型（pity_hit/early_hit/miss/skip/ignore/exchange/no_exchange）+ 资源池/兑换池的扩展类型
2. **每池 GDR 值**：两种模式——累积截止到该池 vs 仅该池独立计算
3. **每池成功判定**：GDR 值 ≥/≤ 阈值 → `pool_success[pool_id]` 布尔标记
4. **交叉分析**：AA（事件模式分布）、BB（成败模式分布）、AB（事件→成败条件概率）、BA（成败→事件条件概率）

### 4.1 合格项

- **事件分类体系的完备性**：基于三个正交维度（是否抽卡 × 是否获得目标 × 是否触发保底）加池类型分支，7+4 种事件类型在逻辑上互斥且完备。这一分类与离散时间随机过程的「状态标注」（state labeling）实践一致。

- **双路径推断等价性**：`_infer_from_draw_sequence`（从逐抽序列）和 `_infer_from_aggregate`（从汇总统计）的核心判定逻辑等价——draws/target_count/pity_count/skip_ignore 判定一致。这确保了模拟模式之间的可比性。

- **AB/BA 的贝叶斯对称性**：AB 计算 P(成功|事件)，BA 计算 P(事件|成功) 和 P(事件|失败)。`P(成功|事件) = P(事件|成功) × P(成功) / P(事件)` 的贝叶斯关系自然成立，两种分析构成完整的预测-诊断框架。

- **Laplace 平滑的采用**：`compute_ab` 中使用 `(success+1)/(total+2)` 而非原始比例，避免了在极少出现的事件模式上产生 0% 或 100% 的极端估计。相比无平滑的原始 MLE，这是对小样本条件下概率估计的实质性改进。

- **累积与单池双模式**：`compute_pool_gdr_cumulative`（累积：反映截至该池的进展）和 `compute_pool_gdr_single_pool`（单池：反映该池独立表现）提供互补视角——前者回答「走到这一步时达成了多少」，后者回答「这个池本身效率如何」。

### 4.2 理论缺陷

**缺陷 P1：每池GDR值的多重比较问题——K 个池 × N 次模拟的探索性检验（严重）**

当前分析流程：
1. 对每个池计算 K 个 GDR 值（每池一个）
2. 将每个值与全局 GDR 阈值比较 → K 个布尔成功判定
3. 在各面板上展示每池成功率
4. `compute_bb` 进一步测试「哪些池的成功率组合是典型的」

问题：用户观察 K=8 个池的每池成功率并关注「哪个池最难」，这正是多重比较的场景——检验了 8 个假设（每个池的「真实成功率是否等于全局平均」），FWER（族错误率）在 α=0.05 时为 `1-(1-0.05)^8 ≈ 34%`。

更微妙的是，各池的成功与否并非独立——早期池的高消耗会损害后续池的成功率，产生负相关结构。传统的 Bonferroni 校正在此场景下过于保守，而标准的 FDR 方法需要检验独立性假设。

**文献支撑**：Benjamini & Hochberg (1995, *JRSS-B*) 的 FDR 控制程序在检验正相关条件下仍然有效（Benjamini & Yekutieli, 2001, *Annals of Statistics*）。

**缺陷 P2：每池 GDR 值的独立性假设被违反——Simpson 悖论风险（严重）**

全局 GDR 与每池 GDR 之间存在类似 Simpson 悖论的结构性风险。考虑以下场景：
- 池 A（角色池）：成功率 95%，消耗 60% 资源——容易成功
- 池 B（武器池）：成功率 40%，消耗 40% 资源——高难度
- 全局成功率：85%

用户看全局 GDR → 85%，满意。看每池 GDR → 发现池 B 仅 40%。这两者不矛盾——但系统没有提醒用户**全局成功可能由容易的池子驱动，掩盖了高难度池子的低成功率**。

这一问题与医学文献中报告的「聚集 AUC 高于各亚组 AUC 的 Simpson 悖论」（MedRxiv 2025 论文报告了 5-16 百分点的差距）在结构上完全类似——聚合度量掩盖了组内异质性。当前代码完全没有对此问题的诊断。

建议：在每池分析面板中增加「异质性诊断」——若各池成功率的变异系数（CV = std/mean）超过 0.3，标注「各池表现高度异质，全局指标可能掩盖池间差异」。

**缺陷 P3：Laplace 加一平滑对 AB 分析过度收缩（中等）**

`compute_ab` 使用 `(success+1)/(total+2)` 作为 P(成功|事件) 的估计。这是 Beta(1,1) 先验下的后验均值——等价于假设在观察数据前，事件发生和不发生时各观察到了一次成功和一次失败。

问题：
- 当 n=100 时，+1 的先验信息贡献了约 1% 的估计值——影响可忽略
- 当 n=3 时，+1 的先验信息贡献了约 40% 的估计值——影响极大
- 在成功概率接近 0 或 1 时，Beta(1,1) 先验（均匀先验）将估计拉向 0.5——这与真实情况可能完全相反

更严重的是，Laplace 平滑在处理不同事件类型时施加了**相同的收缩强度**，而忽略事件类型的基础率信息——出现 1 次的 `pity_hit` 事件和出现 1 次的 `skip` 事件得到相同的平滑处理，尽管后者的成功基础率可能截然不同。

**文献建议**：对于离散事件类型的条件概率估计，层次贝叶斯模型（Gelman et al., 2013, *Bayesian Data Analysis*, 第 5 章）允许使用层次先验共享跨事件类型的信息——基础率高的事件类型收缩较少，基础率低的事件类型收缩较多。

**缺陷 P4：compute_ba 的似然比在分母为零时返回 inf（中等，已在原审查中记录为缺陷 17）**

`ratio = P(事件|成功) / P(事件|失败)`。当 `P(事件|失败) = 0` 时返回 `inf`——这在直觉上是合理的（事件仅在成功路径上出现），但 `inf` 值在下游聚合中破坏所有统计量，且 UI 未对该值做特殊处理。

**缺陷 P5：单池 GDR 的 pseudo_compact 构造不完整（中等）**

`compute_pool_gdr_single_pool` 构造一个伪 `compact` 字典传给 `compute_gdr_from_compact`。但该伪字典缺少以下字段，当前 GDR 指标虽然不全部使用，但未来新增的 GDR 指标可能依赖它们：

- `no_draw_resources` — 无抽卡时的资源值
- `strategy_name` — 策略名
- `result_version` — 结果版本
- 各池资源的完整初始值和增益

这是一个脆弱性风险——如果未来有人添加了一个需要这些字段的新 GDR 指标，单池模式会静默失败（返回 None 或错误值），而调试将极为困难。

**缺陷 P6：事件推断的 pool_target_map 为非必填参数——回退逻辑有已知缺陷（已在原审查中记录为缺陷 16）**

`_resolve_skip_ignore` 优先使用 `pool_target_map`（池→可掉落目标卡映射），未提供时回退到基于 `pool_card_counts` 的启发式推断。原审查已指出旧启发式在 0 抽池上错误标记 `skip`/`ignore`。此修复（引入 `pool_target_map`）未被本模块完整推送——`_infer_from_draw_sequence` 和 `_infer_from_aggregate` 的回退路径仍可能触发旧缺陷。

**缺陷 P7：资源池和兑换池在事件分析中作为「特殊类型」处理——破坏了与 GDR 系统的一致性（低等）**

资源池和兑换池被赋予特殊的事件类型（`resource_draw`/`resource_ignore`/`exchange`/`no_exchange`），但它们在 AB/BA 交叉分析中未被纳入成功/失败判定——始终被排除。这意味着：

1. 兑换池消耗的兑换币不计入资源消耗
2. 资源池产出的资源不纳入资源余额
3. AA 分析中可以看到它们的事件，但无法在 BB/AB/BA 中看到它们的成败影响

在当前实现中，这不影响核心分析——因为兑换池和资源池通常不被视为影响策略成功的主要因素。但如果未来分析想要纳入兑换/资源池的经济影响，需要统一处理。

## 六、方法论总结

### 6.1 四个模块的理论水平特征

这四个补充审查模块展现出与主体审查一致的模式：**工程实现基本正确，统计理论存在系统性薄弱环节**。具体而言：

1. **最好/最差情形**：核心方法（VaR/CVaR/条件分布）的选择是合理的，但缺乏不确定性量化。最严重的缺陷是经验分位数在尾部的高度不稳定性——用户看到的 VaR/CVaR 点估计可能伴随 ±20% 的误差，而系统未提供任何精度提示。`BestCaseAnalysis` 缺乏下尾风险度量那样的公理化基础，其「最好情形」的解释应降级为「乐观情形」。

2. **转变分析**：2x2 马尔可夫转移矩阵的概念框架是正确的，但实现停留在「报告点估计」的水平——无置信区间、无假设检验、无模型诊断。这是四个模块中唯一一个有相对成熟统计文献支撑的模块（转移矩阵推断有 60+ 年的方法论积累），因此改进路径最为清晰。

3. **最少资源与最多目标卡**：这是四个模块中理论最薄弱的——三个搜索算法都是「在实践中可能有效」的启发式，但没有一个满足其声称的数学性质（Pareto 最优性、收敛保证、最优停止）。二分搜索 + 蒙特卡洛噪声的组合是最严重的设计缺陷。

4. **每池GDR**：事件分类体系和双模式设计在概念上是最成熟的，但多重比较和 Simpson 悖论的风险是系统性的——这些不是单点 bug，而是分析框架层面的遗漏。

### 6.2 跨模块共性问题

四个模块共享以下三个系统性薄弱点：

| 问题 | 涉及模块 | 标准实践 |
|------|---------|---------|
| **点估计无不确定性量化** | 全部四个 | Bootstrap 置信区间 / 渐近标准误 / 贝叶斯后验 |
| **多组比较无多重性校正** | 转变、每池GDR | Bonferroni / Holm / Benjamini-Hochberg / Westfall-Young |
| **算法声称超出实际数学保证** | 最差/最好（BestCaseAnalysis 的公理化地位）、资源/目标（Pareto 最优性） | 降级声称或提升算法至所需保证 |

### 6.3 对后续工作的建议

1. **优先修复第一级缺陷**：转变分析的置信区间、二分搜索的噪声处理、Pareto 声称的降级或算法升级
2. **为所有比率/概率估计补充 bootstrap 置信区间**——转变矩阵、每池成功率、条件概率 → 不确定性可视化是建立用户信任的关键
3. **在「Pareto 前沿」UI 中降级声称**为「资源-目标权衡曲线（贪心近似）」并添加支配性过滤
4. **为每池分析增加异质性诊断**（CV 系数、Simpson 悖论警告）
5. **重新设计二分搜索算法**——采用序贯概率比检验（SPRT）替代固定 N 采样，在保证决策可靠性的同时减少 40-60% 的总模拟次数（详见缺陷 R1 的推荐改进方案）
6. **在 TransitionMatrix 中附加连续度量**——「之前成功组的平均资源消耗」和「之前失败组的平均资源消耗」，在不膨胀状态空间的前提下量化转移的强度信息（详见 §2.4）
