## 二、PityEngine 详析

### 构造参数

```python
PityEngine(
    pool_specs: Dict[str, PoolPitySpec],   # pool_id → 保底配置
    pity_defs: Dict[str, PityDefParsed],    # pity_name → 保底定义
    behaviors: Dict[str, PityBehavior],     # pity_name → 行为实例
)
```

### 核心方法

| 方法 | 是否修改 PityState | 返回 | 说明 |
|------|-------------------|------|------|
| `get_probabilities(pool_id, state, base)` | **否** | `Dict[str, float]` | 纯查询——按管道顺序叠加所有保底，返回最终概率分布 |
| `before_draw(pool_id, state, base)` | **是**（`increment` 所有适用保底） | `Dict[str, float]` | 先递增计数器，再调用 `get_probabilities` |
| `after_draw(pool_id, state, reward_id)` | **是**（按重置条件 `reset`） | `None` | 检查 `reward_id` 是否为 SSR/限定，按条件重置 |
| `get_spec(pool_id)` | 否 | `Optional[PoolPitySpec]` | 查询池的保底配置 |

### get_probabilities 管道流程

```
base_probabilities
  → for pname in pool_spec.pity_names (按 pity.txt 出现顺序):
      behavior = behaviors[pname]
      counter = state.get(pname)
      extra = {'resolved_targets': pool_spec.resolved_targets[pname]}  # if exists
      base = behavior.apply(counter, base, extra)
  → return base  # 最终管道输出
```

### after_draw 重置逻辑

```
for pname in pool_spec.pity_names:
    pdef = pity_defs[pname]
    if pdef.reset_condition == 'any_ssr' and reward_id in ssr_ids:
        state.reset(pname)
    elif pdef.reset_condition == 'featured_ssr' and reward_id in featured_ids:
        state.reset(pname)
    elif pdef.reset_condition == 'never':
        pass  # 不重置
```

## 三、PoolPitySpec 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `pity_names` | `List[str]` | 该池适用的保底名称列表（管道顺序） |
| `ssr_ids` | `Set[str]` | 该池的 SSR 卡片 ID 集合（用于 `any_ssr` 重置判定） |
| `featured_ids` | `Set[str]` | 该池的限定卡片 ID 集合（用于 `featured_ssr` 重置判定） |
| `resolved_targets` | `Dict[str, Dict[str, float]]` | `{pity_name: {card_id: weight}}`——每个保底解析后的目标分布 |

## 四、PityState 详析

```python
@dataclass
class PityState:
    counters: Dict[str, int] = field(default_factory=dict)

    def get(self, name: str) -> int:
        return self.counters.get(name, 0)

    def increment(self, name: str):
        self.counters[name] = self.counters.get(name, 0) + 1

    def reset(self, name: str):
        self.counters[name] = 0
```

每次模拟开始时创建新的 `PityState` 实例，确保模拟之间无保底状态泄露。

## 五、配置文件格式

### pity.txt 解析格式

```
pity: 保底名 | type=soft | start_at=50 | end_at=100 | func=linear | target=card_a:1.0,card_b:0.5 | reset=any_ssr | pools=pool_1
pity: 保底名 | type=hard | threshold=90 | target=card_c:1.0 | reset=featured_ssr | pools=pool_1
```

### 保底参数说明

| 参数 | soft | hard | 说明 |
|------|------|------|------|
| `type` | `soft` | `hard` | 保底类型 |
| `start_at` | 必需 | — | 软保底起始计数器值 |
| `end_at` | 必需 | — | 软保底结束计数器值（达到时概率 100%） |
| `threshold` | — | 必需 | 硬保底触发阈值 |
| `func` | 可选(`linear`) | — | 递增函数类型：`linear`/`exp`/`step` |
| `target` | 可选 | 可选 | 目标分布：`card_id:weight,...` |
| `reset` | 必需 | 必需 | 重置条件：`any_ssr`/`featured_ssr`/`never` |
| `pools` | 必需 | 必需 | 适用池列表（逗号分隔或通配符） |
