## 二、SharedResultCollector 详析

### 构造与注册

```python
collector = SharedResultCollector()
collector.add_extractor('success', make_success_extractor(...))
collector.add_extractor('aggregate', extract_aggregate)
collector.add_extractor('process', lambda c: extract_process(c, ...))
```

`add_extractor(name, extract_func)`：
- `name`：提取结果在 `_extractors` 字典中的键
- `extract_func`：签名 `(compact: Dict) -> Any` 的可调用对象
- 必须在首次 `on_result` 调用前完成所有注册

### 数据流

```python
def on_result(self, compact: Dict[str, Any]):
    if compact is None:
        return
    for name, (extract_func, acc) in self._extractors.items():
        acc.append(extract_func(compact))
    self.n_results += 1
```

每次 `on_result` 调用：
1. 遍历所有注册的 extractor
2. 每个 extractor 处理同一个 `compact`，结果追加到各自的 `acc` 列表
3. `compact` 随后由调用者丢弃（或被 Python GC 回收）
4. `n_results` 递增

### 读取结果

```python
success_counts = collector.get_extracted('success')  # → list of (success, total) tuples
aggregates = collector.get_extracted('aggregate')     # → list of dict
```

### 重置

```python
collector.reset()  # 清空所有提取结果，n_results = 0
```

## 四、extract_process 详析

```python
def extract_process(compact, target_ids, target_specs, gdr_key,
                    gdr_threshold, initial_resources=None, ...):
    checker = SuccessChecker(target_specs, gdr_key, gdr_threshold, ...)
    val = checker.compute_gdr(compact)
    pool_events = infer_events(compact, target_ids)

    return {
        'pool_events': {pid: ev.event_type for pid, ev in pool_events.items()},
        'pool_event_details': {
            pid: {
                'event_type': ev.event_type,
                'pity_name': ev.pity_name,
                'draws': ev.draws,
                'counter_max': ev.counter_max,
            }
            for pid, ev in pool_events.items()
        },
        'success': (val <= threshold) if lower_is_better else (val >= threshold),
        'gdr_value': val,
    }
```

组合了 GDR 计算（通过 `SuccessChecker`）和过程事件推断（通过 `infer_events`），单次提取满足过程分析面板的全部需求。

## 五、StreamingSuccessCounter 详析

```python
class StreamingSuccessCounter(StreamingAnalyzer):
    def __init__(self, target_specs, gdr_key, gdr_threshold, ...):
        self._checker = SuccessChecker(...)
        self.total = 0
        self.success = 0

    def on_result(self, compact):
        self.total += 1
        if self._checker.is_success(compact):
            self.success += 1

    def get_probability(self) -> float:
        return self.success / self.total if self.total > 0 else 0.0
```

O(1) 内存——只存储两个整数。`get_probability()` 可在模拟运行期间随时调用，获取当前的成功率估计。

## 六、WorkerLocalExtractor 详析

在 Worker 子进程内执行，每次 `process(compact)` 返回：

```python
{
    'aggregate': extract_aggregate(compact),     # 聚合统计
    'sequence': {                                # 序列数据（仅前 max_keep 个结果保留）
        'draw_card_ids': [...],
        'draw_pool_ids': [...],
        ...
    } if keep else None,
    'heatmap': (ach_bin, res_bin),               # 热力图二元组索引
}
```

主进程收到提取包后，负责合并（如累加直方图 bin 计数、拼接序列数据）。与 `SharedResultCollector` 的分工：
- `WorkerLocalExtractor`：在 Worker 内执行，每次返回轻量包
- `SharedResultCollector`：在主进程执行，累积所有提取结果
