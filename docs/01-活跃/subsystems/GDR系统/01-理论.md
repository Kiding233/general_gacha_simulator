## 一、核心假设

### 1.1 资源单态假设

所有基于资源的 GDR（`resource_remaining`、`resource_consumed`、`resource_efficiency`、`resource_per_card`、`draw_conversion_efficiency`）当前仅跟踪 `'draw_resource'` 键。系统隐含假设**只存在一种有分析价值的资源类型**（抽卡资源），兑换货币等其他资源类型不在 GDR 计算范围内。

**影响**：多货币场景（免费石/付费石/门票）下无法区分，一个 GDR 值混入了不可比的经济维度。

**缓解计划**：P22（多资源类型分布实施方案）——通过 GDR key 参数化（`gdr_key:resource_id` 格式）展开资源类 GDR，使每种资源类型对应独立指标。

### 1.2 可加性假设（weighted_satisfaction）

`weighted_satisfaction` 指标假设跨卡片的可加拟线性效用函数：

```
U = Σ (desire_i × got_i − miss_cost_i × (target_i − got_i)_+)
```

**内核假设**：
1. **可加独立性**：卡片之间无交互项。卡 A 和卡 B 的效用被假定为独立，现实中「卡 A + 卡 B 组队效果翻倍」的协同效应无法建模。
2. **线性效用**：每张卡的边际效用恒定，不受已获得数量影响（无边际效用递减）。
3. **desire 和 miss_cost 外生独立**：在期望效用理论（von Neumann-Morgenstern, 1944）中，desire 和 miss_cost 通过参考点关联。前景理论（Kahneman & Tversky, 1979）表明损失厌恶系数 λ > 1——未获得的痛苦大于获得的快乐。当前设计将它们视为两个独立参数，无参考点校准。

### 1.3 各指标独立性

17 个 GDR 指标被设计为捕捉抽卡结果的不同正交维度，但存在实质性重叠：

- `target_achievement`（加权数量达成率）和 `target_collection`（种类收集率）在「每个目标卡需求数量=1」时高度相关
- `resource_consumed` 和 `resource_remaining` 构成反向关系（剩余 = 初始 + 收入 - 消耗）
- `resource_efficiency` 和 `resource_per_card` 构成倒数关系（效率 = 获得数/消耗，每卡 = 消耗/获得数）

**未做多重共线性处理**：各指标独立计算，下游分析不检测指标间共线性。在条件分布分析中可能产生冗余维度。

### 1.4 阈值判定的确定性

`SuccessChecker` 的 `is_success()` 将浮点数 GDR 值按 `>= threshold`（或 `<=` 若 `lower_is_better`）做硬判定，产生二元 0/1 结果。无模糊判定或概率化输出。这意味着在阈值边界处（如 `target_achievement = 0.999` vs `threshold = 1.0`），微小的数值差异会导致截然不同的成功/失败分类——而这两个结果的实质差异可能微不足道。

## 二、已知局限性（GDR 相关缺陷）

以下缺陷来自 `全模块理论严谨性系统性审查.md` §2.2。

### 缺陷 3：resource_per_card 返回 inf 破坏下游聚合（严重）

```python
# core/gdr.py _gdr_resource_per_card()
if obtained <= 0:
    return float('inf')  # ← 问题
```

任何包含此值的 `mean()` 或 `sum()` 都得到 `inf`，静默破坏所有聚合统计量。例如在 `analysis_panel.py` 中提取 GDR 分布时，`inf` 传入直方图绑图会导致静默丢弃或报错。

**状态**：未修复。P21（GDR 分箱与绘图策略）H4 项已识别此问题并规划了 inf 分离处理方案。

### 缺陷 4：weighted_satisfaction 缺乏公理化效用理论基础（中等）

详见 §1.2 的可加性假设讨论。核心问题：
- 可加性假设过强，无交互项
- desire 和 miss_cost 的独立性违反前景理论的参考点依赖
- 未归一化——所有权重默认为 1.0 时退化为 `Σ(2×got − target)`，跨不同 `target_specs` 配置不可比较

**状态**：P29（GDR系统理论文档补充计划）已制定改进方案——公理化文档 + 归一化选项。

**文献**：von Neumann & Morgenstern (1944), *Theory of Games and Economic Behavior*; Kahneman & Tversky (1979), *Econometrica*; Keeney & Raiffa (1976/1993), *Decisions with Multiple Objectives*

### 缺陷 5：total_card_value 对非目标卡隐式赋值 1.0（中等）

```python
# core/gdr.py _gdr_total_card_value()
total_value += cnt * card_value_weights.get(card_id, 1.0)  # ← 默认 1.0
```

对所有未配置 `card_value_weights` 的卡（包括所有非目标卡）隐式赋予权重 1.0。普通 SSR 被赋予与「任意目标卡的一个副本」相同的价值。

**状态**：P29（GDR系统理论文档补充计划）建议将默认值从 1.0 改为 0.0（破坏性变更）。

### 缺陷 6：资源单态假设（中等）

所有资源类 GDR 硬编码 `'draw_resource'` 键，不支持 `exchange_currency` 等其他资源类型。

**状态**：P22（多资源类型分布实施方案）已制定完整的 GDR key 参数化方案（`gdr_key:resource_id` 格式展开）。

### 缺陷 7：DropRateBetweenT1T2 忽略 t 参数（低等）

位于 `core/generalized_drop_rate.py`。`compute(t, history)` 方法忽略传入的 `t`，始终计算固定窗口 `[t1, t2)` 内的出率。

**状态**：未修复。该文件主要为历史兼容保留，当前所有面板均使用紧凑路径。

### 缺陷 8：weapon_character_ratio 的 weapon_character_map 无配置入口（已知）

`GDRContext.weapon_character_map` 在配置系统中无填写入口，始终为空 `dict`。`_gdr_weapon_character_ratio()` 在 `weapon_character_map` 为空时直接返回 0.0，导致该指标不可用。

**状态**：CLAUDE.md 已知问题 #1。需在配置系统新增角色-武器对应表。

## 三、指标分类

### 3.1 按值域分类

| 类别 | 值域 | 包含指标 | 分箱策略 |
|------|------|---------|---------|
| A 类：有限格点 | `{k/T : k=0..T}` | `target_achievement`、`target_collection`、`ssr_collection` | 条形图（sorted unique values） |
| B₀ 类：二元 | `{0, 1}` | `all_targets` | 成功率数值卡片 + 饼图 |
| B₁ 类：格点步长 | `{R₀ + k·Δ : k∈ℤ}` | `resource_remaining`、`resource_consumed` | 步长检测 + 对齐分箱 |
| C 类：近连续 | ℝ⁺ | `resource_efficiency`、`per_pool_draw_rate`、`weapon_character_ratio`、`draw_conversion_efficiency` | FD + KDE 叠加 |
| W 类：加权 | ℝ（无界） | `weighted_satisfaction`、`total_card_value` | FD 分箱 |
| 特殊 | ℝ⁺ ∪ {inf} | `resource_per_card` | 先分离 inf，再 FD |

### 3.2 按 lower_is_better 分类

| lower_is_better | 指标 |
|-----------------|------|
| **是**（越低越好） | `resource_consumed`、`resource_per_card` |
| 否（越高越好） | 其余 15 个指标 |

### 3.3 按权重需求分类

| needs_weights | 指标 |
|---------------|------|
| `'desire+miss_cost'` | `weighted_satisfaction` |
| `'card_value'` | `total_card_value` |
| 无 | 其余 15 个指标（不依赖用户权重配置） |
