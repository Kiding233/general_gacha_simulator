### GDRDefinition 字段说明

```python
class GDRDefinition(NamedTuple):
    key: str                                  # 唯一标识符（英文 key）
    display_name: str                         # GUI 显示名（中文）
    default_threshold: float                  # 默认成功阈值
    compute_from_compact: Callable[..., float]  # 紧凑路径 compute 函数（必填）
    compute_from_history: Optional[Callable] = None   # 历史路径 compute 函数（可选）
    needs_ssr_ids: bool = False               # 是否需要 SSR ID 集合
    needs_weapon_map: bool = False            # 是否需要武器-角色映射表
    needs_weights: str = ''                   # 权重需求类型：'desire+miss_cost' | 'card_value' | ''
    category: str = 'basic'                   # 分类标签：'basic' | 'resource' | 'weighted'
    lower_is_better: bool = False             # True 表示值越小越好
```

## 三、两种计算路径

### 3.1 紧凑路径（O(1)，主流）

**入口**：`compute_gdr_from_compact(compact, target_specs, gdr_key, ...)`

**流程**：
1. 从 `UNIFIED_GDR_REGISTRY` 按 `gdr_key` 查找 `GDRDefinition`
2. 调用 `defn.compute_from_compact(compact, target_specs=..., **extra_kwargs)`
3. 底层 `_gdr_xxx` 函数从 `CompactResult` 字典中直接读取聚合字段：
   - `card_counts`：`Dict[card_id, count]`
   - `total_draws`：总抽卡次数
   - `pity_triggers`：保底触发次数
   - `total_consumed`：`Dict[resource_id, amount]`
   - `final_resources`：`Dict[resource_id, amount]`
   - `pool_draw_counts`：`Dict[pool_id, count]`
   - `pool_card_counts`：`Dict[pool_id, Dict[card_id, count]]`
   - `pool_types`：`Dict[pool_id, type_name]`

**特点**：
- 无需遍历逐抽历史，时间复杂度 O(1)（字典查找）
- 所有分析面板（`analysis_panel.py`、`process_analysis_panel.py` 等）均使用此路径
- `SuccessChecker.compute_gdr()` 和 `compute_success_probability()` 内部调用此路径

### 3.2 历史路径（O(T)，兼容）

**入口**：`defn.compute_from_history(history, ctx: GDRContext)`

**流程**：
1. 从 `UNIFIED_GDR_REGISTRY` 获取 `compute_from_history` 可调用对象
2. 传入 `history: List[InfoVector]` 和 `GDRContext`
3. 逐抽遍历 `InfoVector`，累加计数/资源消耗/保底触发等

**特点**：
- 时间复杂度 O(T)，T 为总行动数
- 部分指标（`weighted_satisfaction`、`total_card_value`）无 `compute_from_history` 实现（`None`），仅支持紧凑路径
- 向后兼容旧代码（如 `compute_all_gdr()`、`compute_weighted_gdr()`）
- 通过 `_build_legacy_registries()` 生成 `GDR_REGISTRY`（仅含历史路径可用指标）和 `COMPACT_GDR_REGISTRY`

### 3.3 累积快照适配器

**函数**：`compute_gdr_from_cumulative(cum_snapshot, ...)`

将逐池/逐时间的累积快照（`cumulative_card_counts`、`cumulative_draws`、`cumulative_consumed` 等）组装为伪 `CompactResult` 字典，然后走紧凑路径。用于逐池 GDR 分析和过程分析的 GDR 时间序列。

## 四、SuccessChecker

### 构造参数

```python
SuccessChecker(
    target_specs,        # Dict[str, int] 目标卡规格
    gdr_key,             # str GDR 指标 key
    gdr_threshold=None,  # float 成功阈值（None 时使用 default_threshold）
    desire_weights,      # 加权满意度 desire 权重
    miss_cost_weights,   # 加权满意度 miss_cost 权重
    card_value_weights,  # 总出卡价值权重
    ssr_ids,             # SSR ID 集合（用于 ssr_collection）
    weapon_character_map # 武器-角色映射表
)
```

### 核心方法

| 方法 | 返回 | 说明 |
|------|------|------|
| `compute_gdr(compact)` | float | 计算单个模拟的 GDR 值 |
| `is_success(compact)` | bool | 判定单个模拟是否成功（`>= threshold` 或 `<=` 若 `lower_is_better`） |
| `check_batch(results)` | tuple[int, int, float] | 批量判定，返回 `(success_count, total, probability)` |
| `from_registry(gdr_key, target_specs, ...)` | SuccessChecker | 工厂类方法 |

### 使用位置

所有 10 个分析面板的 Worker 线程均通过 `SuccessChecker` 判定每次模拟的成功/失败状态，消除了各面板重复实现判定逻辑的问题。

## 五、扩展机制

### 添加新 GDR 指标

1. 在 `core/gdr.py` 实现 `_gdr_xxx(compact, target_specs, **kwargs)` 紧凑路径函数
2. 可选：实现 `xxx(history, ctx: GDRContext)` 历史路径函数
3. 在 `UNIFIED_GDR_REGISTRY` 添加 `GDRDefinition` 条目
4. 或运行时调用 `register_gdr(GDRDefinition(...))`
5. 所有面板的 GDR 下拉列表通过 `populate_gdr_combo()` 自动更新

### 注册表展开（P22 计划）

`expand_resource_gdrs(resource_defs)` 函数（P22 阶段 1 计划实现）将为资源类 GDR 按 `resource_defs` 中的每个资源类型展开独立的 `{gdr_key}:{resource_id}` 条目（如 `resource_remaining:exchange_currency`），使每种资源类型对应独立的虚拟 GDR 指标。

## 七、统一分箱模块（P21 ✅ 2026-06-04）

### 文件：`gacha_simulator/core/gdr_binning.py`（~305行）

**职责**：所有 GDR 绘图分箱策略的单一入口，纯数据驱动——不做按 GDR 类型预设分箱。

### 层次结构

| 层 | 函数 | 触发条件 | 输出 |
|----|------|---------|------|
| L1 有限格点 | `_bins_finite_grid()` | 唯一值 ≤ 20 | `bar_mode=True`，go.Bar |
| L2 步长检测 | `_bins_stepped()` | `cost_per_draw` 非空 或 `use_draw_units=True` | 对齐分箱 edges |
| L3 连续分布 | `_bins_continuous()` | 回退 | FD 规则 + IQR 保护 |
| 特殊 | `_bins_with_inf()` | `resource_per_card` 含 inf | 分离有限值 + inf 标注 |

### 公共 API

- `compute_bins(gdr_key, samples, *, cost_per_draw, use_draw_units, fixed_edges)` → `BinningResult`
- `detect_step_size(values)` → `Optional[float]`
- `compute_aligned_bins(values, step)` → `np.ndarray`

### 接入点

| 面板/模块 | 调用数 | 说明 |
|-----------|--------|------|
| `analysis_panel.py` | 8 | GDR分布、最差/最好、条件分布、山脊线 |
| `comparison_analysis_panel.py` | 1 | L1 描述统计 PMF/ECDF |
| `process_analysis_panel.py` | 1 | 条件 GDR 分布 |
| `vulnerability.py` | 2 | 全局分箱 + 每池回退 |

### 渲染层适配

`plotly_charts.py` `_build_histogram()` 通过 `bar_mode` layout_hint 分流：
- `bar_mode=True` → `_build_bar_from_grid()`（go.Bar 离散柱状图）
- `bar_mode=False` → go.Histogram（共享 xbins 的对齐直方图）
