## 三、各条件判定逻辑详解

### AllPoolsEndCondition

```python
def check(self, state, history, stats):
    return state.real_time >= self.end_time
```
最常用的条件——在所有池的排期结束后自然停止。`end_time` 由排期管理器计算（所有池的最大结束时间）。

### FixedActionCountCondition

```python
def check(self, state, history, stats):
    if stats:
        return stats.total_actions >= self.max_actions
    return len(history) >= self.max_actions
```
双路径判定：有 `stats` 时 O(1)，无 `stats` 时 O(1)（`len(history)` 也是 O(1)）。

### ResourceThresholdCondition

```python
def check(self, state, history, stats):
    current = state.resources.get(self.resource, 0)
    if self.operator == '<=': return current <= self.threshold
    elif self.operator == '>=': return current >= self.threshold
    elif self.operator == '==': return current == self.threshold
```
三种比较运算符。注意 `==` 在浮点资源值下可能因精度问题失效——资源值通常为整数（`int`），当前无此风险。

### ConsecutivePoolTargetCondition（最复杂）

```python
def check(self, state, history, stats):
    if state.resources.get(self.resource_name, 0) <= 0:  # 资源耗尽
        return True
    if self.end_time is not None and state.real_time >= self.end_time:  # 超时
        return True
    if stats:
        for pool_id, _start, end in self.pool_schedules:
            if state.real_time < end: break  # 跳过未结束的池
            target_card = self.pool_targets.get(pool_id)
            if target_card and stats.card_counts.get(target_card, 0) < 1:  # 已结束的池目标未达成
                return True
    return False
```
遍历已结束的池，检查每个池的目标卡是否获得。若任何已结束池的目标未达成且资源耗尽或超时，停止模拟。用于「连续池必须全部达成目标」的自动化场景。

## 四、工厂函数

### create_stop_condition()

```python
def create_stop_condition(name: str, params: Optional[Dict[str, Any]] = None) -> StopCondition:
    entry = STOP_CONDITION_REGISTRY.get(name)
    if entry is None:
        raise ValueError(f"Unknown stop condition: {name!r}")
    cls = entry['class']
    resolved = dict(params) if params else {}
    param_defs = entry.get('params', {})
    for pname, pdef in param_defs.items():
        if pname not in resolved:
            resolved[pname] = pdef['default']
    return cls(**resolved)
```
参数自动填充默认值。不进行类型转换（由 `__init__` 处理）。

## 五、扩展机制

### 添加新停止条件

1. 实现条件类（继承 `StopCondition`，实现 `check()` 和 `description()`）
2. 在 `STOP_CONDITION_REGISTRY` 添加条目（含 `display_name`、`class`、`params`）
3. GUI 停止条件下拉列表自动包含新条件

### 约束

- `check()` 必须是无副作用的纯函数——不修改 `state`、`history`、`stats`
- `description()` 返回人类可读的中文描述，格式为「[条件]」
- 条件应为单调的（一旦为 `True` 不会回退为 `False`）
