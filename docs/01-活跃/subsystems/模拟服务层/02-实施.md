## 两种模拟模式详解

### 紧凑模式（CompactCollector，主路径）

```python
service = GachaService(pools, strategy, stop_cond, ...)
compact = service.run_simulation_compact(state)
# compact: CompactResult（dict 或 dataclass，含以下字段）
#   - draw_card_ids: List[str]      逐抽获得卡牌 ID
#   - draw_pool_ids: List[str]      逐抽所在池 ID
#   - draw_times: List[float]       逐抽时间点
#   - draw_pity: List[int]          逐抽保底计数器值
#   - draw_pity_names: List[str]    逐抽触发的保底名称
#   - draw_pity_counter_max: List[int] 逐抽保底最大计数器
#   - card_counts: dict             各卡获得次数
#   - total_draws: int              总抽数
#   - total_consumed: dict          各资源总消耗
#   - pool_draw_counts: dict        各池抽数
#   - pool_end_resources: dict      各池结束时资源快照
#   - pity_triggers: dict           各保底触发次数
#   - final_resources: dict         最终资源
#   - final_time: float             最终时间
```

### 完整模式（InfoVectorCollector，兼容旧代码）

```python
service = GachaService(pools, strategy, stop_cond, ...)
info_list = service.run_simulation(state, collector=InfoVectorCollector())
# info_list: List[InfoVector]（每抽一个条目）
```

## BatchResult 类

```python
class BatchResult:
    """向后兼容 list 接口的结果容器。"""
    
    def __init__(self, results: List[Optional[Dict]], extraction: Optional[dict] = None):
        self._results = results
        self.extraction = extraction  # 各 worker 的聚合提取数据
    
    def __getitem__(self, idx): return self._results[idx]
    def __len__(self): return len(self._results)
    def __iter__(self): return iter(self._results)
    def __bool__(self): return bool(self._results)
```

## SimulationEnvBuilder.from_config_store() 流程

```
ConfigStore
  → load pools (Pool.from_config)
  → load pity (PityEngine.from_config)
  → load schedule (ScheduleManager)
  → load resource_gain (ResourceGainSchedule)
  → load card_defs
  → load initial_resources
  → load targets + SSR IDs + all_drawable_ids
  → build GDRContext (target_specs, resource_map, weights)
  → create Strategy object (via STRATEGY_REGISTRY)
  → create StopCondition object (via STOP_CONDITION_REGISTRY)
  → SimulationEnv(pools, schedule_mgr, end_time, pity_engine, ...)
```
