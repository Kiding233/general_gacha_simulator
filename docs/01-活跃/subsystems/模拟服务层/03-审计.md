## 待审计项

| 项目 | 优先级 | 说明 |
|------|--------|------|
| `_wk_run_single` 异常处理 | 高 | 子进程异常是静默吞掉（返回 None）还是传播到主进程？当前行为是否正确？ |
| `pool.terminate()` 资源泄漏 | 中 | 自适应停止时使用 `terminate()` 强制终止池，是否可能泄漏共享内存/文件句柄？ |
| 单进程路径的全局变量重置 | 中 | 单次调用后 `_wk_env` 等全局变量应重置为 None，否则下次调用可能使用残留引用 |
| `CompactResult` 字段可选性 | 低 | 部分调用方（如 CLI）可能不填充某些字段（如 `draw_pity_names`），下游是否做了空值防护？ |
| `merge_extraction_packets` 的线程安全性 | 低 | 该方法在主进程中调用，但若未来改为多线程需加锁 |
