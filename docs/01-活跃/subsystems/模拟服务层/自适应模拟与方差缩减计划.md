## 二、理论基础（保留自设计文档，指导实现决策）

### 2.1 蒙特卡洛精度定律

```
SE = σ / √N
RSE = SE / |θ̂| = σ / (|θ̂| × √N)
要达到目标 RSE < ε，所需样本量：N ≥ (σ / (ε × |θ̂|))²
```

### 2.2 不同估计量的精度特征

**概率估计（伯努利）**：`SE = √(p(1-p)/N)`，`RSE = √((1-p)/(N×p))`

| p | N=1,000 RSE | N=10,000 RSE | N=100,000 RSE |
|---|------------|-------------|--------------|
| 0.5 | 3.2% | 1.0% | 0.32% |
| 0.9 | 1.1% | 0.33% | 0.11% |
| 0.99 | 3.2% | 1.0% | 0.32% |

**分位数估计**：`SE(x_p) ≈ √(p(1-p)) / (f(x_p) × √N)`。5% VaR 的标准误是中位数的 ~28 倍。

**稀有事件**：概率 q 的事件，要观察到 ≥k 次需 `N ≥ k/q`。

### 2.3 各分析类型的推荐模拟量（合并自设计文档）

| 分析类型 | 估计目标 | 推荐 N | 精度依据 |
|---------|---------|--------|---------|
| 成功率判定 | 概率 p | 1,000-5,000 | RSE(p) < 5% |
| GDR 分布（均值/直方图） | 连续分布 | 5,000-10,000 | SE(均值) < 0.5% |
| VaR/CVaR（5% 尾部） | 尾部分位数 | 10,000-50,000 | SE(VaR) 可接受 |
| 脆弱性分析 | 条件概率函数 | 10,000-50,000 | 每个直方图 bin ≥30 样本 |
| 过程分析（常见模式 q>1%） | 模式概率 | 10,000-50,000 | 每种模式 ≥100 次观察 |
| 过程分析（罕见模式 q~0.1%） | 罕见模式概率 | 100,000-500,000 | 每种模式 ≥100 次观察 |
| 最差影响分析 | 条件分布 | 5,000-10,000 | 与 GDR 分布相同 |

### 2.4 方差缩减技术评估（5 种，实现 2 种）

| 技术 | 实现复杂度 | 适用场景 | 收益 | 决策 |
|------|----------|---------|------|------|
| 重要抽样（IS） | 高 | 极低成功率 <1% | 极高（10-1000x） | **暂不实现**，未来按需 |
| **对偶变量法** | **中-高**（v3 修正） | GDR 均值估计 | 中（1.2-2x） | ⚠️ **暂缓**（v3 重评估，见 §四） |

- [ ] **1.10: 实现 AdaptiveSimController 类**

```python
"""自适应模拟控制器——实时 RSE 监控，达到目标精度自动停止"""
from typing import List, Dict, Optional, Callable


class AdaptiveSimController:
    def __init__(self, target_rse=0.05, min_samples=1000,
                 max_samples=100000, check_interval=500,
                 consecutive_passes=2,  # v3 新增：缓解停止偏误
                 gdr_keys=None, target_specs=None):
        from gacha_simulator.core.gdr import compute_gdr_from_compact
        self._compute_gdr = compute_gdr_from_compact
        self._target_rse = target_rse
        self._min_samples = min_samples
        self._max_samples = max_samples
        self._check_interval = check_interval
        self._consecutive_passes = consecutive_passes
        self._gdr_keys = gdr_keys or ['target_achievement']
        self._target_specs = target_specs or {}
        self._values = {k: [] for k in self._gdr_keys}
        self._converged = False
        self._n_done = 0
        self._n_consecutive = 0  # v3 新增
        self._worst_rse = float('inf')
        self._worst_key = None

    def on_result(self, compact):
        for key in self._gdr_keys:
            val = self._compute_gdr(compact, self._target_specs, key)
            self._values[key].append(val)
        self._n_done += 1
        if self._n_done >= self._min_samples and \
           self._n_done % self._check_interval == 0:
            self._check_convergence()

    def _check_convergence(self):
        worst_rse = 0.0
        worst_key = None
        for key, values in self._values.items():
            n = len(values)
            if n < 100:
                continue
            mean = sum(values) / n
            if abs(mean) < 1e-10:
                continue
            variance = sum((v - mean) ** 2 for v in values) / (n - 1)
            se = variance ** 0.5 / n ** 0.5
            rse = se / abs(mean)
            if rse > worst_rse:
                worst_rse = rse
                worst_key = key
        self._worst_rse = worst_rse
        self._worst_key = worst_key
        if worst_rse < self._target_rse:
            self._n_consecutive += 1
            if self._n_consecutive >= self._consecutive_passes:
                self._converged = True
        else:
            self._n_consecutive = 0

    def should_stop(self):
        return self._converged or self._n_done >= self._max_samples

    def get_status(self):
        return {
            'n_done': self._n_done,
            'converged': self._converged,
            'worst_rse': self._worst_rse,
            'worst_key': self._worst_key,
            'target_rse': self._target_rse,
            'consecutive_passes': self._n_consecutive,  # v3 新增
        }
```

## 四、Task 2：对偶变量法（v3 重写）

> **⚠️ v3 重大修正：v2 的种子互补方案（`seed_b = MAX_UINT32 - seed_a`）理论上不成立，本节已完全重写。**

### 4.1 v2 方案的问题

v2 方案在 `batch_simulator` 层面对种子取补：

```python
# v2 错误方案——不要使用
seed_b = MAX_UINT32 - seed_a
```

**为什么不成立**：两个独立随机种子产生两个**独立**的随机流。独立随机变量的协方差为零——`Cov(h(X), h(X')) = 0`——对方差缩减的贡献为零。对偶变量法要求 `(U, 1-U)` 在**同一次抽卡**内配对，而非在两次独立模拟间配对种子。

### 4.2 正确方案

对偶变量法的正确实现需要在**单次模拟内部**对每次抽卡的均匀随机数做配对：

1. **主路径**：正常抽卡，使用 `u ~ U(0,1)` 决定抽卡结果
2. **对偶路径**：使用 `1-u` 决定抽卡结果，其余状态（保底计数器、资源、已获得卡片）与主路径**共享初始状态但独立演化**
3. **最终估计**：`θ̂_AV = (θ̂_main + θ̂_anti) / 2`

这要求改造 RNG 管线：

```
Pool.draw() → 需支持外部注入均匀随机数（而非内部调用 rng.random()）
GachaService.run_simulation_compact() → 需在同一初始状态下运行双路径
batch_simulator → 每"对"模拟产出一个配对估计值
```

### 4.3 实现复杂度重新评估

| 维度 | v2 评估 | v3 评估 |
|------|---------|---------|
| 实现复杂度 | 低（~50行） | **中-高**（需改造 RNG 管线，涉及 Pool / GachaService / 策略层） |
| 涉及文件 | batch_simulator + gacha_panel | + `Pool.draw()` + `GachaService` + 策略层（`StrategyContext` 需感知对偶模式） |
| 风险 | 低 | **中**（RNG 管线侵入性改动，需确保主/对偶路径状态隔离） |
| 预计工期 | 半天 | **3-5 天** |
| 收益 | 1.2-2x | 1.2-2x（不变） |

### 4.4 决策：暂缓实施

**建议暂缓 Task 2**，理由：

1. **性价比**：Task 1（自适应停止）已解决「需要多少模拟」的问题——用户不再需要手动猜测 N，系统自动跑到精度达标。Task 3（EVT）解决精度最差的尾部分位数估计。对偶变量法的 1.2-2x 收益在自适应停止后边际价值降低。

```python
def fit_gpd_tail(data, threshold_method='quantile_90'):
    """对数据尾部拟合 GPD（委托 scipy.stats.genpareto）。
    
    threshold_method:
      - 'quantile_90': 90% 分位数作为阈值（默认）
      - 'quantile_95': 95% 分位数
      - 'stability': 参数稳定性检测自动选择
    
    返回 (ξ, β, threshold)。
    超阈值样本 < 10 时回退：(0.0, std(data), threshold)。
    """
    import numpy as np
    from scipy.stats import genpareto
    
    x = np.array(data, dtype=np.float64)
    
    if threshold_method == 'quantile_90':
        threshold = np.percentile(x, 90)
    elif threshold_method == 'quantile_95':
        threshold = np.percentile(x, 95)
    elif threshold_method == 'stability':
        threshold = gpd_threshold_stability(x)
    else:
        raise ValueError(f"未知阈值方法: {threshold_method}")
    
    exceedances = x[x > threshold] - threshold
    if len(exceedances) < 10:
        return 0.0, float(np.std(x)), threshold
    
    shape, loc, scale = genpareto.fit(exceedances, floc=0)
    return float(shape), float(scale), threshold
```

```python
def gpd_threshold_stability(data, n_candidates=10):
    """参数稳定性检测法选择阈值（Coles 2001, §4.3.4）。
    
    在 85%-95% 分位数范围取 n_candidates 个候选阈值，
    分别拟合 GPD，选择使修正形状参数 ξ 最稳定的阈值。
    
    稳定性判据：修正形状参数 ξ* = ξ - β/(1+ξ) 在相邻阈值间变化最小。
    """
    import numpy as np
    from scipy.stats import genpareto
    
    x = np.array(sorted(data), dtype=np.float64)
    candidates = np.percentile(x, np.linspace(85, 95, n_candidates))
    
    best_u = np.percentile(x, 90)
    best_score = float('inf')
    prev_params = None
    
    for u in candidates:
        exceed = x[x > u] - u
        if len(exceed) < 30:
            continue
        try:
            shape, loc, scale = genpareto.fit(exceed, floc=0)
        except Exception:
            continue
        
        if prev_params is not None:
            prev_shape, prev_scale = prev_params
            # 修正形状参数的变化量
            adj_shape = shape - scale / (1 + shape) if shape > -1 else shape
            prev_adj = prev_shape - prev_scale / (1 + prev_shape) if prev_shape > -1 else prev_shape
            score = abs(adj_shape - prev_adj)
            if score < best_score:
                best_score = score
                best_u = u
        
        prev_params = (shape, scale)
    
    return float(best_u)
```

- [ ] **3.11: 实现 `evt_var(data, p, xi, beta, threshold)`**

```python
def evt_var(data, p, xi, beta, threshold):
    """EVT VaR 计算。
    
    VaR_p = u + (β/ξ) × [(n/Nu × (1-p))^(-ξ) - 1]  for ξ≠0
    VaR_p = u + β × log(Nu / (n × (1-p)))          for ξ=0
    
    若 p 在阈值覆盖范围内（Nu/n ≥ 1-p），返回 None——调用方应使用经验分位数。
    """
    import numpy as np
    x = np.array(data, dtype=np.float64)
    n = len(x)
    nu = np.sum(x > threshold)
    
    if nu < 10:
        return None
    
    tail_prob = n / nu * (1 - p)
    
    if tail_prob < 1.0:
        return None  # p 在阈值覆盖范围内，不需要 EVT 外推
    
    if abs(xi) < 1e-8:
        var = threshold + beta * np.log(tail_prob)
    else:
        var = threshold + (beta / xi) * (tail_prob ** (-xi) - 1)
    
    return float(var)
```

```python
def evt_cvar(data, p, xi, beta, threshold):
    """EVT CVaR（Expected Shortfall）计算。
    
    CVaR_p = VaR_p + (β + ξ × (VaR_p - u)) / (1 - ξ)  for ξ < 1
    """
    var_p = evt_var(data, p, xi, beta, threshold)
    if var_p is None:
        return None
    
    if xi >= 1.0:
        return float('inf')  # 一阶矩不存在
    
    if abs(xi) < 1e-8:
        cvar = var_p + beta
    else:
        cvar = var_p + (beta + xi * (var_p - threshold)) / (1 - xi)
    
    return float(cvar)
```

**pyextremes vs QuantLite 对比**：

| 维度 | pyextremes | QuantLite |
|------|-----------|-----------|
| 成熟度 | 252 stars，经典教材（Coles 2001）实现 | 2025 年新包，较新 |

| 厚尾分布族 | 无 | 有：Student-t, Lévy stable, Kou jump-diffusion |
| 推荐场景 | 经典平稳极值分析，学术严谨性优先 | EVT+风险度量一体化，工程便利性优先 |

**当前建议**：v3 方案已用 scipy 替代手写 MLE，覆盖基本需求。`pyextremes` 或 `QuantLite` 作为未来可选升级路径记录，不在本次实施范围内。
