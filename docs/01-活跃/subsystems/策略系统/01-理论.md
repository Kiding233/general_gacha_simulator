## 一、核心假设

### 1.1 启发式决策假设（缺陷 12）

所有 7 种策略均为**基于规则的启发式**，不追求期望效用最大化或贝尔曼最优性。

**内核假设**：
1. **一阶前瞻（或零阶）**：策略检查当前条件后立即行动，不递归评估未来状态的期望价值。
2. **无延迟价值评估**：没有策略权衡「现在花费资源抽此池」vs「等待更好的池子」。
3. **无保底跨期价值**：没有策略能预测保底计数器在未来池子中的价值。

**理论立场**：
> 抽卡游戏的最优策略通常是 NP-hard 的——多池 × 多保底 × 多资源 × 多目标的随机动态规划问题，其状态空间呈组合爆炸。实际游戏中玩家也使用启发式（如「攒石等到限定池」）。因此，策略系统采用基于规则的启发式而非贝尔曼最优是合理的工程决策。理论审查的重点不是要求实现最优策略，而是**明确文档化这一设计决策**，并在 UI 中标注「启发式结果」。

**文献**：Simon (1956), *Psychological Review*——满意原则（satisficing）；Bellman (1957), *Dynamic Programming*

### 1.2 无统一效用函数（缺陷 13）

各策略服务于不同的隐式目标——成功率最大化 / 资源最小化 / 风险规避——无共同度量标准。

**后果**：
- 「策略 A 优于策略 B」的断言在不指定元目标时无定义。
- 策略比较面板的排名依赖于 GDR 指标的选取——不同 GDR 下排名可能反转。
- 无法在多维目标间做有原则的权衡（如「成功率低 5% 但省 30% 资源」是否更好）。

**理论建议**（P30 方案）：
1. 策略比较结果应明确说明比较维度和 GDR 指标
2. 提供多维雷达图（成功率 / 平均资源消耗 / 最差资源消耗 / 目标卡收集率）替代单一排名
3. 元目标选择器：允许用户选择「按成功率」「按资源效率」「按稳健性」比较

### 1.3 StrategyContext 信息不完备（缺陷 14）

`StrategyContext` 当前不提供：
- 未来资源收入预测（每日任务奖励）
- 未来池子的奖励分布（只有 `pool_id` 和时间）
- 时间贴现因子
- 池间保底链接关系

**影响**：即使未来实现了贝尔曼最优策略，也无法做出准确的跨期决策——缺少未来状态的值函数评估所需的关键信息。

## 二、策略分类学

### 2.1 按决策复杂度

| 层级 | 策略 | 前瞻 | 状态依赖 |
|------|------|------|---------|
| 零阶 | `fixed_count`、`stop_on_target`、`draw_target` | 无 | 仅累计计数 |
| 一阶-简单 | `target_hunting`、`pool_quota` | 无 | 池可用性 |
| 一阶-综合 | `smart` | 无（但含兑换优先逻辑） | 目标/资源/池可用性 |
| 一阶-风险感知 | `pity_reserve` | 无（但使用保底概率） | 保底概率阈值 |

### 2.2 按风险态度

| 风险态度 | 策略 | 特征 |
|---------|------|------|
| 风险中性 | `smart`、`target_hunting`、`pool_quota` | 不考虑保底概率，按需求抽卡 |
| 风险规避 | `pity_reserve` | 仅在保底概率达到阈值时抽卡，等待更高出率 |
| 风险偏好 | 无 | 当前无策略偏好高风险高回报的「梭哈」行为 |

## 三、与外部理论的对照

| 概念 | 文献 | 一致性 |
|------|------|--------|
| 启发式决策 | Simon (1956) 满意原则 | ✅ 与项目一致的工程选择 |
| 贝尔曼最优性 | Bellman (1957) 动态规划 | ❌ 未实施（NP-hard，启发式替代） |
| 期望效用最大化 | von Neumann-Morgenstern (1944) | ❌ 无统一效用函数（缺陷 13） |
| 多目标决策 | Keeney & Raiffa (1976) | ❌ 无多维权衡框架 |
| 策略注册表模式 | GoF Strategy Pattern (1994) | ✅ 标准的策略模式实现 |
| 上下文对象 | Fowler (2002) Context Object | ✅ `StrategyContext` 正确实现 |
