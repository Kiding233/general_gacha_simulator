## P1：设计澄清

### 缺陷 12：所有策略均为贪婪/启发式，不满足贝尔曼最优性

- **来源**：全模块审查 §2.4 + P30 计划
- **严重度**：严重（设计澄清，非代码缺陷）
- **位置**：`core/strategy.py`——7 种策略全部
- **问题**：所有策略均为一阶前瞻（或零阶）：检查当前条件，立即行动，不递归评估未来状态的期望价值。
  - 没有策略能权衡「现在花费资源抽此池」vs「等待更好的池子」
  - 没有策略能预测保底计数器在未来池子中的价值
  - `future_schedules` 已传入 `StrategyContext` 但**没有任何策略使用它**
- **理论立场**：抽卡最优策略为 NP-hard 的随机动态规划问题，启发式是合理的工程选择。需要的是**文档化**而非重写。
- **推荐方案**：
  1. 在 `strategy.py` 模块级 docstring 中明确说明启发式设计决策
  2. UI 中标注「启发式结果」
  3. `future_schedules` 使用或标注为预留字段
  4. 可选：贝尔曼基准（极简化场景的 DP 求解，研究级增强）
- **当前状态**：P30 计划中，未实施

## P2：改善与增强

### 缺陷 13：策略间无统一效用函数

- **来源**：全模块审查 §2.4 + P30 计划
- **严重度**：中等
- **位置**：`core/strategy.py` + `gui/strategy_comparison_panel.py`
- **问题**：各策略服务于不同隐式目标——成功率最大化 / 资源最小化 / 风险规避——无共同度量标准。「策略 A 优于 B」的断言在不指定元目标时无定义。
- **推荐方案**：
  1. 策略比较面板文档：明确说明比较维度（GDR 阈值）
  2. 多维雷达图替代单一排名
  3. 元目标选择器：按成功率 / 按资源效率 / 按稳健性
- **当前状态**：P30 计划中，未实施

### 缺陷 14：StrategyContext 信息不完备

- **来源**：全模块审查 §2.4 + P30 计划
- **严重度**：中等
- **位置**：`core/strategy.py` `StrategyContext` dataclass
- **问题**：不提供未来资源收入预测、未来池奖励分布、时间贴现因子、池间保底链接关系。即使将来实现贝尔曼最优策略，也无法做出准确的跨期决策。
- **推荐方案**（P30 增量补充，低风险）：
  1. `future_resource_gains`：从 `schedule_mgr` 计算未来每日资源收入
  2. `inter_pool_pity_links`：从 `PityEngine` 提取跨池保底继承关系
  3. 暂不补充：未来池奖励分布（超出配置范围）、时间贴现因子（需用户研究）
- **当前状态**：P30 计划中，未实施
