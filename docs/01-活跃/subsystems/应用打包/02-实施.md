## 二、`paths.py` 详析（新增）

### 设计

```python
# gacha_simulator/paths.py
"""路径解析工具——同时支持开发环境和 PyInstaller 打包环境。

使用方式：
    from gacha_simulator.paths import get_config_dir, get_resource, get_user_data_dir

    config_dir = get_config_dir()           # → config/
    icon = get_resource('app_icon.png')     # → resources/app_icon.png
    out_dir = get_user_data_dir('output')   # → %APPDATA%/GachaStat/output/
"""

import os
import sys


def get_base_dir() -> str:
    """返回应用根目录（只读——捆绑的资源文件）。

    开发环境：paths.py 所在目录（= gacha_simulator/ 包目录 = 项目根）
    打包环境：sys._MEIPASS（PyInstaller 的 _internal/ 目录）
    """
    if getattr(sys, 'frozen', False):
        # PyInstaller onedir 模式：sys._MEIPASS 即 _internal/
        return sys._MEIPASS
    # 开发环境：paths.py 位于 gacha_simulator/ 包根
    return os.path.dirname(os.path.abspath(__file__))


def get_config_dir() -> str:
    """返回默认配置目录的绝对路径（只读——捆绑的默认配置）。"""
    return os.path.join(get_base_dir(), 'config')


def get_resource(filename: str) -> str:
    """返回 resources/ 目录下指定文件的绝对路径（只读——捆绑的资源）。"""
    return os.path.join(get_base_dir(), 'resources', filename)


def get_user_data_dir(*subdirs: str) -> str:
    """返回用户数据目录的绝对路径（可写——用户生成的数据）。

    开发环境：项目根下的对应子目录
    打包环境：%APPDATA%/GachaStat/ 下的对应子目录

    用法：
        get_user_data_dir()                  # → %APPDATA%/GachaStat/
        get_user_data_dir('output')          # → %APPDATA%/GachaStat/output/
        get_user_data_dir('output', 'analysis')  # → %APPDATA%/GachaStat/output/analysis/
    """
    if getattr(sys, 'frozen', False):
        base = os.path.join(os.environ.get('APPDATA', os.path.expanduser('~')), 'GachaStat')
    else:
        base = os.path.dirname(os.path.abspath(__file__))
    target = os.path.join(base, *subdirs)
    os.makedirs(target, exist_ok=True)
    return target
```

### 关键设计决策

1. **`get_base_dir()` 在开发环境返回 `gacha_simulator/` 包目录**（而非项目根 `抽卡模拟与分析程序/`）——因为 `gacha_simulator/config/` 目录就位于 `gacha_simulator/` 包内。打包时 `--add-data` 将 `gacha_simulator/config` → `config`，落在 `_internal/config/`，与包结构对齐。

2. **只读 vs 可写路径分离**：`get_config_dir()` 和 `get_resource()` 返回捆绑在包内的只读资源路径（基于 `get_base_dir()`）；`get_user_data_dir()` 返回用户可写数据目录：
   - 开发环境 → 项目根（与 `get_base_dir()` 相同，方便调试）
   - 打包环境 → Windows `%APPDATA%/GachaStat/`（应用数据标准位置）
   - `os.makedirs(target, exist_ok=True)` 确保目录存在，调用方无需手动创建

3. **不导出 `get_base_dir`**——避免调用方绕过封装直接拼接路径。只暴露 `get_config_dir`、`get_resource`、`get_user_data_dir` 三个语义化接口。

4. **无 `try/except` 守卫**——`sys.frozen` 的 `getattr` 默认值 `False` 已处理开发环境回退。`sys._MEIPASS` 在 PyInstaller 中始终存在，不需要额外守卫。

if __name__ == '__main__':

    from multiprocessing import freeze_support

### 变更点

| 行 | 原 | 新 | 原因 |
|----|-----|-----|------|
| +1 | — | `from multiprocessing import freeze_support` + `freeze_support()` | Windows spawn 模式在打包后必需 |
| ~23 | `os.path.join(this_dir, 'gacha_simulator', 'resources', 'app_icon.png')` | `get_resource('app_icon.png')` | 双重嵌套路径打包后失效；`paths` 统一处理 |

**注意**：`freeze_support()` 必须在 `if __name__ == '__main__':` 护卫块的首行调用——在任何其他导入之前。这是 Python `multiprocessing` 文档的要求。

**不影响**：`this_dir` 和 `parent_dir` 的 `sys.path` 插入逻辑（line 4-7）保持不变——打包后 `sys.path` 由 PyInstaller 管理，这些操作无害（`sys.path.insert` 在已有路径时无副作用）。

## 五、`GachaStat.spec` 详析（新增）

import sys
from pathlib import Path

# 项目根目录（.spec 文件所在目录）
_SPEC_DIR = Path(SPECPATH)  # SPECPATH 由 PyInstaller 注入


a = Analysis(

        'sqlite3',
        'test',
        'unittest',
        'pytest',
        'pdb',
        'distutils',

)

    a.pure,
    a.zipped_data,
    cipher=None,
)


exe = EXE(
    pyz,
    a.scripts,
    a.binaries,
    a.zipfiles,
    a.datas,

    [],
    name='GachaStat',               # 可执行文件名 → GachaStat.exe

    bootloader_ignore_signals=False,
    strip=False,
    upx=False,                       # 禁用 UPX——避免杀软误报 + 启动解压开销

### 关键配置说明

| 配置项 | 值 | 原因 |
|--------|-----|------|
| `console=False` | 无控制台窗口 | GUI 应用——避免后台黑窗 |
| `upx=False` | 禁用 UPX 压缩 | 压缩触发更多杀软误报；文件夹分发无需压缩 |
| `name='GachaStat'` | 品牌名 | 与 `app.setApplicationName` 一致 |
| `hiddenimports` | 15+ 模块 | QtWebEngine/matplotlib/scipy 的 C 扩展 + 项目内部动态导入 |
| `excludes` | 6 个 Qt 模块 + tkinter 等 | 减去不使用的 Qt 组件，减少 ~15-25MB |
| `icon` | `app_icon.png` | 应用图标 |

## 六、`analysis_panel.py` 修改详析

### 问题

[analysis_panel.py:1314](gacha_simulator/gui/analysis_panel.py#L1314) 硬编码输出目录为 `os.getcwd()`：

```python
self.output_dir = Path(os.getcwd()) / 'output' / 'analysis'
```

打包后 `os.getcwd()` 不可控——用户从桌面双击 exe 就在桌面建 `output/` 目录。应使用 `paths.get_user_data_dir()` 将输出写入应用数据目录。

### 修改后

```python
from gacha_simulator.paths import get_user_data_dir

# 类初始化中
self.output_dir = Path(get_user_data_dir('output', 'analysis'))
```

### 变更点

| 维度 | 原 | 新 | 原因 |
|------|-----|-----|------|
| 输出根 | `os.getcwd()` | `%APPDATA%/GachaStat/` | 可预测的用户数据目录 |
| 子目录 | `output/analysis` | `output/analysis` | 结构不变 |
| 创建 | 手动 `mkdir` | `paths.get_user_data_dir()` 内部 `os.makedirs` | 调用方无需处理 |
