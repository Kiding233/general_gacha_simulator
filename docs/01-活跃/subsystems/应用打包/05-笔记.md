## 2026-06-04 · 首版实施——8 任务完成

**构建结果**：
- 产物：`dist/GachaStat/` 文件夹（~714MB），含 `GachaStat.exe` + `_internal/`
- PyInstaller 版本：6.20.0
- 构建时间：~75 秒（分析 ~30s + PYZ ~4s + PKG ~80s + COLLECT ~15s）

**实际发现与方案偏差**：

1. 🔴 **spec 缺少 COLLECT**：初始 spec 只有 EXE() 导致生成单文件 337MB。补充 COLLECT() 后正确生成文件夹分发。

2. 🔴 **distutils 与 setuptools 冲突**：排除 `distutils` 导致 `hook-distutils.py` 别名映射失败。移除排除项解决。

3. 🔴 **unittest 被 numpy.testing 依赖**：排除 `unittest` 导致 `scipy→numpy.testing→unittest` 链断裂。移除排除项——增量约 1MB。

4. 🟡 **排除 numba/llvmlite**：无代码引用，节省 ~106MB。

5. 🟡 **实际体积 ~714MB** vs 预估 180-250MB：主要因为 QtWebEngineCore.dll 196MB + Qt6/bin/ 309MB + scipy 55MB。此体积对 QtWebEngine+科学计算栈属正常范围。

6. 🟢 **GUI 在无图形环境启动**：报 ChartWebView 加载失败（plotly.js 文件路径 —— 实际是 QWebEngineView 在 headless 环境无法渲染），ImportError 链路已清除。

**scipy 隐藏导入验证**：5 个 C 扩展全部正确收集，无缺失。无需追加。

**待验证（需桌面环境）**：完整 GUI 功能测试（多进程模拟、图表渲染、用户数据目录、配置导入导出、图标、版本号）。

## 2026-06-04 · 体积优化——714→627MB（-12%）

**手段**：

| 措施 | 节省 |
|------|------|
| matplotlib→惰性导入（`font_config.py` 无调用方） | 12 MB |
| PIL（matplotlib 传递依赖） | 12 MB |
| pandas 排除 | 16 MB |
| statsmodels 排除 | 6.5 MB |
| lxml 排除 | 7 MB |
| Qt 未用 DLL 过滤（3D/PDF/Multimedia/额外皮肤） | ~28 MB |
| plotly widgetbundle.js + 数据集 | ~5 MB |
| **合计** | **~87 MB** |

**剩余体积分布**：
```
PyQt6/Qt6    488 MB  (76%)  ← QtWebEngineCore 195MB 无法缩减
scipy         53 MB  ( 8%)  ← scipy.stats 自身耦合
numpy.libs    20 MB  ( 3%)  ← OpenBLAS
scipy.libs    19 MB  ( 3%)
plotly        13 MB  ( 2%)  ← plotly.min.js + validators
其他          34 MB  ( 5%)
```

**关键发现**：`configure_chinese_font()` 在整个应用代码中无调用方（仅测试脚本引用），其 matplotlib 导入完全死路。惰性导入后 matplotlib + PIL 可安全排除。PyInstaller 静态分析会扫到函数体内的 `import matplotlib`，必须同时在 `excludes` 中显式声明。
