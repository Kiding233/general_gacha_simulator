## 业务问题

当前项目以 Python 源码形式分发，要求用户具备完整的 Python 运行环境：

1. **安装门槛高**：用户需自行安装 Python 3.10+、`pip install -e ".[dev]"`、处理 PyQt6/QtWebEngine 的平台特定依赖
2. **依赖脆弱**：PyQt6 DLL 版本冲突、scipy/numpy 的 MKL/OpenBLAS 后端差异、QtWebEngine Chromium 沙箱兼容性——每个都是潜在的「跑不起来」
3. **启动链路长**：`python -m gacha_simulator.main` 需先解析包路径再加载 Qt 栈，对非技术用户不友好
4. **无卸载机制**：源码目录直接删除即「卸载」，无开始菜单快捷方式、无文件关联

目标：生成一个**自包含的文件夹分发**，用户双击 `.exe` 即可启动 GUI，无需安装 Python 或任何依赖。

## 设计意图

### 核心抽象

- **`paths.py`**：路径解析工具模块——统一开发环境与打包环境的路径获取。封装 `sys.frozen` 检测和 `sys._MEIPASS` 回退。分离只读路径（`get_config_dir`、`get_resource`）与可写路径（`get_user_data_dir`）。
- **`GachaStat.spec`**：PyInstaller 构建配置文件——声明入口点、数据文件、隐藏导入、排除项。
- **`multiprocessing.freeze_support()`**：Windows spawn 模式的必要调用——确保打包后的多进程 Worker 能正确重新导入主模块。

### 打包策略：PyInstaller — 文件夹模式（onedir）

选择 PyInstaller 而非 Nuitka/cx_Freeze/Briefcase 的核心原因：

1. **运行时性能差距可忽略**（<5%）：项目热点在 numpy/scipy C 扩展层（矩阵运算、随机数生成、统计推断），Nuitka 的 Python→C 编译无法加速这些已编译的扩展，只能加速胶水代码
2. **构建速度差 10-20 倍**：PyInstaller 30秒~2分钟 vs Nuitka 10-30分钟
3. **QtWebEngine 成熟度**：PyInstaller 有专门的 QtWebEngine hook，自动收集 Chromium 资源文件和 `QtWebEngineProcess.exe`
4. **文件夹模式优势**：无单文件解压开销（冷启动 <2秒）、杀软误报显著减少、文件级更新可能

### 路径适配模型

```
开发环境                      打包环境
──────────                   ──────────
os.path.dirname(__file__)    sys._MEIPASS (即 _internal/)
    ↓                              ↓
paths.py 封装 get_base_dir()    paths.py 封装 get_user_data_dir()
    ↓                              ↓
get_config_dir()               get_user_data_dir()
→ base_dir/config/             → %APPDATA%/GachaStat/
get_resource(x)                (可写——用户数据、输出文件)
→ base_dir/resources/x
(只读——捆绑资源)
```

### 产物结构

```
dist/GachaStat/
├── GachaStat.exe              ← 启动入口（--windowed，无控制台）
├── _internal/
│   ├── python312.dll          ← 嵌入式 Python 解释器
│   ├── PyQt6/                 ← Qt Python 绑定
│   ├── Qt6/                   ← Qt C++ 运行时 + QtWebEngineProcess.exe
│   ├── numpy/ scipy/ matplotlib/ plotly/
│   ├── config/                ← 用户配置文件（通过 --add-data 注入）
│   │   ├── cards.txt
│   │   ├── gains.txt
│   │   ├── pity.txt
│   │   ├── pools/

```
