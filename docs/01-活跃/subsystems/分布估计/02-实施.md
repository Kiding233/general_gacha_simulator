## 二、退化检测已实现

### 2.1 `_count_distinct()` —— 惰性计算 + 缓存

```python
def _count_distinct(self) -> int:
    """不同取值数——惰性计算，用于退化数据检测（<20 跳过 EVT）。"""
    if self._distinct_count is None:
        if self._n == 0:
            self._distinct_count = 0
        else:
            cnt = 1
            for i in range(1, self._n):
                if self._sorted[i] != self._sorted[i - 1]:
                    cnt += 1
            self._distinct_count = cnt
    return self._distinct_count
```

### 2.2 守卫位置

- `_evt_quantile()` (line 138)：`if self._count_distinct() < 20: return None`
- `_evt_cvar()` (line 190)：`if self._count_distinct() < 20: return None`

两条路径都在首次缓存检查后立即执行退化检测——在调用 `fit_gpd_*()` 和 `evt_var_right()` 之前。

### 2.3 EVT 跳过条件完整清单

| 检查位置 | 条件 | 跳过行为 |
|----------|------|---------|
| `quantile()` | `use_evt=False` | 直接走经验分位数 |
| `quantile()` | `n < 100` | 不触发 EVT，走经验分位数 |
| `quantile()` | `p ∈ (0.1, 0.9)` | 非极端分位数，走经验分位数 |
| `_evt_quantile()` | `distinct < 20` | 退化数据，回退经验分位数 |
| `_evt_cvar()` | `distinct < 20` | 退化数据，回退经验 CVaR |
| `fit_gpd_upper()` | `n < 100` | 样本不足，返回 None |
| `fit_gpd_upper()` | `n_exc < 10` | 超额样本不足，返回 None |
| `_fit_gpd()` | `n_exc < 10` | 超额样本不足，返回 None |
| `_fit_gpd()` | `ξ < -1` | MLE 不存在（Smith 1985），返回 None |
| `_fit_gpd()` | `ξ < -0.5` | MLE 渐近性质不成立，仅警告，点估计仍可用 |
| `evt_var_right()` | `tail_prob > φ` | q 在阈值覆盖范围内无需外推，返回 None |
| `evt_var_right()` | `var ≥ endpoint`（ξ < 0 有界支撑） | 外推越界，返回 None |

## 三、外部依赖

### 3.1 `scipy.stats.genpareto`

- **`genpareto.fit(data, floc=0)`**：对超额值拟合 GPD，floc=0 确保阈值一致性
- **`genpareto.cdf(x, xi, scale=beta)`**：用于 MLE-IC 区间删失似然计算（未来使用）
- **`genpareto.rvs(xi, scale=beta, size=n)`**：用于 Bootstrap 的参数重抽样（`_bootstrap_tail_gpd()`）
- **已知限制**：`genpareto.fit()` 在超额样本 < 10 时数值不稳定——已通过 `_fit_gpd()` 前置守卫处理

### 3.2 `scipy.stats.bootstrap`

- 由 `BootstrapEngine` 使用，分布估计模块不直接依赖
- 提供 BCa（偏差校正加速）和 percentile 两种 CI 方法
- 对 GPD 尾部，当前使用独立路径（`_bootstrap_tail_gpd()`），计划改为委托 EVT 核心

### 3.3 `numpy`

- 排序、分位数计算（`np.quantile()`）、对数运算（`np.log()`）
- 下尾取负法中的取反操作
- `fit_gpd_upper()` 中自适应阈值的分位数选择

## 五、与 Bootstrap 的 GPD 拟合路径重复

### 当前状态：两套独立 GPD 拟合

| 组件 | 文件 | GPD 拟合方式 | 退化检测 |
|------|------|-------------|---------|
| EVT 核心 | `evt_tail.py` | `scipy.stats.genpareto.fit()` + MLE 正则性检查 | 依赖调用方的 `_count_distinct()` 守卫 |
| Bootstrap GPD | `bootstrap.py:205-236` | 直接 `genpareto.fit(-excess, floc=0)` | 仅 `len(excess) < 20` 检查 |
| EmpiricalDistribution | `distribution.py` | 委托给 `evt_tail.py` | `_count_distinct() < 20` |

### 问题

1. **重复实现**：`_bootstrap_tail_gpd()` 重新实现 GPD 拟合逻辑
2. **退化检测不一致**：Bootstrap 路径无 `_count_distinct()` 检查
3. **ξ<-1 正则性检查缺失**：Bootstrap 路径无 Smith (1985) MLE 正则性条件检查
4. **阈值策略不同**：EVT 核心自适应阈值（100-500），Bootstrap 固定 q=0.2

### 计划修复（B2.8）

`bootstrap.py` 的 `_bootstrap_tail_gpd()` 改为委托 `evt_tail.fit_gpd_lower()` + `evt_tail.evt_var_right()`，统一拟合路径。详见 `00-档案.md` §与 Bootstrap 的关系。
