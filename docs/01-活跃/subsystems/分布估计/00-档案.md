## 业务问题

所有模拟结果——GDR 值序列、资源消耗序列、成功/失败标志——都是一组 N 个样本点的无序集合。决策者需要从这些样本中提取两类信息：

1. **中心趋势与离散度**：均值、中位数、标准差、任意分位数。标准统计量，用于回答「典型情况下收益如何」。
2. **尾部极端情形**：下尾 VaR/CVaR（最差 5%/1% 情形下的损失/收益）和上尾乐观分位数（最好 5% 情形下的收益上限）。这些极端分位数对决策敏感性高——5% 和 4.9% 的分位数可能差了几百抽——但尾部样本天然稀少（n=1000 时 α=0.05 仅 50 个样本，α=0.01 仅 10 个），直接使用经验分位数不稳定。

传统做法是直接排序取经验分位数，但对极端尾部（p<0.1 或 p>0.9）尾部样本稀疏导致估计方差大。极端值理论（EVT）通过 GPD 尾部外推——利用头部的超额样本拟合参数化分布，外推到比经验数据更深的分位数——在样本量不变的情况下改善尾部估计精度。

## 设计意图

- **输入**：模拟完成后的 N 条个体 GDR 值（float 序列）
- **处理**：`EmpiricalDistribution` 对排序样本提供经验分位数（中心区域 p∈[0.1,0.9]）和 EVT GPD 外推分位数（尾部 p<0.1 或 p>0.9）。首次调用时惰性拟合 GPD 并缓存参数，后续调用直接复用缓存。
- **输出**：
  - 单变量：`quantile(p)`、`cvar(alpha)`、`summary()` → `DistributionSummary` dataclass（均值/中位数/标准差/极值/分位数字典）
  - 双变量：`JointSamples` 支持联合分布查询——条件分布、条件尾部分布、双变量配对统计
  - 风险分析：`WorstCaseAnalysis`（最差情形 VaR/CVaR）和 `BestCaseAnalysis`（最好情形上分位数）
- **自动退化检测**：不同取值数 < 20 时（如二元 `all_targets`、有限格点 `target_collection`）自动跳过 EVT，回退经验分位数。此时经验分位数本身已精确（格点间距即分辨率上限），EVT 外推无额外收益。
- **下尾统一取负法**：对 Y = -X 拟合上尾 GPD，结果再取负回原尺度——使上尾和下尾共用同一套拟合代码路径，消除上下尾实现不对称。

## 技术实现

### 调用链

```
面板 Worker._run_impl()
  → GachaService.run_simulation(collector=CompactCollector())
  → result_collector.on_result(compact)
  ↓
面板分析代码
  → EmpiricalDistribution(values).quantile(0.05, use_evt=True)
    → p 在中心区域 (0.1, 0.9) → _empirical_quantile(p)（线性插值）
    → p 在尾部 (≤0.1 或 ≥0.9) + n≥100 → _evt_quantile(p)
      → _count_distinct() < 20 → 回退经验分位数（退化检测）
      → p ≤ 0.1（下尾）：fit_gpd_lower(samples) → 取负法 → evt_var_right(q_y, ...) → -VaR_Y = VaR_X
      → p ≥ 0.9（上尾）：fit_gpd_upper(samples) → evt_var_right(p, ...)
  → EmpiricalDistribution(values).cvar(0.05, use_evt=True)
    → 同 quantile，但使用 evt_cvar_right() 公式
  → EmpiricalDistribution(values).summary() → DistributionSummary
```

### 核心类

| 类 | 文件 | 职责 |
|----|------|------|
| `EmpiricalDistribution` | `distribution.py` | 单变量经验分布——排序存储、分位数（经验 + EVT）、CVaR（经验 + EVT）、CDF、条件子集、退化检测、summary |
| `JointSamples` | `distribution.py` | 双变量联合分布——配对存储、条件分布（按第一/第二变量筛选）、条件尾部分布 |
| `DistributionSummary` | `distribution.py` | 不可变 dataclass——mean/median/std/min/max/sample_count/quantiles dict |
| `WorstCaseAnalysis` | `distribution.py` | 最差情形封装——固定 α 下的 VaR/CVaR/条件尾部报告 |
| `BestCaseAnalysis` | `distribution.py` | 最好（乐观）情形封装——固定 α 下的上分位数报告 |

### EVT 尾部拟合函数（`evt_tail.py`）

| 函数 | 职责 |
|------|------|
| `_fit_gpd(exceedances)` | 对超额值拟合 GPD(ξ, β)，floc=0。含 MLE 正则性检查（ξ<-1 拒绝，ξ<-0.5 警告） |
| `fit_gpd_upper(data)` | 上尾 POT——自适应阈值（100-500 超额样本），返回 (ξ, β, threshold, φ) |
| `fit_gpd_lower(data)` | 下尾 POT——统一取负法：Y=-X 后调 `fit_gpd_upper`，返回 (ξ, β, threshold_Y, φ) |
| `evt_var_right(q, ξ, β, u, φ)` | 右尾 VaR 统一公式——分 ξ≈0（指数极限）和 ξ≠0 两路径，含有限支撑端点约束 |
| `evt_cvar_right(q, ξ, β, u, φ)` | 右尾 CVaR 统一公式——ξ<1 时有效（ξ≥1 时一阶矩不存在返回 +inf） |
| `gpd_threshold_stability(data, thresholds)` | 阈值稳定性诊断工具——多候选阈值的 ξ 和 β* 参数稳定性图（Coles 2001, §4.3.4） |

### 关键设计决策

1. **惰性拟合 + 缓存**：`_evt_lower` / `_evt_upper` 实例变量在首次调用 `_evt_quantile()` 时填充，后续调用直接复用。避免重复拟合。
2. **统一取负法**：`fit_gpd_lower(data)` = `fit_gpd_upper(-data)`，消除上下尾的代码重复，同时保证 VaR/CVaR 回原尺度的公式一致。
3. **多级回退**：EVT 拟合路径中任何一步失败（样本不足、GPD 拟合失败、正则性不满足、外推越界）都优雅回退经验分位数，对调用者透明。
4. **委托 scipy**：GPD 拟合委托 `scipy.stats.genpareto.fit(floc=0)`——成熟库维护，规避自研 MLE 实现风险。
5. **条件子集返回 EmpiricalDistribution**：`conditional_bottom_alpha()` / `conditional_top_alpha()` 返回新的 `EmpiricalDistribution`——调用者可在条件子集上继续 query VaR/CVaR/分位数。

### 与 Bootstrap 的关系

`BootstrapEngine._bootstrap_tail_gpd()` 当前直接调用 `genpareto.fit()` 自行拟合 GPD，与 `evt_tail.py` 重复。计划（B2.8）改为委托 `evt_tail.fit_gpd_lower()` + `evt_tail.evt_var_right()` 统一拟合路径。此外 Bootstrap 的百分位法 CI 针对 GPD POT 框架存在覆盖率低估（Schendel & Thongwichian 2017），计划（B2.9）升级为检验反演 Bootstrap（TIB）。
