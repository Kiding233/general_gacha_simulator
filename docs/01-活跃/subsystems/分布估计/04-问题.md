## P2 — 低等级

### 已知问题 1：`weapon_character_ratio` 始终为 0

- **位置**：`core/gdr.py` — `GDRContext.weapon_character_map` 无配置入口
- **影响**：该 GDR 恒为 0 → 完全退化 → `_count_distinct()` 返回 1 → EVT 被正确跳过。不影响其他 GDR，但该指标无实际分析价值（直到配置入口被添加）。
- **处置**：非本模块职责——属于配置系统的缺口。已触发正确的退化检测回退。

### 已知问题 2：Bootstrap-EVT 双套 GPD 拟合路径

- **位置**：`core/bootstrap.py:205-236` vs `core/evt_tail.py`
- **影响**：
  1. 重复实现——维护负担
  2. 退化检测不一致——Bootstrap 路径仅有 `len(excess) < 20`，无 `_count_distinct()` 检查
  3. 正则性检查缺失——Bootstrap 路径无 Smith (1985) ξ<-1 检查
  4. 阈值策略不同——EVT 自适应 vs Bootstrap 固定 q=0.2
- **修复**：已在 P25 中列为 B2.8 子条目

### 已知问题 3：多面板 CPU 争抢（非模块专属）

- **影响**：各分析面板 Worker 线程无全局协调。多面板同时运行时，GPD 拟合在多个线程中被重复触发。
- **缓解**：每个面板的 `EmpiricalDistribution` 实例独立，GPD 拟合缓存是实例级的，至少在同一面板内不重复拟合。跨面板的重复拟合不可避免，但 GPD 拟合开销极小（~几次 MLE 迭代）。
- **处置**：非模块职责——属于全局 Worker 调度问题。

### 已知问题 4：`_extract_cost_per_draw` 只取第一个池

- **位置**：`gui/resource_search_panel.py` (line 80-91)、`gui/retreat_search.py` (line 107+)
- **影响**：多池成本不同时「每抽成本」可能不准确。但这是 GUI 层的成本提取逻辑，不影响 `EmpiricalDistribution` 本身。
- **处置**：非模块职责——属于 GUI 面板修复。
