# Bootstrap 引擎 -- 理论

> 创建日期：2026-06-02（首次建档）
> 理论来源：Efron & Tibshirani (1993) *An Introduction to the Bootstrap*、scipy 实现、Schendel & Thongwichian (2017)

---

## 核心假设

1. **独立同分布（i.i.d.）**：Bootstrap 假设输入数据是 i.i.d. 样本。本项目中，N 次模拟使用不同随机种子独立运行，满足该假设。对偶变量法（antithetic variates）会打破独立性——对此提供 `paired=True` 模式，重抽样 N/2 对而非 N 个个体。

2. **渐近正态性（BCa 需要）**：BCa 方法的二阶精度依赖于统计量抽样分布的渐近正态性。对于偏离正态严重的统计量（如极端分位数），BCa 的加速因子 a（通过 Jackknife 估计）可能不稳定。

3. **GPD 拟合需 n >= 10 超额**：`_bootstrap_tail_gpd()` 要求门槛值以下的超额数量 >= 20（代码实际为 >= 20，严于理论最低要求 >= 10），否则回退标准 Bootstrap。

4. **Hill 估计量仅适用于正数数据**：`hill_estimator()` 过滤 `data > 0`，n < 20 时返回 `inf`（即非厚尾）。

## 已知局限性

### 局限性 1：Jackknife 上限 1000 截断

**代码位置**：`bootstrap.py` `_scipy_bootstrap` 内部隐含（scipy 的 BCa 方法内部对 n > 1000 时 Jackknife 截断为前 1000 个 leave-one-out 估计）。

**影响**：加速因子 a 的精度被截断，收敛速度从 O(n^(-1)) 降为 O(1000^(-1))。n 越大，截断损失越大。

**方案**：n > 1000 时随机抽样 1000 个 Jackknife 点（而非固定取前 1000），保持无偏性。

### 局限性 2：厚尾检测未自动采纳

**代码位置**：`bootstrap.py` `bootstrap_mean()` 和 `bootstrap_quantile()` 有 `auto_heavy_tail=False` 默认关闭。

**影响**：`detect_heavy_tail()` 返回 `{alpha, heavy_tail, recommendation}`，但公开方法不自动调用它。面板需主动检查——当前无面板集成，形同虚设。

**方案**：在 `bootstrap_mean()` 中开启 `auto_heavy_tail=True` 时自动检测并切换 GPD-param Bootstrap。

### 局限性 3：BCa 在小 n 时不稳定

**代码位置**：`bootstrap_probability()` 和 `bootstrap_mean()` 在 `use_bca=True` 时无条件尝试。

**影响**：Jackknife 加速因子 a 在 n < 30 时方差极大（分母 `sum(jk_dev^2)^1.5` 极不稳定）。当前代码通过 try/except 捕获 scipy 异常后回退百分位法，但没有 n < 50 时的主动守卫。

**方案**：n < 50 时主动跳过 BCa，直接使用百分位法，标注 `'percentile (n<50, BCa unstable)'`。

### 局限性 4：m-out-of-n 的 m 选择策略缺失

**现状**：`_select_m(n)` 仅使用 Bickel & Sakov (2008) 经验公式 `m = n^(2/3)`，下限为 20。

**影响**：m 选太小增加方差，选太大保留偏差。经验公式在极端厚尾（α < 1）下可能选得过大。

### 局限性 5：Hill 估计量的 k 选择缺失

**现状**：`_select_hill_k()` 实现了基于 Hill 图拐点检测的自动选择，在 k 的稳定区域选取中位 k。

**影响**：Hill 估计量对 k（顺序统计量个数）极度敏感。拐点检测方法在样本量较小或尾部噪声较大时可能不可靠。当前无 k 选择的交叉验证或 Hall 自适应方法。

### 局限性 6：BCa 在离散 Bootstrap 分布下可能失效

**现状**：`bootstrap_probability()` 在 use_bca=True 时先尝试 BCa，失败则回退。

**影响**：罕见事件（p ~ 0.1%）的成功/失败数据产生的 Bootstrap 均值分布高度离散（B 个复制值只能取有限个离散值），BCa 的连续分布假设可能不成立。当前通过 try/except 回退，但没有主动的离散度检测。

## Bootstrap 方法选择逻辑

```
输入数据 → 判断类型
  ├── 二分类（success_flags）
  │     ├── n >= 50, p 不在边界 → BCa
  │     └── n < 50 或 p ~ 0/1 → percentile
  ├── 连续量（values）
  │     ├── 均值 → auto_heavy_tail?
  │     │   ├── 是 + 厚尾检测 → m-out-of-n
  │     │   └── 否 → BCa（n >= 50）或 percentile
  │     └── 分位数 → q <= 0.1?
  │           ├── 是 → GPD-param Bootstrap
  │           └── 否 → percentile
  └── 离散分布 → TVD
```

## 与其他子系统理论的交叉

- **分布估计/EVT**（`evt_tail.py`）：GPD-param Bootstrap 的 GPD 拟合逻辑应委托 EVT 模块统一接口，避免重复实现。当前为独立调用 `genpareto.fit()`。
- **GDR 系统**：Bootstrap CI 嵌入 GDR 统计表格（analysis_panel 均值/中位数/VaR 列），连续型 GDR 指标的理论分布直接决定 Bootstrap 方法选择（是否需要厚尾检测）。
- **小样本概率估计**：process_analysis_panel 的概率列已有 Wilson 得分区间（专门针对二项比例），Bootstrap CI 在此场景下冗余且可能不如 Wilson 精确。
