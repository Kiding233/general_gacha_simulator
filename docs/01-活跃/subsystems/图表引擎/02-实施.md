## 二、PlotlyRenderer.build_figure() 分发逻辑

```python
class PlotlyRenderer:
    def build_figure(self, spec: ChartSpec) -> go.Figure:
        if spec.chart_type == "histogram":
            return self._build_histogram(spec)
        elif spec.chart_type == "cdf":
            return self._build_cdf(spec)
        elif spec.chart_type == "ridge":
            return self._build_ridge(spec)
        # ... 10 种类型全部分发
```

每个 `_build_*` 方法：
1. 从 `spec.data` 提取数据
2. 构建 `go.Figure`（或 `make_subplots`）
3. 应用 `spec.annotations`（`add_vline`/`add_hline`）
4. 应用 `spec.shaded_regions`（`add_vrect`/`add_hrect`）
5. 应用 `spec.layout_hints` 中的布局提示
6. 设置中文字体（`font_family="Microsoft YaHei"`）

## 三、ChartWebView 详析

### 初始化流程

```
ChartWebView.__init__()
  → QWebEngineView.__init__(parent)
  → _setup_profile()  # 配置缓存路径、下载处理
  → settings().setAttribute(...)  # 禁用不需要的功能
  → loadFinished.connect(_handle_load_finished)
  → setHtml(_SKELETON_HTML, baseUrl=QUrl.fromLocalFile(...))
  # 此时 WebView 异步加载骨架页
```

### set_charts() 流程

```python
def set_charts(self, charts: List[Tuple[str, ChartSpec]]):
    figures_json = {}
    for name, spec in charts:
        fig = PlotlyRenderer().build_figure(spec)
        figures_json[name] = fig.to_json()

    js_code = f"rebuildAll({json.dumps(figures_json)});"
    self.page().runJavaScript(js_code)
```

### 增量更新

```python
def update_chart(self, name: str, spec: ChartSpec):
    fig = PlotlyRenderer().build_figure(spec)
    js_code = f"rebuildChart('{name}', {fig.to_json()});"
    self.page().runJavaScript(js_code)
```

### 防闪烁机制

- `_pending_charts`：若 `set_charts()` 在骨架页加载完成前被调用，暂存到此列表
- `_handle_load_finished()`：加载完成后处理 `_pending_charts`

    samples=gdr_values,
    title="GDR 分布",
    xlabel="GDR 值",
    mean_line=True,
    quantile_lines=[0.05, 0.95],

    ("分布直方图", hist_spec),
