## 潜在改进方向（非问题，记录备查）

以下条目不属于问题，但若未来有扩展需求，可供参考：

1. **HTML 模板合并**：`plotly_charts.py` 中的 `_HTML_TEMPLATE` 和 `chart_webview.py` 中的 `_SKELETON_HTML` 存在内容重复。可考虑将 HTML 模板提取为独立文件或统一到 `chart_webview.py` 中。
2. **渲染后端抽象**：当前 `PlotlyRenderer` 直接 `import plotly`，无后端接口抽象。若未来需要支持 ECharts 或 D3 作为替代后端，需要引入 `ChartRenderer (ABC)` 接口。
3. **ChartSpec 序列化**：`ChartSpec` 包含 `np.ndarray`，不能直接用 `json.dumps` 序列化。若未来需要跨进程传递 `ChartSpec`（如子进程生成图表），需要增加序列化/反序列化支持。
4. **图表导出**：当前无「导出图表为 PNG/SVG/PDF」的功能。Plotly 支持 `fig.write_image()`（需要 kaleido 包），可在 `ChartWebView` 中增加导出按钮。
