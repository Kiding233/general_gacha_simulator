## 二、缺陷详解与改进方案

### 2.1 缺陷 R1 / 26：二分搜索 + 蒙特卡洛噪声 → 收敛无保证（严重）

**现状**：`search_min_resource` 标准二分搜索每步固定运行 500 次模拟估计 `p(resource_mid)`，然后根据 `p >= threshold` 决定搜索方向。当真实成功率接近阈值时，SE ≈ 0.022（p=0.95），二分决策约 22% 概率错误选择方向。

**理论后果**（Waeber, Frazier & Henderson, 2013, *Operations Research*）：
- 单步决策错误概率 q 导致最终估计的期望绝对误差 ≥ O(q × 区间宽度)
- 噪声将二分搜索的对数收敛率退化为线性
- 同一数据集两次搜索可能返回差异显著的结果（如 12000 vs 14500）

**推荐方案：序贯概率比检验（SPRT）替代固定 N 采样**

```
while r_hi - r_lo > epsilon:
    r_mid = (r_lo + r_hi) / 2
    while not decided:
        run_one_simulation(r_mid)
        p_hat = current_success_rate
        n = current_sample_size
        # 若 p_hat 的置信区间完全高于或低于 threshold，则决策
        if ci_lower(p_hat, n) > threshold → r_hi = r_mid
        if ci_upper(p_hat, n) < threshold → r_lo = r_mid
        if n > max_samples → 保守决策（选择 r_hi）
```

**优势**：
- 远离阈值的点（p≈0.3 或 p≈0.99）只需很少模拟（可能 <100 次）
- 靠近阈值的点自动分配更多模拟（可达 2000+ 次），确保决策可靠
- 总体模拟次数通常比固定 N=500 减少 40-60%
- 每步决策附带置信水平，提供自然的不确定性量化

**风险**：低——SPRT 不改变二分搜索整体结构，仅替换内层固定 N 采样循环。

**文献**：Waeber et al. (2013); Bechhofer, Santner & Goldsman (1995), *Design and Analysis of Experiments for Statistical Selection*

### 2.2 缺陷 24 / R2：Pareto 声称不严格（严重）

**现状**：
- `search_max_targets` 仅探索 N+1 个点（全量 → 按错失代价权升序逐一移除），而非 2^N 个可能子集
- `search_pareto` 调用 `search_max_targets`，同样仅探索 N+1 个点
- 移除顺序固定（按 miss_cost 权重），不重新考虑已移除的卡
- 不执行支配性过滤——被支配点仍作为「前沿」呈现

**推荐方案（两阶段）**：

**阶段 1（低风险，立即执行）——UI 降级声称 + 支配性过滤**：
1. GUI 中将「Pareto 前沿搜索」改为「资源-目标权衡曲线（贪心近似）」
2. 在 `search_pareto` 结果返回前增加支配性过滤：点 A 支配点 B 当且仅当 A 在所有目标上 ≥ B 且至少在一个目标上 > B，被支配点标记但不移除（UI 中可切换显示/隐藏）
3. 补充 Tooltip 说明：「此为贪心近似，非严格 Pareto 前沿。算法沿单一移除路径搜索，可能遗漏非支配解。」

**阶段 2（中风险，后续实施）——真正的 ε-Pareto 前沿**：
1. 引入分支定界或 NSGA-II 风格的进化搜索
2. 至少保证非支配集是真实 Pareto 前沿的 ε-近似（Miettinen, 1999）
3. 利用 `JointSamples` 的条件分布机制共享模拟信息（见缺陷 R4）

**文献**：Miettinen (1999), *Nonlinear Multiobjective Optimization*; Chvátal (1979), *Mathematics of Operations Research*

---

### 2.3 缺陷 25 / R3：翻倍上界可能失败且两实现不一致（中等）

**现状**：
- `retreat_search.py`：翻倍 10 次（50×2^10 = 51200 抽）
- `resource_search_panel.py`：翻倍 15 次（50×2^15 ≈ 1,638,400 抽）
- 两者均无理论推导的最坏情况保证
- 均无上界检测：若实际需求 > r_hi，二分搜索在无效区间上运行，返回的点声称 `probability ≈ threshold` 但实际不满足

**推荐方案**：
1. 统一两个实现的翻倍参数为 15 次
2. 将翻倍逻辑提取为 `_find_upper_bound(initial_guess, cost_per_draw, max_iter=15)` 公共函数
3. 二分搜索结束后检测 `probability_at_r_hi < threshold`——若成立，报告「搜索上界不足」而非返回无效结果
4. 考虑基于保底最坏情况的解析上界：`max_draws = sum(各池硬保底阈值) × 各池单抽成本`——对于典型配置，这通常 < 50,000 抽，可验证翻倍上界的充分性

---

### 2.4 缺陷 R4：每步独立模拟不共享信息（中等）

**现状**：`search_pareto` 对每个目标子集调用 `search_min_resource`，后者在二分搜索每步运行 500 次模拟。N=5 张目标卡时最坏约 24,000 次独立模拟。

**推荐方案（供阶段 2 考虑）**：
- 采用**随机克里金（Stochastic Kriging）**或**贝叶斯优化**——将模拟结果视为黑箱函数 `p(resource, target_set)` 的噪声观测
- 高斯过程代理模型预测未探索区域的概率，指导下一探索点选择
- 可在数倍样本量下达到相同精度

**文献**：Frazier (2018), *Operations Research*; Ankenman, Nelson & Staum (2010), *ACM TOMACS*

**依赖**：此改进应在 SPRT 方案（缺陷 R1）之后实施，因为 SPRT 改变了单次搜索的采样模式。

---

### 2.5 缺陷 R5：错失代价权重缺乏基数解释（中等）

**现状**：`search_max_targets` 按 miss_cost 权重升序移除目标卡。权重由用户主观输入（默认 1.0），既无理论校准也无基数含义：
- 所有权重默认相同时按卡 ID 字母序移除——完全任意
- `miss_cost=1.5` vs `1.0` 的 1.5 倍增量无实际含义

**推荐方案**：
1. 在 UI 中标注：「权重仅影响目标卡的移除顺序，不代表绝对价值比例」
2. 为默认权重相同的情况，提供替代排序方案（如按稀缺度/所在池顺序），而非卡 ID 字母序
3. 长期方向：采用 Swing Weighting（von Winterfeldt & Edwards, 1986）引导用户系统引出权重，但此为 UX 研究课题，远超当前范围

**文献**：Keeney & Raiffa (1976/1993), *Decisions with Multiple Objectives*; Saaty (1980), *The Analytic Hierarchy Process*

---

### 2.6 缺陷 R6：epsilon 精度缺乏统计依据（低等）

**现状**：`precision_draws=1` → `epsilon = cost_per_draw × 1`。对于 160 成本的单抽，epsilon=160。但蒙特卡洛噪声的标准差通常远大于此（p=0.5 时约 0.022 × 总资源值，典型 ~500-1000）。

**推荐方案**：
1. 将 epsilon 与蒙特卡洛噪声水平关联：`epsilon = max(cost_per_draw, z_{1-α/2} × SE × resource_range)`
2. 至少将 `precision_draws` 默认为 5-10 抽而非 1 抽
3. 在 UI 中提示当前精度设置与估计噪声水平的关系

## 四、与现有计划的关系

- **P8（面板合并）**：P8 关注 UI 层面合并三个搜索面板，本计划关注底层算法的理论正确性。P8 合并后的统一面板应使用本计划修复后的算法，建议先执行本计划阶段一（UI 降级 + 上界统一），再进行 P8 面板合并，避免在合并代码中固化当前有缺陷的算法行为。
- **P18（Bootstrap 修复）**：Bootstrap 为二分搜索提供置信区间基础设施。P18 的 CI 方法可复用于 SPRT 方案中的置信区间计算。
- **P3（Bootstrap 引擎）**：SPRT 决策依赖 `ci_lower/ci_upper`，可复用 P3 的 Bootstrap 置信区间方法（BCa 或百分位法）。
