## 二、ConfigPanel 类结构

### 2.1 信号

```python
config_changed = pyqtSignal(dict)  # 配置变更时发出，携带变更摘要
```

### 2.2 核心方法

| 方法 | 用途 |
|------|------|
| `set_store(store: ConfigStore)` | 接收外部配置对象，触发表格刷新（`_refresh_all()`） |
| `get_store() -> ConfigStore` | 返回当前配置对象（与 `set_store` 传入的同一引用） |
| `_setup_ui()` | 构建整体布局（QSplitter + left_tabs + 预览面板） |

### 2.3 子标签 setup 方法

| 方法 | 子标签 |
|------|--------|
| `_setup_card_def_tab()` | 卡牌定义表格 |
| `_setup_resource_tab()` | 资源管理表格 + 添加/删除按钮 |
| `_setup_pool_config()` | 卡池配置（模板表格 + 排期表格 + 分布编辑弹窗） |
| `_setup_pity_config()` | 保底机制表格 + 参数编辑 |
| `_setup_strategy_tab()` | 策略选择 + 目标卡表格 + 初始资源表格 + 收入规则表格 |
| `_setup_weight_config()` | 三级权重配置（desire/miss_cost/card_value） |
| `_setup_preview()` | 预览面板（定时器驱动的摘要视图） |

### 2.4 关键设计模式

- **原地编辑**：`ConfigPanel` 直接修改 `set_store()` 传入的 `ConfigStore` 对象的所有字段。`get_store()` 返回同一引用。这简化了数据流但放弃了 undo/redo 能力
- **信号驱动**：表格编辑通过 PyQt6 信号（`cellChanged`、`itemChanged`）自动触发 `config_changed` 信号，`MainWindow` 连接此信号以更新预览
- **弹窗编辑复杂字段**：分布文件通过 `DistributionEditor` 弹窗编辑，兑换池目标卡通过专用弹窗编辑
