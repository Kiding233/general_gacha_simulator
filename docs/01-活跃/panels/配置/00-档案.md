# 配置面板 -- 档案

> 创建日期：2026-06-02（首次建档）
> 关联模块：`gui/config_panel.py`、`core/config_store.py`、`core/config_io.py`、`config/` 默认配置目录
> 关联计划：无（稳定/简单模块）

## 业务问题

抽卡模拟系统需要管理复杂的配置数据——多个卡池、多套保底机制、目标卡清单、资源收入规则、策略选择参数、加权满意度权重等。用户需要一个集中的界面来编辑、保存和加载这些配置，而非手动编辑散落的竖线分隔文本文件。

配置面板作为 10 个 Tab 中的第一个，是用户启动模拟前的唯一配置入口。它承担三个职责：
1. **编辑**：为所有配置项提供 GUI 控件（表格、下拉框、旋钮、文本输入）
2. **持久化**：将内存中的 `ConfigStore` 对象序列化为 `config/` 目录下的竖线分隔文本文件
3. **预览**：实时展示当前配置的汇总视图，帮助用户快速验证配置正确性

## 设计意图

### 核心抽象

- **ConfigStore**（`core/config_store.py`）：纯数据 dataclass，是所有配置项的 in-memory 表示。包含 10 个列表字段：`resource_defs`、`card_defs`、`pool_defs`、`pity_config`、`gain_rules`、`day_overrides`、`target_cards`、`initial_resources`、`card_weights`、`strategy_params`。无任何 GUI 依赖。
- **ConfigPanel**（`gui/config_panel.py`，2927 行）：QWidget，通过 `set_store()` 接收 `ConfigStore` 实例，通过 `get_store()` 返回编辑后的实例。编辑过程直接在传入的 `ConfigStore` 对象上原地修改，因此调用方（`MainWindow`）无需额外的提取步骤。
- **竖线分隔文本格式**（`core/config_io.py`）：`load_store_from_directory()` / `save_store_to_directory()` — 将 8 个文本文件（`schedule.txt`、`targets.txt`、`gains.txt`、`pity.txt`、`cards.txt`、`resources.txt`、`initial_resources.txt`、`weights.txt` 及 `pools/*.txt`）与 `ConfigStore` 对象之间双向转换。

### 界面布局

- 左侧：QSplitter 中的 QTabWidget，含 6 个子标签（卡牌定义、资源管理、卡池配置、保底机制、策略与目标、权重配置）
- 右侧：预览面板，实时汇总当前配置的关键信息（使用定时器轮询更新）
- 底部：加载/保存按钮，支持从 `config/` 目录加载默认配置或保存当前配置

## 技术实现

### 数据流

```
config/ 目录（竖线分隔文本）
    ↓ load_store_from_directory()        [core/config_io.py]
ConfigStore（dataclass）
    ↓ MainWindow._load_default_config()  [gui/main_window.py]
    ↓ config_panel.set_store(store)
ConfigPanel 编辑（原位修改 store）
    ↓ MainWindow._dispatch_config()
各面板 set_store(store_copy)
    ↓ + 模拟触发
SimulationEnvBuilder.from_config_store() [service/batch_simulator.py]
    ↓
SimulationEnv → Worker → CompactResult
```

### 子标签详情

| 子标签 | UI 控件 | 对应 ConfigStore 字段 | 对应配置文件 |
|--------|---------|----------------------|-------------|
| 卡牌定义 | QTableWidget（可编辑表格） | `card_defs: List[CardDefEntry]` | `cards.txt` |
| 资源管理 | QTableWidget + 添加/删除按钮 | `resource_defs: Dict[str, str]` | `resources.txt` |
| 卡池配置 | 模板表格 + 排期表格 + 分布编辑弹窗 | `pool_defs: List[PoolEntry]` | `schedule.txt` + `pools/*.txt` |
| 保底机制 | 保底表格 + 参数编辑 | `pity_config: PityConfig` | `pity.txt` |
| 策略与目标 | 策略下拉框 + 目标卡表格 + 初始资源表格 + 资源收入表格 | `target_cards`、`initial_resources`、`gain_rules`、`strategy_params` | `targets.txt`、`initial_resources.txt`、`gains.txt` |
| 权重配置 | 可选权重系统——渴望度/未命中成本/卡牌价值三级配置 | `card_weights: List[CardWeightEntry]` | `weights.txt` |

### 关键辅助函数

- `_bonus_to_text()` / `_parse_bonus_text()`：将额外资源字段（首次奖励、第 N 次奖励、超出奖励）序列化/反序列化为紧凑文本格式（`ft:... & nth:... & xs:...`）
- `_infer_pool_type()`：根据池 ID 前缀（`pool_w`→武器、`pool_e`→兑换）自动推断池类型
- `_pool_row_bg()`：根据池类型返回行背景色（兑换=浅黄、武器=浅蓝、复刻=浅绿）
- `_gain_rule_type_to_gui()` / `_gui_gain_type_to_store()`：资源收入规则类型在 GUI 中文名与内部 `rule_type` 字符串之间的互转

### 实施历史

- **v1.0~v1.8**：配置面板随项目逐步扩展，从最初的简单表格演变为 6 子标签 + 分布编辑弹窗的完整编辑器
- **v1.9**：引入权重配置子标签（渴望度/未命中成本/卡牌价值三级配置），预留权重方案切换控件
- **当前状态**：稳定。配置面板是项目最成熟的 GUI 组件，功能覆盖所有 `ConfigStore` 字段，无已知严重缺陷
