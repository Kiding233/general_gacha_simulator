## 一、业务问题

抽卡游戏中有多种决策策略——按需追卡、保底预留、指定池配额、目标即停、固定次数等等。不同策略在相同的游戏环境中可能产生截然不同的结果。玩家和决策者需要回答：

1. **哪个策略的效率最高？**（平均抽卡数、平均目标获取数、成功率）
2. **哪个策略最稳健？**（在运气好/差的极端情况下表现如何）
3. **策略之间的差异是否统计显著？**还是仅仅是随机波动？
4. **在多目标空间中，各策略的帕累托前沿在哪里？**（效率 vs 风险 vs 资源消耗）

### 关键业务场景

- 攻略组发布「不同抽卡策略的横向评测」
- 玩家根据自身风险偏好选择策略
- 游戏设计师评估不同策略对玩家留存/付费的影响
- 版本更新后（保底规则变更）重新评估策略排名变化

## 二、设计意图

### 2.1 当前实现（v2，P19 Phase 1+2 已完成）

#### 数据管理层

- **ResultStore** (`core/result_store.py`): 集中式 QObject 数据管理层，CRUD + 信号 + JSON 持久化
- **StoredDataset**: 封装原始模拟数据 + ComparabilityFingerprint + 分析缓存
- **ComparabilityFingerprint**: 9 维度指纹（config_hash / strategy_name / target_cards / initial_resources / stop_condition / seed_start / seed_end / num_simulations / pool_ids）
- **ComparabilityDiff**: 4 种比较模式识别（策略比较/配置对比/多重差异/变异分析）

#### 比较分析引擎

- **ComparisonAnalyzer** (`core/comparison_analyzer.py`): 四层递进分析
  - L1: DescriptiveStats（均值/中位数/std/偏度/CVaR₀.₀₅/成功率/min/max）
  - L2: DD Bootstrap 随机占优（FSD/SSD/TSD）
  - L3: 假设检验（KS/MWU/ttest + Holm-Bonferroni/Benjamini-Hochberg）
  - L4: 双 GDR 帕累托前沿

#### GUI 面板

- **数据管理面板** (`gui/data_manager_panel.py`): 数据集表格 + 可比性检查 + 工具栏
- **比较分析面板** (`gui/comparison_analysis_panel.py`): L1-L4 递进分析 + Plotly ECDF/PMF/帕累托图表 + 后台线程

#### 核心设计理念

```
模拟与分析彻底分离
  ↓
ResultStore（集中式数据管理）
  ├─ 数据集 A：策略 smart × 配置 V1 × N=1000
  ├─ 数据集 B：策略 pool_quota × 配置 V1 × N=1000
  └─ 数据集 C：策略 smart × 配置 V2 × N=1000
       ↓
比较分析面板（从 ResultStore 选取 2+ 数据集）
  ├─ 配对模式（同种子+同N）→ 反事实遗憾 + 配对 t 检验
  └─ 分布模式（不同种子）→ KS 检验 + 随机占优 + MWU
```

**比较范围扩展**：不仅比较策略，也支持比较「同一策略 + 不同配置」、「同一配置 + 不同初始资源」等任意数据集组合。

### 2.3 比较分析引擎（ComparisonAnalyzer）

P19 计划在 `core/` 中新增独立模块，从各数据集的 `PerSimValues` 中提取指标进行成对比较：

**两种模式自动检测**：

| 模式 | 条件 | 方法 |
|------|------|------|
| 配对模式 | seed 相同 + N 相同 | 反事实遗憾、配对 t 检验、Wilcoxon 符号秩 |
| 分布模式 | 否则 | KS 检验、Mann-Whitney U、置换检验、随机占优、Cohen's d |

**多重比较校正**：所有假设检验输出原始 p 值 + Holm-Bonferroni（FWER）+ Benjamini-Hochberg（FDR）三组矩阵。

**可视化**：

| 图表 | 内容 |
|------|------|
| CDF 叠加图 | 选定指标的所有数据集 CDF 曲线叠加 |
| 帕累托散点图 | X=期望收益，Y=风险(CVaR)，每个数据集一个点 |
| 效应量热力图 | n×n 矩阵，颜色深浅 = Cohen's d |
| 指标对比明细表 | 每行一个数据集，均值/中位数/标准差/CVaR₅%/成功率 |
| 占优矩阵 | FSD/SSD 随机占优关系 |

### 3.1 当前文件结构

| 组件 | 文件 | 职责 |

| `ResultStore` | `core/result_store.py` | 集中式数据管理：CRUD + 信号 + JSON 持久化 |

| `DataManagerPanel` | `gui/data_manager_panel.py` | 数据管理 Tab：数据集 CRUD + 可比性检查 |
| `ComparisonAnalysisPanel` | `gui/comparison_analysis_panel.py` | 比较分析面板：L1-L4 UI + Plotly 图表 |
| `ComparisonWorker` | `gui/comparison_analysis_panel.py` | QThread：后台计算 L1+L2+L3 |

### 3.2 当前数据流

```
批量模拟完成 → on_simulation_finished()
  ├─ 构建 ComparabilityFingerprint（9 维度）
  ├─ 构建 StoredDataset（原始数据 + fingerprint + GDRContext 序列化）
  └─ ResultStore.add() → data_manager_panel._refresh_table()

数据管理面板：
  ├─ 表格展示所有数据集
  ├─ 勾选 ≥2 → 可比性检查（8 维度差异矩阵 + 比较模式提示）
  └─ 点击「开始比较」→ 切换到比较分析面板

比较分析面板 (ComparisonWorker QThread)：
  ├─ compute_gdr_values_for_datasets() → 逐次 GDR 值
  ├─ L1: DescriptiveStats.compute() × N
  ├─ L2: compute_dominance_matrix() × 3 (FSD/SSD/TSD, n_bootstrap=1000)
  └─ L3: compute_pvalue_matrix() (KS/MWU/ttest + Holm/BH/raw)
       │
       ▼
  finished.emit(results) → 主线程更新：
    ├─ L1: 描述统计表 + ECDF/PMF Plotly (ChartWebView)
    ├─ L2: FSD/SSD/TSD HTML 矩阵 (颜色编码)
    ├─ L3: p 值矩阵 (QTableWidget, 颜色编码)
    └─ L4: 帕累托前沿 Plotly 散点图 (手动触发)
```

### 3.3 GUI 布局（v2）

```
┌─ 控制栏 ─────────────────────────────────────────────────┐
│ GDR指标:[▼] 阈值:[1.0] 检验:[MWU▼] 校正:[BH▼] [运行分析] │
├─ 可滚动内容区 ───────────────────────────────────────────┤

│ │ FSD(HTML表格) | SSD(HTML表格) | TSD(HTML表格)         │ │
│ ├─ L3 假设检验 ────────────────────────────────────────┤ │
│ │ p 值矩阵 (QTableWidget, 颜色编码)                      │ │
│ ├─ L4 帕累托前沿 ──────────────────────────────────────┤ │
│ │ X轴GDR:[▼] Y轴GDR:[▼] [计算前沿]                     │ │
│ │ 帕累托散点图 (ChartWebView, Plotly)                    │ │
│ └────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```
