## P1 — 高优先级缺陷

### ~~P1-01：面板独立王国——无法与其他面板联动~~ ✅ 已解决

- **来源**：P19 问题诊断第 3 条
- **解决日期**：2026-06-04 (P19 Phase 1)
- **方案**：ResultStore 集中式数据管理 + 数据管理 Tab。模拟完成自动存入 ResultStore，所有面板共享同一批数据集。比较分析面板从 ResultStore 加载数据而非自行运行模拟。
- **相关提交**：`c4d628f` `c6d75c1` `81058d3` `6f7871a`

### ~~P1-02：成功率定义与 GDR 框架脱节~~ ✅ 已解决

- **来源**：代码审查
- **解决日期**：2026-06-04 (P19 Phase 2)
- **方案**：ComparisonAnalyzer 通过 `compute_gdr_values_for_datasets()` 使用 GDR 框架统一判定成功/失败。用户可在比较分析面板自由选择 13 种 GDR 指标和自定义阈值。
- **相关提交**：`fb15090` `5432462`

### ~~P1-03：无统计检验——肉眼比较不可靠~~ ✅ 已解决

- **来源**：P19 问题诊断第 3 条 + L1
- **解决日期**：2026-06-04 (P19 Phase 2)
- **方案**：ComparisonAnalyzer 提供 KS/MWU/ttest 三种检验 + Holm-Bonferroni (FWER) / Benjamini-Hochberg (FDR) / 原始 p 值三种输出。L2 随机占优 (FSD/SSD/TSD) 通过 DD Bootstrap 计算。
- **相关提交**：`fb15090` `5432462`

### P1-04：无配对/反事实分析 ⏸ 暂缓

- **来源**：P19 关键设计决策第 5 条
- **严重度**：高（丢弃了大量信息）
- **现状**：Phase 3 暂缓。当前机制：若两个数据集种子区间和 N 相同，可比性检查面板会标注为「变异分析」，提示用户可评估蒙特卡洛噪声幅度。正式配对模式（配对 t 检验、反事实遗憾）尚未实现。
- **计划方案**：P19 Phase 3——配对模式 + 配对 t 检验 + 反事实遗憾分析
- **状态**：暂缓（Phase 3）

## P2 — 低优先级改进

### ~~P2-01：指标贫乏——仅有 5 个均值~~ ✅ 已解决

- **来源**：代码审查
- **解决日期**：2026-06-04 (P19 Phase 2)
- **方案**：L1 描述统计现在包含均值、中位数、标准差、偏度、CVaR₀.₀₅、成功率、min、max。外加 ECDF/PMF 分布图、L2 随机占优矩阵、L3 p 值矩阵、L4 帕累托前沿。
- **相关提交**：`fb15090` `5432462`

### ~~P2-02：无持久化——结果随面板关闭丢失~~ ✅ 已解决

- **来源**：P19 问题诊断第 1/2 条
- **解决日期**：2026-06-04 (P19 Phase 1)
- **方案**：ResultStore.save_all() / load_all() JSON 持久化。数据管理面板提供导入/导出按钮。
- **相关提交**：`c4d628f` `c6d75c1`

### ~~P2-03：无法比较「相同策略 + 不同配置」~~ ✅ 已解决

- **来源**：P19 关键设计决策第 4 条
- **解决日期**：2026-06-04 (P19 Phase 1)
- **方案**：比较对象从「策略列表」扩展为「任意数据集组合」。ResultStore 中数据集为任意配置+策略的模拟结果。可比性检查面板自动识别 4 种比较模式（策略比较/配置对比/多重差异/变异分析）。
- **相关提交**：`c4d628f` `c6d75c1`

### P2-04：其他策略使用空参数 `sparams = {}`

- **来源**：代码审查（旧 ComparisonWorker）
- **严重度**：低等（已随旧代码移除而自然消除）
- **现状**：旧 `ComparisonWorker` 已随 `strategy_comparison_panel.py` 一同删除。新架构中用户在批量模拟面板分别运行各策略→自动存入 ResultStore→在比较分析面板加载比较。每个策略均使用各自配置的参数。
- **状态**：已消除（旧代码删除）

### P2-05：停止按钮的优雅性

- **来源**：代码审查（旧 ComparisonWorker）
- **严重度**：低等（已随旧代码移除而自然消除）
- **现状**：旧 `ComparisonWorker` 已删除。新架构中比较分析不再自己运行模拟，无需停止按钮。批量模拟面板的停止按钮独立运作。
- **状态**：已消除（旧代码删除）

### P2-06 (新增)：L5 偏好排序未实现

- **来源**：2026-06-04 会话审查
- **严重度**：低等（功能缺失）
- **现状**：P19 计划中的 L5「偏好排序」（用户通过 λ 滑块调节效率/风险权重，输出综合排名）尚未实现
- **计划方案**：P19 Phase 3
- **状态**：暂缓

### P2-07 (新增)：Almost Stochastic Dominance 未实现

- **来源**：2026-06-04 会话审查
- **严重度**：低等（学术完整度）
- **现状**：当前仅支持严格随机占优 (FSD/SSD/TSD)，不提供 Almost SD（允许 ε-松弛的近似占优）
- **计划方案**：P19 Phase 3
- **状态**：暂缓
