| # | 项目 | 暂缓理由 | 原标注 |

## 跨面板数据整合（设计文档 §5，未排期）

设计文档调研了 7 个分析面板的产出数据，按可比较程度分三级，但全部未排入实施计划：

### 直接可比的标量（可立即纳入 L1 描述统计表）

| 来源面板 | 统计量 | 现状 |
|---------|--------|------|
| process_analysis (BB) | 整体成败率 / per-pool success_rate | 未接入 |
| retreat | overall_failure_rate / per-pool failure_rate + pity 统计 | 未接入 |
| resource_search | min_draws | 未接入 |

### 需转换后可比（暂缓）

| 来源 | 可提取的标量 | 暂缓原因 |
|------|-------------|---------|
| process AB | P(success \| event) 条件概率 | 需事件特征标准化 |
| process BA | P(pattern \| success) / P(pattern \| failure) 诊断优势比 | 同上 |
| retreat | lp_curve 固定资源点的失败概率 / AUC | 需统一资源网格 |
| retreat | vulnerability_intervals max_ratio | 依赖 α 和分箱参数 |
| analysis | transition_matrix SS / FS 概率 | 需池对齐 + success_criterion 统一 |
| analysis | draws_vs_gdr Spearman ρ | 需决定使用哪个 GDR |
| retreat_search | Pareto 前沿点 (extra_resource, target_count) | 依赖脆弱性分析结果 |

### 不可比

- worst_impact 的后续池子模拟（强假设依赖用户自定义配置）
- strategy_panel 的具体 card_id 集合（配置特定）
- 个体轨迹 / 瀑布图（纯可视化产物，非聚合统计）
