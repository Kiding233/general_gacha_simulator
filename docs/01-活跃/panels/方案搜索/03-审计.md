## 一、设计 ↔ 实施对齐检查

| 设计项 | 实施状态 | 备注 |
|--------|---------|------|
| 3 种 QRadioButton 搜索模式 | ✅ | `mode_group` QButtonGroup（最少资源/最多目标卡/Pareto） |
| 起始池 QComboBox（含「(从头开始)」） | ✅ | `pool_combo`，默认选中第一项 |
| 灰显替代消失（setEnabled） | ✅ | `_on_mode_changed()` / `_on_pool_changed()` |
| 统一双列权重表 | ✅ | `weight_table`（抽取意愿 + 错失代价） |
| 搜索方向 QRadioButton | ✅ | `dir_forward` / `dir_backward`，最多目标卡模式默认前进，Pareto 模式默认后退 |
| Pareto 默认后退可切前进 | ✅ | `_on_mode_changed()` 设 `dir_backward.setChecked(True)`；方向按钮在 target + pareto 模式下启用 |
| 结果快照系统 | ✅ | `SnapshotManager`——内存上限 20，跨模式共存 |
| 3 个结果页（QStackedWidget） | ✅ | ResourceResultPage / TargetResultPage / ParetoResultPage |
| 自动命名 `{模式}·{方向} @{HH:MM}` | ✅ | `_on_finished()` 中生成 |
| 统一错误处理 | ✅ | `DetailedError` 包装 + `traceback.print_exc()` |
| 引擎统一停止 | ✅ | `PlanSearchWorker.stop()` → `engine.stop()` |
| cost_per_draw 归并 | ✅ | 模块级 `get_cost_per_draw()`（retreat_search.py:535-556） |
| 前进法退路点支持 | ✅ | `search_max_targets_forward()` 通过 `from_pool_id` 自动分叉 |
| 完整时间线 Pareto 支持 | ✅ | `PlanSearchEngine(from_pool_id=None).search_pareto()` |
