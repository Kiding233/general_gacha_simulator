## 一、三种搜索模式的理论基础

### 1.1 最少资源（二分搜索）

**问题形式化**：给定目标卡集合 T，求 `min r` 使得 `P(success | resource = r) ≥ θ`

**算法**：
1. 翻倍法找上界：从 `r_hi = 50 × cost_per_draw` 开始，逐次翻倍直到 P ≥ θ（最多 10 次，上限 51200 抽）
2. 二分收敛：`while r_hi − r_lo > epsilon`，每步 N 次模拟估计 P(r_mid)，收敛到精度 ε = cost_per_draw × precision_draws

**核心假设**：
- H1（单调性）：P(success | r) 随 r 单调递增——经济直觉支持（资源越多成功率越高），但算法不强制
- H2（无噪声平台）：P(success | r) 在目标阈值 θ 附近不是平坦的——若 P(r) 在 θ 附近几乎是常数（如 0.94~0.96），二分搜索可能因噪声振荡无法收敛

**已知局限**：
- 二分搜索作用于蒙特卡洛噪声目标函数——每次评效 P(r_mid) 是有限样本估计，含抽样误差
- 翻倍上界可能不足（理论最大需求超过 51200 抽）且无检测
- epsilon 精度设置缺乏统计依据（噪声 ~1/√N ≈ 4.5% 而精度可能设为 ~160）
- 多池成本不同时资源↔抽数换算为近似值（不影响搜索正确性，搜索在资源空间运行）

### 1.2 最多目标卡（前进法 / 后退法）

**问题形式化**：给定固定资源 R，求目标卡子集 S ⊆ T 最大化 |S| 使得 P(success | S, R) ≥ θ

**算法**：
- **前进法**：从空集开始，按抽取意愿（desire）降序添加，每次添加后评估成功率——若跌破阈值则回退到上一个有效状态
- **后退法**：从全量目标集开始，按错失代价（miss_cost）升序逐一移除，每次移除后评估成功率——若达到阈值则停止

**核心假设**：
- H3（贪心最优性）：按权重排序的贪心路径能找到最优子集——这要求权重与实际边际成功率单调相关，目前无理论保证
- H4（权重可解释性）：desire/miss_cost 权重反映了目标卡间的真实偏好顺序——权重仅影响排序，非基数效用

**已知局限**：
- 仅探索 N+1 个点（全量 + 逐一移除后的 N 个子集），搜索空间为 2^N
- 每步独立模拟不共享信息——N+1 次独立批量模拟，效率极低
- 权重缺乏基数解释（miss_cost 仅影响移除顺序，desire 仅影响添加顺序）

### 1.3 Pareto 前沿

**问题形式化**：在成功率 ≥ θ 的约束下，求目标集大小 k 与最少所需资源 r*(k) 之间的权衡曲线

**算法**（支持前进/后退两种方向）：
- **后退法**（默认）：从全量目标集开始 → 按 miss_cost 升序逐张移除，每步搜索满足阈值的最少资源
- **前进法**：从最有价值单卡开始 → 按**抽取意愿（desire）降序**逐张添加，每步搜索满足阈值的最少资源
- 两种方向产生相同的点集，仅迭代顺序不同

**核心假设**：
- H5（贪心前沿 = 真实前沿）：贪心移除顺序下的 k-子集是使 r*(k) 最小的 k-子集——无理论保证，可能遗漏非支配点
- H6（前沿单调性）：r*(k) 随 k 递减——直觉上目标越少所需资源越少，但贪心路径下不一定单调

**已知局限**：
- 「Pareto 最优」定义不严格——当前为贪心近似，非真正 Pareto 前沿
- 被支配点可能包含在内（无支配性过滤）
- N 张卡 × ~12 次二分迭代 = 计算量大，无缓存机制

## 二、起点差异的理论含义

### 2.1 完整时间线（from_pool_id=None）

- 所有池 + 完整排期，初始资源来自 ConfigStore
- 模拟从第一个池开始运行，策略可自由跨池调度资源
- 结果对应「从零开始规划」的场景

### 2.2 截断时间线（from_pool_id=str）

- 从指定池开始，时间线截断（该池之前的所有池被移除）
- 资源/保底计数器初值来自脆弱性分析（用户选择分位数或自定义）
- 通过 `RetreatConfigBuilder.build()` 构造截断 ConfigStore：
  - 保留 from_pool_id 及之后的池
  - 时间偏移（各池 start_day/end_day 减去 offset_day）
  - 注入保底计数器初值
- 结果对应「中途止损后重新规划」的场景——这是两类起点最核心的理论差异

## 四、假设违反的后果速查

| 假设 | 违反情景 | 后果 |
|------|---------|------|
| H1: 单调性 | 资源极低时的幸存者偏差（只有极幸运的模拟才能以低资源到达该池） | 左尾部 P(success\|r) 异常低，二分搜索下界偏高 |
| H2: 无噪声平台 | P(r) 在 [0.90, 0.96] 之间几乎平坦 | 二分搜索无法收敛——持续振荡，消耗大量迭代 |
| H3: 贪心最优性 | 最佳子集需要同时添加 A 和 B（单独添加都会降低成功率） | 前进法遗漏最优解，后退法同样 |
| H4: 权重可解释性 | 用户随意填写权重（如全部为 1.0） | 退化为按 ID 字母序移除——与真实偏好无关 |
| H5: 贪心前沿 | 某 k-子集的 r*(k) 比贪心路径上的 r*(k) 更小 | Pareto 图遗漏非支配点，前沿估计偏保守（高估资源需求） |
| H6: 前沿单调性 | 移除某卡后 r* 反而增加（因该卡有资源奖励） | Pareto 图出现非单调段，视觉上「倒退」 |
