## P1：P26 阶段二 —— SPRT 序贯采样

### P1-A：SPRT 决策模块

| 属性 | 内容 |
|------|------|
| 涉及文件 | `core/retreat_search.py`（新增 `SPRTDecisionMaker`） |
| 预计工作量 | 3-5 天 |
| 依赖 | 可复用 P3 的 Bootstrap CI 方法（BCa 或百分位法）计算 `ci_lower/ci_upper` |
| 风险 | 低——SPRT 不改变二分搜索整体结构，仅替换内层固定 N 采样为序贯决策 |
| 文献 | Waeber, Frazier & Henderson (2013); Bechhofer et al. (1995) |

**修复内容**：
1. 实现 `SPRTDecisionMaker`：给定 H₀: p ≤ θ−δ 和 H₁: p ≥ θ+δ，序贯采样直到做出决策或达到最大样本量
2. 替换 `search_min_resource()` 的内层固定 N=500 循环
3. 利用 Binomial SPRT 或 Bootstrap SPRT（复用 P3 的 CI 基础设施）
