## 二、实验 2b：二分搜索 k 敏感性

**核验主张**：核验报告 §2.3 的敏感性警示——「p ∈ [0.93, 0.97] 的比例在 40–80% 之间波动」

**设计**：固定 N=500, θ=0.95, r₀=10000，变化 k：

| k | 函数特征 | 预期 p 值分布 | 预期偏差 |
|----|---------|-------------|---------|
| 0.0001 | 极平坦——p 从 0.5 到 0.95 需 ~30000 单位 | 更多点接近阈值 | 可能更大 |
| 0.0002 | 平坦 | 更多点接近阈值 | 可能更大 |
| 0.0005 | 当前基线 | 55.6% 在 [0.93, 0.97] | 基线 −48 |
| 0.001 | 较陡 | 更少点接近阈值 | 可能更小 |
| 0.002 | 很陡——接近阶跃 | 极少点接近阈值 | 可能极小 |

**输出**：每个 k 的偏差均值、标准差、p 值分布、错误决策率

## 四、实验 6（新）：all_targets 饱和实证

**核验主张**：P26 §2.11 N5——「高成功率时二分搜索失去辨别力」

**设计**：构造 p(r) = 1 − ε(r)，其中 ε(r) 随 r 增大快速衰减（模拟「接近必然成功」场景）：

```
p_sat(r) = 1 - 0.5 * exp(-0.001 * (r - 5000))   # 在 r=10000 时 p≈0.993
```

运行二分搜索（θ=0.95, N=500, 200 次），观测：
- 最终估计的标准差是否异常小（< 正常噪声水平）→ 过早「收敛」
- 二分搜索是否在远低于真根的位置停止
- 估计值对初始上界的敏感度

**输出**：偏差、标准差、「过早停止」比例（定义为 |最终估计 − 真根| > 3×SE 的搜索比例）

## 五、实验 7（新）：SPRT 嵌入二分搜索

**核验主张**：核验报告 §2.3.2——「SPRT 在临界区域的表现可能不优于固定 N」

**设计**：将 Wald SPRT 嵌入 `noisy_binary_search` 的内层（替代固定 N=500）：

```
while not decided and n < max_samples:
    run one simulation
    n += 1
    decision = wald_sprt_decision(successes, n, p0, p1)
```

对比三种方案（同一 logistic 函数，θ=0.95，各 200 次）：
- **固定 N=500**：当前基线
- **SPRT δ=0.02**：推荐校准
- **SPRT δ=0.05**：激进校准

**输出**：
| 方案 | 偏差 | 标准差 | 平均总模拟次数 | 决策正确率 |
|------|------|--------|-------------|----------|
| 固定 N=500 | | | | |
| SPRT δ=0.02 | | | | |
| SPRT δ=0.05 | | | | |

**关键问题**：SPRT 的总模拟次数节省在嵌入二分搜索后是否仍然成立？还是会被临界区域的决策困境抵消？

## 六、实验 8（新）：lower_is_better 单调性方向反转

**核验主张**：P26 §2.7 N1——「更多资源可能导致更低的成功率」

**设计**：构造递减成功率函数（模拟 resource_consumed GDR）：

```
p_lower_better(r) = 1 - logistic_p(r)  # r 越大 → p 越低
```

即 p(0) ≈ 1.0, p(∞) ≈ 0.0——资源越多、成功率越低（因为「成功」定义为 resource_consumed ≤ threshold）。

二分搜索假设 `prob >= threshold → r_hi = r_mid`（单调递增），但对递减函数，正确决策应该是 `prob >= threshold → r_lo = r_mid`（方向相反）。

运行二分搜索 200 次：
- 观测是否收敛到错误方向（远离真根而非靠近）
- 测量「正确收敛」比例（应接近 0%——因为代码不做方向判断）

**输出**：收敛方向错误率 + 验证「翻转比较方向后正确收敛」

## 八、实验 14（新增）：多 GDR p(r) 曲线形状对比 ✅ 已完成

**核验主张**：阶跃函数是 `all_targets`（二值 GDR）的特例，还是抽卡系统的通用属性？

**方法**：
```
加载 config/ 默认配置（6 目标 SSR）
对每个 GDR（all_targets, target_achievement, resource_efficiency）：
  ① 二分搜索确定收敛点（用于确定扫描范围）
  ② 在收敛点附近做 20 点对数网格扫描（r ∈ [160, ~2×收敛点]）
  ③ 检测跳跃（Δp > 3×SE）
对比三种 GDR 的 p(r) 形状
```

**结果**（2026-06-10，耗时 3.9 min）：
- `all_targets`（二值，θ=1.0）：2 跳跃（5.9×SE, 99.3×SE），收敛 72363——阶跃确认
- `target_achievement`（连续，θ=0.8）：2 跳跃（7.8×SE, 73.1×SE），收敛 58887——**连续 GDR 也是阶跃**
- `resource_efficiency`（连续，θ=0.3）：0 跳跃，收敛失败（翻倍至 153,600,000 仍 p=0）——**N9 第三类退化确认**

**结论**：阶跃函数不是二值 GDR 的特例——连续 GDR 同样表现为阶跃（跳跃幅度 73×SE）。

## 九、实验 15（新增）：简化配置阶跃验证 ✅ 已完成

**核验主张**：std=0 可能来自 6 卡多池交互的「平均化」效应——需用 1–2 卡排除此替代解释。

**方法**：
```
1 卡配置：仅 limited_ssr_1 × 1（移除其余 5 张目标卡）
2 卡配置：limited_ssr_1 × 1 + limited_ssr_2 × 1
各 20 次独立二分搜索，N=200/步
```

**结果**（2026-06-10，耗时 14.5 min）：
- 1 卡：收敛 3760，std=0.0，20/20 成功
- 2 卡：收敛 16006，std=0.0，20/20 成功
- 与 6 卡（72188，std=0.0）一致——阶跃函数在最简配置上仍然成立

**结论**：多卡交互的「平均化」假说被排除——阶跃是抽卡系统 p(r) 的本质属性，根因是硬保底的确定性触发。
