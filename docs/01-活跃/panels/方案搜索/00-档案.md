## 一、业务问题

方案搜索面板解决的是抽卡规划中的核心最优化问题——**优化的不是成功率本身，而是在成功率 ≥ 阈值的约束条件下，通过搜索方法最大化某一目标参数**。

三种具体形式：

1. **最少资源搜索**：固定目标卡集合，在成功率 ≥ 阈值的约束下，**最小化额外资源投入**——翻倍法找上界 + 二分搜索沿资源维度收敛
2. **最多目标卡搜索**：固定资源预算，在成功率 ≥ 阈值的约束下，**最大化目标卡数量**——贪心前进法（按抽取意愿降序添加）或后退法（按错失代价升序移除）
3. **Pareto 前沿**：在成功率 ≥ 阈值的约束下，描绘资源投入与目标卡数量的**权衡边界**——枚举目标集大小，每步搜索满足阈值的最少资源

两种时间线起点：
- **完整时间线**：从第一个池开始，考虑全部排期
- **截断时间线**（退路点）：从脆弱性分析指定的池截断，模拟「中途止损」场景

## 二、模块定位

### P8 面板合并的产物

方案搜索面板（PlanSearchPanel）是 P8 面板合并计划的产物，将原有的三个独立面板整合为一个：

| 旧面板 | 旧 Tab 名 | 功能 | 合并后 |
|--------|----------|------|--------|
| `strategy_panel.py` | 「最多目标卡」 | 前进/后退法搜索目标卡组合 | PlanSearchPanel「最多目标卡」模式 |
| `resource_search_panel.py` | 「最少资源」 | 二分搜索最少所需资源 | PlanSearchPanel「最少资源」模式 |
| `retreat_search_panel.py` | 「退路分析→方案搜索」子Tab | 退路点三类搜索 | PlanSearchPanel「Pareto」模式 + 截断时间线 |

合并动机：
1. 三个面板底层做同一件事——迭代模拟→搜索最优方案，差异仅为参数
2. 消除「最多目标卡」命名的混淆
3. 补全功能缺口：完整时间线 Pareto 前沿 + 退路点前进法
4. 消除 GUI 中唯一的子 Tab 嵌套（Tab 数从 11→10）

## 四、技术实现

### 4.1 数据流

```
ConfigStore + 脆弱性分析结果
      │
      ▼ PlanSearchWorker.run()
      │
      ├─ PlanSearchEngine(config_store, from_pool_id=None|str, ...)
      │     │
      │     ├─ _build_env() → SimulationEnv（完整或截断）
      │     ├─ _simulate_with_resource() → run_batch_parallel() → 成功率
      │     │
      │     ├─ search_min_resource()     → 翻倍上界 + 二分收敛
      │     ├─ search_max_targets()      → 贪心后退法
      │     ├─ search_max_targets_forward() → 贪心前进法
      │     └─ search_pareto()           → 枚举 + 二分
      │
      ▼ PlanSearchResult（统一 dataclass）
      │
      ▼ SnapshotManager.push() → 结果区显示
```

### 4.2 关键数据结构

| 数据结构 | 位置 | 说明 |
|----------|------|------|
| `RetreatSearchPoint` | `core/retreat_search.py:7-10` | 单个搜索点（额外资源/目标规格/成功率） |
| `PlanSearchResult` | `core/retreat_search.py:14-59` | 统一结果 dataclass，兼容旧 `RetreatSearchResult` |
| `PlanSearchEngine` | `core/retreat_search.py:66-524` | 统一搜索引擎（from_pool_id=None=完整时间线） |
| `SearchSnapshot` | `gui/plan_search_panel.py:26-29` | 快照条目（名称 + 结果 + 可选图表 JSON） |
| `SnapshotManager` | `gui/plan_search_panel.py:32-82` | 内存快照管理器（上限 20，跨模式共存） |

### 4.3 四个搜索算法（五种搜索路径）

| 方法 | 行号 | 算法 | 核心缺陷（详见 P26） |
|------|------|------|---------------------|
| `search_min_resource()` | 168-259 | 翻倍法找上界 + 二分收敛 | 噪声振荡 + 收敛无保证；epsilon 缺统计依据 |
| `search_max_targets()` | 301-368 | 贪心后退法——按 miss_cost 升序逐一移除 | 仅探索 N+1 个点；权重缺基数解释 |
| `search_max_targets_forward()` | 370-462 | 贪心前进法——按 desire 降序逐一添加 | 同上，方向相反 |
| `search_pareto(direction='backward')` | 512-598 | 从全量目标集开始，逐张移除 + 二分搜索最少资源 | 非真正 Pareto，仅贪心近似 |
| `search_pareto(direction='forward')` | 512-598 | 从最有价值单卡开始，逐张添加 + 二分搜索最少资源 | 同上 |

## 五、GUI 架构

### PlanSearchPanel（`gui/plan_search_panel.py`，~1280 行）

**布局**：QSplitter 左右分栏
- **左栏**（可滚动，360-420px）：5 个 QGroupBox（搜索模式 → 起始状态 → 权重配置 → 搜索参数 → 按钮+进度）
- **右栏**：快照导航栏 + QStackedWidget（3 个结果页）

**灰显策略**（v8 设计）：
- 最少资源/Pareto 模式：权重表灰显（不需要），二分参数可用
- 最多目标卡模式：权重表可用，二分参数灰显（不需要）
- 从头开始模式：基准资源/保底水位灰显

**快照系统**：
- 内存存储，上限 20 条
- 跨模式共存（同一会话可保存最少资源+最多目标卡+Pareto 快照）
- 自动命名：`{模式}·{方向} @{HH:MM}`
- 导航：prev/next 按钮 + 下拉列表 + 删除

| 类 | 基类 | 行号 | 角色 |
|----|------|------|------|
| `PlanSearchWorker` | `QThread` | 89-189 | 工作线程——创建 PlanSearchEngine，执行搜索 |
| `ResourceResultPage` | `QWidget` | 196-240 | 最少资源结果页（摘要 + 图表 + 步骤表） |
| `TargetResultPage` | `QWidget` | 243-302 | 最多目标卡结果页（摘要 + 图表 + 步骤表） |
| `ParetoResultPage` | `QWidget` | 305-348 | Pareto 结果页（摘要 + 图表 + 明细表） |
| `PlanSearchPanel` | `QWidget` | 355-949 | 主面板 |
