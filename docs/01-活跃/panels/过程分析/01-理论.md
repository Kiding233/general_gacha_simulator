## 一、核心假设

### 1.1 事件类型的互斥完备性

7 种事件类型（`pity_hit`、`early_hit`、`miss`、`skip`、`ignore`、`exchange`、`no_exchange`）被设计为在每个池上互斥且完备——任何池在任何一次模拟中必落入唯一的事件类型。

**判定优先级**（按代码逻辑）：池类型 > 抽卡次数 > 目标卡命中 > 保底触发

1. 资源池 → `resource_draw` / `resource_ignore`（不在 7 种事件中，跳过）
2. 兑换池 → `exchange` / `no_exchange`
3. 抽卡池 → 0 抽 → `skip` / `ignore`（由 `_resolve_skip_ignore()` 判定）
4. 抽卡池 → >0 抽 + 命中 + 保底 → `pity_hit`
5. 抽卡池 → >0 抽 + 命中 + 非保底 → `early_hit`
6. 抽卡池 → >0 抽 + 未命中 → `miss`

**完备性验证**：以上 6 条覆盖全部（池类型 × 抽卡次数 × 命中 × 保底）的组合空间，无遗漏。

### 1.2 池成败的独立判定

每个池的成功/失败通过池级 GDR 阈值独立判定。假设：

1. **池间独立**：一个池的成败不影响另一个池的判定（在 B 维度）
2. **阈值一致性**：所有池使用相同的 GDR 指标（key）和阈值
3. **判定函数**：`SuccessChecker.is_success(pool_compact)` —— 将单池数据包装为伪 compact 并传入统一的 `SuccessChecker`

如果池之间存在真实的相关性（如资源池提供金币、抽卡池消费金币），这一「独立判定」假设不反映因果依赖——但交叉统计（AB/BA）正是为了揭示这种依赖模式。

### 1.3 交叉统计的合理性

四种交叉统计（AA/BB/AB/BA）来自 2×2 列联表分析的标准框架：

- **AA（事件→事件）**：联合分布 `P(E₁=e₁, E₂=e₂, ...)`，回答「哪些事件组合最常见？」
- **BB（成败→成败）**：联合分布 `P(S₁=s₁, S₂=s₂, ...)`，回答「哪些池的成败联动？」
- **AB（事件→成败）**：条件分布 `P(Sⱼ=sⱼ | Eᵢ=eᵢ)`，回答「给定某池的事件类型，该池成功的概率？」
- **BA（成败→事件）**：条件分布 `P(Eᵢ=eᵢ | Sⱼ=sⱼ)`，回答「给定某池的成败，事件分布如何？」

理论依据：列联表分析（Pearson, 1904），在此维度灾难通过事件组合模式的离散化缓解。

### 1.4 Laplace 平滑的无信息先验

在条件概率估计中，对零计数单元格添加 Laplace 平滑（+1 伪计数）：

```
P̂(X=x | Y=y) = (count(x, y) + 1) / (count(y) + K)
```

其中 K=事件类型数量。此平滑等价于 Dirichlet(1, 1, ..., 1) 先验下的贝叶斯后验均值（Jaynes, 2003, *Probability Theory* §18.4.2）。

## 二、已知局限性

### 2.1 维度灾难

随着池子数量增加，事件组合空间指数增长（K 个池 × 7 种事件类型 → 理论上有 7^K 种组合）。实际通过 5 种事件组合模式（sequence/set/count_set/custom）降维，但对 5 个以上池的场景，即使 `count_set` 模式的组合数也可能超过有效样本量。

### 2.2 low_sample 阈值的校准

`low_sample` 标记的阈值（默认 30）基于经验法则（n>30 近似正态），但对：
- 高度偏斜的概率（p<0.1 或 p>0.9）：需要更大样本量
- Wilson 区间的小样本覆盖率：虽然优于 Wald，但在 n<10 时仍有不足

### 2.3 事件类型「miss」的歧义

「miss」表示「抽了但未获得目标卡」——但实际原因可能包括：
- 保底概率不够高（软保底未生效或生效但未触发）
- 运气差（即使在硬保底计数器顶部，也可能因分布权重未命中目标卡）
- 策略选择（在特定池只抽有限次数）

当前实现不区分这些子原因，「miss」是一个聚合类别。这限制了事件→成败分析的因果解释力。
