# 最差影响 — 理论

> 建档日期：2026-06-02 | 版本：v1

## 一、核心假设

本分析建立在以下假设之上，每一条都影响结果的解释方式：

### A1：初始资源分布的经验分位数替代真实条件分位数

当前实现从历史批量模拟结果中提取条件资源分布的 α 分位数（`EmpiricalDistribution.quantile()`），以此作为「最差情况资源预算」的估计。

- **成立条件**：历史模拟的批量大小 N 足够大（建议 N >= 1000），使得 α-分位数附近的经验分布能合理近似真实分布
- **脆弱环节**：
  - 当 α 很小（如 0.01）且 N=1000 时，α-分位数对应的样本量仅约 10 个，估计波动大
  - 若条件资源分布为退化分布（所有值相同），EVT 外推无意义——代码中已对资源类 GDR + success 条件做跳过 EVT 处理（`_skip_evt` 标记）
  - EVT（极值理论）的 GPD 尾部外推可能将 α-分位数外推到实际样本范围之外（当 α 小且样本不足时）

### A2：条件分布分位数 = 未来模拟初始预算（逻辑跳跃）

步骤 1 提取的 q_α 被直接用作步骤 2 新模拟的初始资源。这隐含假设：

> 「在过去失败 + 剩余资源为 q_α」≈「从头开始 + 预算为 q_α」

但两者不等价：
- 真实失败路径中，玩家可能已累计保底计数器（靠近下一次 SSR 触发点）
- 真实失败路径中，兑换点数可能已被消耗

新模拟从清空历史开始，条件相对有利。这意味着连续成功池数可能被**高估**。详见缺陷 27（`P31 最差影响分析剩余缺陷改进计划.md` 第 2.1 节）。

### A3：虚拟池同质化

所有 99 个虚拟池共享相同的参数（卡牌分布、单抽消耗、持续时间），仅 featured 卡 ID 不同。实际游戏中：
- 池子成本可能不同（如角色池 160 钻/抽，武器池 100 钻/抽）
- 基础出率可能不同（如周年庆池 up 概率更高）
- 持续时间可能不同（如活动池 21 天 vs 常驻池长期开放）

同质化假设系统性消除了「连续高成本池」的失败模式。详见缺陷 22/28（同上计划第 2.2 节）。

### A4：无资源收入假设

虚拟池模拟中 `resource_gain=None`——假设玩家无任何后续资源收入（每日任务、活动奖励、月卡等）。这确保了下界（最差情况下保守估计），但绝对连续成功池数被**低估**。

### A5：MAX_POOLS = 99 截断

成功次数分布在 K=99 处截断。对于初始资源极为充裕（如 >= 10 倍典型池消耗）的场景，expected_pools 被低估。当前截断值为硬编码常数 99。

### A6：保底引擎通配符匹配

保底规则通过 `fnmatch.fnmatch(pid, pdef.pools)` 匹配虚拟池 ID（`_worst_impact_pool_0` 等）。P17 已修复匹配失败的静默 bug，但确保虚拟池保底行为与真实池一致依赖通配符模式 `*` 覆盖新增 ID。

### A7：最后一个池作为「典型池」

`_prepare_pool_info()` 默认使用配置中最后一个启用的池（`pool_entries[-1]`）作为虚拟池模板。这意味着假设「下一个池与最近一个已配置池相似」——在已配置池序列有意义代表未来时成立。

## 二、已知局限性

### L1：无法模拟跨池资源反馈

真实游戏中的资源奖励（如抽到 SSR 返还碎片、兑换商店的连锁反应）在多池连抽中可能有乘数效应。当前虚拟池的奖励分布复制自模板池，但各池独立——池 A 抽到的资源奖励不会影响池 B 的策略决策。

### L2：单一资源维度

当前只跟踪一种资源（`draw_resource`，即抽卡券/钻石）。多资源经济体系（如「付费钻+免费钻+限定券+通用券」）被简化为单资源，忽略了资源间不可替代性带来的约束。

### L3：「连续成功」的二元判定

连续成功判定为「是否获得该池的 featured 卡（数量 >= 1）」。这忽略：

- 多 up 池：只获得一个 up 但没获得另一个算成功吗？当前判定为只要获得 featured 卡就算成功
- 部分成功：获得 0.5 个目标（如 50% 进度）被视为失败
- 溢出价值：同一池抽出多张 featured 卡的边际效用递减被忽略

### L4：单线程嵌套模拟

虚拟池模拟在面板的后台线程中运行，且 `max_workers=1`（串行）。对于 num_simulations=1000 的 99 池场景，单线程运行时间可能达数分钟，阻塞 UI 直到完成（虽然通过 QThread 隔离，不会冻结界面，但用户仍需等待）。

### L5：保底计数器继承

虚拟池模拟中保底计数器初始化自 `store.pity.counter_init`，而非从原批量模拟的失败路径中提取条件计数器分布。这意味着模拟起始的保底状态只反映「配置的初始计数器」，而非「坏运气历史后的真实计数器状态」。

### L6：兑换机制的简化处理

原始模拟中的兑换（exchange）行为在虚拟池模拟中被重置为初始状态——可兑换次数重置为 `initial_exchange`。在真实失败场景中，兑换可能已被消耗。

## 三、概率模型

### 3.1 条件资源分布模型

设批量模拟结果集合为 `R = {r_1, ..., r_N}`，每个 `r_i` 包含最终资源 `w_i`。

按成功判定函数 `S: r → {0, 1}` 分类：

- 成功组：`W_s = {w_i | S(r_i) = 1}`
- 失败组：`W_f = {w_i | S(r_i) = 0}`

条件 α-分位数：`q_α = F^{-1}_cond(α)`，其中 F_cond 是目标条件的经验 CDF。

分位数计算使用插值（等价于 R 语言的 type=7）：
```
idx = α × (n - 1)
lo = floor(idx), hi = ceil(idx)
若 lo == hi → sorted[lo]
否则 → sorted[lo] × (1 - frac) + sorted[hi] × frac
```

### 3.2 保底覆盖模型

大保底触发消耗 = 保底结束计数 × 单抽成本

覆盖倍数 = 最差资源 / 保底消耗 = `worst_resource / (pity_end_at × cost_per_draw)`

覆盖倍数 < 1 表示在最差情况下甚至不够触发一次硬保底。

### 3.3 连续成功模型

`X = 连续成功池数 = max{k | 池 0..k-1 均成功 }`

`X` 的支撑集为 {0, 1, ..., 98, 99}，被 MAX_POOLS 截断。

期望：`E[X] = Σ k × P(X = k)`

条件概率：`P(X >= k)` 表示「至少连续成功 k 个池」的概率。

## 四、参考外部文献/理论

无外部论文直接引用。方法论受到以下领域的启发：

- 极值理论（EVT）：尾部风险估计（`EmpiricalDistribution.quantile` 中的 GPD 外推）
- 保险精算学：破产概率分析 —— 给定初始准备金和随机赔付序列，评估资金枯竭概率
- 库存理论：连续盘点库存模型 —— 初始库存 + 随机需求 → 缺货概率
