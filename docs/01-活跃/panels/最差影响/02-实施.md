## 五、模拟调用路径

```
WorstImpactAnalyzer.analyze()
  └─ prepare_simulation_config()
       └─ 返回 dict 包含:
            pools: List[Pool]              (99 个)
            schedule_mgr: PoolScheduleManager
            end_time: float                (99 * pool_duration)
            pity_engine: PityEngine | None
            resource_gain: None            (无资源收入)
            pity_state_init: dict          (初始保底计数器)
            card_defs: List[dict]          (featured + standard SSR)
            target_specs: dict             ({f'_wi_featured_{i}': 1 for all i})
            initial_resources: dict        ({draw_resource: worst_resource})
            ssr_ids: Set[str]              (所有 SSR ID)
            stop_condition: ConsecutivePoolTargetCondition
            pool_targets: dict             ({pool_id: featured_id})

  └─ SimulationEnvBuilder.from_dict(sim_config)
       └─ 构建 SimulationEnv

  └─ run_batch_parallel(env, ..., strategy_name='draw_target', strategy_params={'pool_id': ''}, on_result=collector.on_result)
       └─ 串行或并行运行 num_simulations 次
       └─ 每次: DrawTargetStrategy 无条件从当前池抽卡直到资源不足或停止条件触发
       └─ 停止条件: ConsecutivePoolTargetCondition — 检测连续获取 featured 卡的池数
```

### `ConsecutivePoolTargetCondition` 的判定逻辑

对于池序列 `(pool_0, pool_1, ..., pool_98)`（按时间排序），每个池有一个目标卡（`pool_targets[pid]`）。停止条件在每次抽卡后检查：

- 当前池的目标卡是否已获得（>= 1 张）
- 如果获得：继续下一个池
- 如果资源耗尽（无法支付任一池的单抽）且当前池目标未获得：触发停止

这意味着「连续成功 K 个池」定义为前 K 个池每个都至少获得 1 张目标卡，且第 K+1 个池未能获得（资源耗尽或其他原因）。
