## 二、SimulationThread.run() 流程

```
1. 进度初始化: progress(0, N) + status_update("正在构建模拟环境…")
2. SimulationEnvBuilder.from_config_store(config_store)
   → env (SimulationEnv)
3. 提取 target_ids, ssr_ids, pool_end_times, gdr_context
4. 设置 env.return_compact = False  （P0: worker 内并行提取）
5. 构建 _batched_progress 闭包  （P1: 每 N/50 次 emit 一次）
6. run_batch_parallel(env, target_specs, N, max_workers, seed, ...)
   → batch_result (BatchResult)
7. no_draw 基线计算（max_workers=1, strategy_name='no_draw'）
   → no_draw_resource, no_draw_pool_resources
8. 从 batch_result.extraction 解包 worker 内提取数据
   → heatmap_bins, result_bundle 字典
9. finished.emit(result_bundle)
```

## 三、GachaPanel UI 布局

| 控件 | 类型 | 说明 |
|------|------|------|
| 模拟次数 | QSpinBox | 默认 1000，范围 1-999999 |
| 并行进程数 | QSpinBox | 默认 4，范围 1-16（超过 CPU 核心数无益） |
| 随机种子 | QSpinBox | 默认 42，可固定以复现结果 |
| 进度条 | QProgressBar | 0-N 线性进度 |
| 状态标签 | QLabel | 显示当前阶段文本（构建环境/模拟中/计算基线） |
| 开始按钮 | QPushButton | 触发 `_start_simulation()` |
| 日志输出 | QTextEdit | 只读，记录各阶段耗时和统计摘要 |
