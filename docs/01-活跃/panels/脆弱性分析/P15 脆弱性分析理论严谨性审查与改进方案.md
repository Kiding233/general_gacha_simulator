## 一、审查动机与范围

本计划源于对 `core/vulnerability.py` 中脆弱性分析的理论审查，重点回答两个问题：

1. **脆弱区间检测是否遗漏了失败率 100% 的区域？**
2. **整个分析管线的理论严谨性如何？**

审查通过逐环节代码分析 + 文献对照的方式进行。本计划不涉及 GUI、配置系统、并行架构等外围模块。

## 二、问题发现

### 2.1 严重问题（影响结论正确性）

#### 问题 A：r_grid 边界硬限制，无法检测数据范围外的脆弱区域

**位置**：[vulnerability.py:304-306](gacha_simulator/core/vulnerability.py#L304-L306)

```python
r_min = float(arr_all.min())
r_max = float(arr_all.max())
r_grid = np.linspace(r_min, r_max, num_curve_points)
```

LLR 曲线仅在 `[r_min, r_max]`（该池观测到的资源最小值~最大值）上估计。脆弱区间通过 `lp_curve > alpha` 检测，只能在 r_grid 上进行。

**后果**：如果观测数据的资源范围是 [200, 5000]，但真实失败概率在资源 < 100 时为 100%，脆弱区间的**左边界不可能低于 200**。极端情况下（如所有到达该池的模拟资源剩余都 > 500），即使数据范围内失败率都不超过 α，也会报告「无脆弱区间」——尽管在资源接近 0 时失败是必然的。

该问题被 `_make_vulnerability_interval_from_curve` 进一步固化（[line 413-416](gacha_simulator/core/vulnerability.py#L413-L416)）：

```python
data_min = min(all_values) if all_values else lower
data_max = max(all_values) if all_values else upper
lower = max(lower, data_min)    # 脆弱区间下界不低过数据最小值
upper = min(upper, data_max)    # 脆弱区间上界不高过数据最大值
```

这显式阻止了区间涵盖数据支持范围外的资源水平。

#### 问题 B：局部线性回归（LLR）在左边界处的系统性偏差

LLR 对边界偏差有一阶修正（优于 NW 核回归的零阶），但并未完全消除[^Fan1992]。在资源分布左边界（低资源区），核函数右侧有数据而左侧没有，估计值向数据内部收缩。

**后果**：P(失败 | 资源=r) 在低资源区域被系统性低估，脆弱区间左边界偏保守（偏高）。问题 A 和问题 B 叠加后，低资源区域的脆弱性被双重遮掩。

### 2.2 中等问题（影响统计严谨性）

#### 问题 C：二元响应变量使用线性回归而非逻辑回归

当前使用局部**线性**回归拟合二元响应变量（failure ∈ {0, 1}），存在三个子问题：

**C1. 预测值可超出 [0, 1]**：[line 121-124](gacha_simulator/core/vulnerability.py#L121-L124) 的 `np.clip(pred, 0.0, 1.0)` 是事后补救。裁剪后边界处估计有偏——真实概率接近 0 或 1 时曲线被压平。

**C2. 异方差性被忽略**：二元变量的方差为 `P(1-P)`，在 P≈0.5 时最大。LLR 拟合阶段假设等方差，虽然方差估计阶段（`_kernel_smooth_variance`）是非参数的，但点估计的效率损失已发生。

**C3. 渐近正态置信带不够准确**：[line 644-648](gacha_simulator/core/vulnerability.py#L644-L648) 的 `±1.96 * sqrt(var)` 置信带基于渐近正态性假设，在极端概率（接近 0 或 1）和样本量较小处不再可靠。

Signorini & Jones (2004) 的系统比较表明，**局部线性逻辑回归（local linear logistic）是单带宽二元回归的最佳选择**[^SignoriniJones2004]。Loader (1999) 的专著第 4 章给出了局部似然（local likelihood）的完整理论框架[^Loader1999]——通过最大化局部加权对数似然代替局部加权最小二乘，自然约束预测值在 [0,1]，正确处理异方差性。

#### 问题 D：带宽选择使用了密度估计而非回归的最优公式

当前 Silverman 规则（[line 117](gacha_simulator/core/vulnerability.py#L117)）：

```python
h = 1.06 * sigma * (n ** (-0.2))
```

这是为**核密度估计**推导的规则[^Silverman1986]，假设数据来自正态分布以估计最优带宽中的未知泛函 ∫(f'')²。对于回归问题，最优带宽涉及**回归函数曲率**而非密度函数曲率[^FanGijbels1996]：

```
h_opt ∝ n^(-1/5) · [∫ σ²(x)/f(x) dx / ∫ (m''(x))² f(x) dx]^(1/5)
```

Silverman 规则等价于假设 m''(x) 为常数——即回归函数是全局二次的。当 P(失败|r) 存在急剧转折（从「资源充裕一定成功」到「资源不足一定失败」的过渡带），带宽可能偏宽，转折被过度平滑。

Fan & Gijbels (1995) 提出了针对回归的数据驱动带宽选择方法（基于残差平方准则 + 偏差-方差逼近）[^FanGijbels1995]，Calonico, Cattaneo & Farrell (2018) 进一步提出了稳健偏差校正（RBC）置信区间，比传统欠平滑方法收敛更快[^CalonicoCattaneoFarrell2018]。

代码中仅有下界保护 `max(h, span/n * 2)`，无上界约束。

#### 问题 E：脆弱区间检测缺乏多重性校正

当前在 200 个格点上同时做 `lp_curve > alpha` 的阈值判断，这是一个**多重检验**问题。即使零假设（整个定义域上 P(失败|r) ≤ α）为真，也可能因随机波动而产生伪「脆弱区间」。

改进方向：
- 使用 **同时置信带（simultaneous confidence band）**而非逐点置信区间做检测——要求整条下置信界在某区域超过 α，而非仅点估计超过 α
- Härdle & Mammen (1991) 的 **Wild Bootstrap** 可构造同时置信带[^HardleMammen1991]
- Wasserman (2006) 第 5 章讨论了基于高斯过程极值分布的置信带构造[^Wasserman2006]

#### 问题 F：无最小区间宽度（persistence）过滤

一个仅 1~2 格点宽的 P(失败|r) 尖峰也会被当作脆弱区间。这在核平滑变异性下很可能只是噪声伪影。

改进方向：增加 `min_width` 参数（至少 3~5 个连续格点），或使用基于同时置信带的检测（下置信界持续超过 α 的区域）。

### 2.3 低等问题（改进空间但非紧急）

#### 问题 G：全局等带宽不适应局部数据密度变化

低资源区通常数据稀疏（失败罕见 → 到达该池的失败样本少）。全局等带宽在稀疏区方差过大，在密集区偏差过大。

改进方向：Fan & Gijbels (1995) 的变带宽方法[^FanGijbels1995]，或 LOESS 风格的最近邻比例带宽（tri-cube 权重 + `span` 参数）。

#### 问题 H：直方图分箱规则的边界效应

`freedman_diaconis_bins`（[distribution.py:253-266](gacha_simulator/core/distribution.py#L253-L266)）使用 IQR 估计数据离散度[^FreedmanDiaconis1981]。对于脆弱性分析中重尾的资源分布，IQR 比标准差更稳健（不受极端资源值影响），总体是合适的选择。但当资源分布严重偏斜时（长尾向右），直方图在低资源区的分辨率可能不足。

改进方向：对偏斜数据考虑自适应分箱或 Scott 规则（正态数据两者几乎一致，偏斜时 FD 更优但仍有局限）。

#### 问题 I：因果性声明的隐式假设

分析估计 P(失败 | 资源剩余=r) 是关联关系，非因果关系。低资源和高失败率可能共享同一原因（运气差）。在模拟数据背景下可接受（用户追问「如果资源落到 X，有多大把握成功」），但应在 UI 中明确注记。

## 四、改进方案

### 4.1 阶段一：核心修正（问题 A、B、C）——预计 3~5 天

#### 修改 1：扩展 r_grid 覆盖 0 到 r_max

**文件**：`gacha_simulator/core/vulnerability.py`

将 r_grid 从 `[r_min, r_max]` 改为 `[0, r_max]`。对于 r_grid 上低于 r_min 的区域，LLR 会退化（外推），采用以下策略：

- **方案 A（保守）**：r < r_min 时，用数据范围左端的核加权平均作为 P(失败|r) 的下界估计；若左端失败比例高（如 > 0.8），将脆弱区间下界扩展到 0
- **方案 B（推荐）**：增加单调性约束——即假设 P(失败|r) 在 r 下降时非递减。对于 r < r_min，将 P(失败|r) 设为 max(r_min 处 LLR 估计值, 数据范围内左尾失败率)

**理由**：资源为 0 时失败在逻辑上必然（无法继续抽卡），且经济学直觉要求资源越少失败概率越高。

#### 修改 2：LLR 边界偏差缓解

**文件**：`gacha_simulator/core/vulnerability.py`

不改变估计方法，但在脆弱区间边界上加上基于核权重非对称性的调整量。具体：在左边界格点，计算核权重在左侧和右侧的比例，若左侧比例 < 阈值（如 0.3），则额外将左边界向左扩展一个带宽的距离（不超过 0）。

**理由**：在不替换整个回归方法的前提下减少左边界偏差。

#### 修改 3：用局部线性逻辑回归替代 LLR

**文件**：`gacha_simulator/core/vulnerability.py`

新增 `_fit_local_logistic()` 函数，用迭代加权最小二乘（IRLS）实现局部线性逻辑回归：

```
算法：对 r_grid 上每个评估点 x₀:
  1. 初始化 β = (logit(总体失败率), 0)
  2. 迭代直到收敛（通常 5~10 步）：
     a. 计算 p_i = logit⁻¹(β₀ + β₁(x_i - x₀))
     b. 计算权重 w_i = K((x_i - x₀)/h) · p_i · (1-p_i)
     c. 加权最小二乘更新 β
  3. 返回 p = logit⁻¹(β₀)
```

保留当前 LLR 作为回退路径（IRLS 不收敛时使用）。

**IQR 自适应带宽**：使用 `h = 1.06 · min(σ, IQR/1.34) · n^(-1/5)` 替换纯 σ 版本，增强离群值鲁棒性。

**理由**：Signorini & Jones (2004) 的经验比较表明局部逻辑回归在二元响应上一致优于局部线性回归；Loader (1999) 第 4 章提供了完整的理论框架。

**风险**：IRLS 收敛性——二元响应的局部似然通常是良态的，但极端稀疏区可能失败，需要回退路径。

### 4.2 阶段二：统计严谨性提升（问题 D、E、F）——预计 2~4 天

#### 修改 4：残差平方准则（RSC）带宽选择

**文件**：`gacha_simulator/core/vulnerability.py`

实现 Fan & Gijbels (1995) 的残差平方准则带宽选择：

```
RSC(h) = n⁻¹ Σᵢ (Yᵢ - m̂₋ᵢ(Xᵢ; h))² · (1 + (nh)⁻¹ · 2V)
```

其中 `m̂₋ᵢ` 是留一估计，`V` 是等效核的方差项。对 h 在 `[0.5·h_silverman, 2·h_silverman]` 范围内格点搜索最小化 RSC(h)。

**风险**：RSC 计算量 O(n²)（留一估计），n 较大（如 > 5000）时需使用广义交叉验证（GCV）近似。

#### 修改 5：基于 Wild Bootstrap 同时置信带的区间检测

**文件**：`gacha_simulator/core/vulnerability.py`

使用 Härdle & Mammen (1991) 的 Wild Bootstrap 构造同时置信带：

```
1. 估计原始 LLR 曲线 m̂(x)
2. 计算残差 ε̂ᵢ = Yᵢ - m̂(Xᵢ)
3. B 次 bootstrap（推荐 B=999）：
   a. 生成 ε̂*ᵢ = ε̂ᵢ · ηᵢ（ηᵢ 取 ±1 各 50% 或 N(0,1)）
   b. 计算 Y*ᵢ = m̂(Xᵢ) + ε̂*ᵢ
   c. 估计 LLR 曲线 m̂*(x)
   d. 计算 supₓ |m̂*(x) - m̂(x)| / se(m̂*(x))
4. 取 B 次模拟的 (1-α) 分位数作为同时置信带的临界值
5. 脆弱区间 = 下置信界 > α 的区域
```

**理由**：解决了多重检验问题——同时置信带保证了 1-α 的覆盖概率意义下不会有伪区间；Wild Bootstrap 不依赖方差齐性假设。

**性能**：B=999 × n=200 格点 × IRLS 迭代 ≈ 可接受（通常在 1~2 秒内），可加 `fast=True` 参数使用渐近近似跳过多重 bootstrap。

#### 修改 6：增加 persistence 过滤

**文件**：`gacha_simulator/core/vulnerability.py`

新增参数 `min_interval_width: Optional[float] = None`。若设置，宽度小于该值的脆弱区间被过滤。默认行为：至少 3 个连续格点才构成区间（`min_points=3`）。

**理由**：消除格点级别的噪声伪影，提升区间结果的可靠性。

### 4.3 阶段三：完善与优化（问题 G、H、I）——预计 1~2 天

#### 修改 7：最近邻自适应带宽作为可选项

**文件**：`gacha_simulator/core/vulnerability.py`

新增参数 `bandwidth_method: str = 'silverman'`，可选 `'silverman'`、`'rsc'`（修改 4）、`'knn'`（最近邻）。

k-NN 带宽：`h_knn(x₀) = |x₀ - x_{(k)}|`，其中 `x_{(k)}` 是离 x₀ 第 k 近的观测点，`k = α·n`（α 默认 0.3，即可调 span 参数）。

**默认保持 Silverman**（简单、快速），k-NN 作为高级选项。

## 七、与其它计划的关系

- **无依赖**：本计划独立于 P11（Plotly 迁移）、P8（面板合并）等计划
- **被依赖**：无——其他计划不依赖脆弱性分析的具体实现
- **与 P3（Bootstrap 稳定性分析）的关系**：本计划的 Wild Bootstrap 用于同时置信带构造（推断），P3 的 Bootstrap 用于估计量的稳定性评估（误差）。两者互补但不重叠——P3 是对「P(失败|r) 估计值本身的变异」做 Bootstrap，本计划修改 5 是对「整条曲线同时置信带」做 Bootstrap
- **与 P4（自适应模拟）的关系**：无直接关系。P4 的自适应模拟可产生更多低资源区域的样本，间接改善本分析的边界估计质量

## 八、验收标准

### 阶段一验收

1. 构造测试用例：资源 < 300 时 100% 失败，资源 ≥ 300 时 0% 失败，数据范围 [300, 1000]。当前算法检测不到脆弱区间（因为 r_min=300 处失败率才从 0 开始上升），改进后应检测到 `[0, 300)` 为脆弱区间
2. 局部逻辑回归估计值始终在 [0, 1] 内（无需裁剪）
3. 确定性测试对比 LLR 和逻辑回归在边界样本上的估计偏差

### 阶段二验收

4. RSC 带宽选择给出与 Silverman 不同的 h，且在新测试用例上 MISE 更低
5. Wild Bootstrap 同时置信带覆盖概率 ≥ 名义水平（通过已知 DGP 的模拟验证）
6. persistence=3 过滤掉单格点宽的伪区间

### 阶段三验收

7. KNN 带宽在数据稀疏区给出比全局带宽更宽的 h（方差缩减）
8. UI 中因果性注记正确显示
