## 一、业务问题

本模块解决一个核心决策问题：**在各池结束时，资源剩余量与最终成败之间存在怎样的概率关系？**

具体而言：

- **在某池以资源 R 结束时，最终失败的概率有多大？**（P(失败 | 资源剩余=r)）
- **存在哪些资源水平区间，使得失败概率超过用户容忍阈值 α？**（脆弱区间检测）
- **各池的脆弱性分布模式如何影响总体规划？**（多池山脊图对比）

典型场景：用户计划抽满 5 个角色，当前在第 3 个池结束时还有 12000 抽，想知道从第 4 个池开始继续抽，有多大把握最终拿到全部目标。如果资源降到 8000 以下失败概率 > 50%，则 8000 就是一个「脆弱边界」。

## 二、模块定位

### 诊断端——先诊断再决策

```
[批量模拟面板] → 产出 N 次 CompactResult 列表
                        ↓
              ┌───────────────────────┐
              │    RetreatPanel       │ ← 脆弱性诊断（本面板）
              │  (retreat_panel.py)    │
              │                       │
              │  配置 α / bins / GDR  │
              │  → 脆弱区间 + 折线图   │
              │  → 用户了解各池风险    │
              └───────────┬───────────┘
                          │ vulnerability_finished 信号
                          ↓
              ┌───────────────────────┐
              │   PlanSearchPanel     │ ← 方案决策（独立一级 Tab）
              │ (plan_search_panel.py) │
              │                       │
              │  接收脆弱性结果        │
              │  → 退路点方案搜索      │
              └───────────────────────┘
```

**P8 面板合并前**：脆弱性分析与退路搜索共处一个 Tab 下的父子面板架构。

**P8 面板合并后**：两模块拆分为独立一级 Tab——`脆弱性分析`（诊断）+ `方案搜索`（决策）。RetreatPanel 通过 `vulnerability_finished` 信号将分析结果传递给 PlanSearchPanel，作为退路点搜索的起点数据。

## 三、技术实现

### 3.1 数据流

```
CompactResult 列表
      │
      ↓ compute_vulnerability_analysis()  [vulnerability.py]
      │
      ├─ 按 pool_end_resources 分组 —— 每池的 (资源剩余, 成功/失败) 点集
      ├─ 二元响应回归 —— 局部逻辑回归（主曲线）+ 局部线性回归（方差估计）
      ├─ 方差估计 —— 高斯核平滑残差方差
      ├─ 脆弱区间检测 —— lp_curve > α 的连续区域提取
      ├─ 保底统计快照 —— PityStatSnapshot（均值/中位数/p25/p75）
      │
      ↓
VulnerabilityAnalysisResult
  ├─ pool_results: List[PoolVulnerabilityResult]
  ├─ alpha, gdr_key, gdr_threshold, n_simulations
  └─ overall_failure_rate, global_bin_edges
```

### 3.2 关键数据结构

| 数据结构 | 位置 | 说明 |
|----------|------|------|
| `VulnerabilityInterval` | `core/vulnerability.py:10-14` | 单个脆弱区间（下界/上界/均值/最大比率） |
| `PityStatSnapshot` | `core/vulnerability.py:17-23` | 保底计数器的汇总统计 |
| `PoolVulnerabilityResult` | `core/vulnerability.py:26-43` | 单池脆弱性结果（分箱/LLR曲线/方差/区间列表/保底快照） |
| `VulnerabilityAnalysisResult` | `core/vulnerability.py:46-54` | 整体分析结果（池列表 + 元数据） |

### 3.3 核心函数

| 函数 | 位置 | 说明 |
|------|------|------|
| `compute_vulnerability_analysis()` | `vulnerability.py:224-464` | **主入口**——处理结果列表 → 分箱 → 回归 → 区间检测 |
| `_fit_local_polynomial()` | `vulnerability.py:162-192` | 局部逻辑回归拟合 P(失败\|资源)，LLR 用于方差估计 |
| `_logistic_ll_fit()` | `vulnerability.py:106-158` | 局部线性逻辑回归（IRLS 迭代），阶段一新增 |
| `_ll_fit()` | `vulnerability.py:80-103` | 局部线性回归闭式解（加权最小二乘） |
| `_kernel_smooth_variance()` | `vulnerability.py:57-77` | 高斯核平滑方差估计 |
| `_find_contiguous_intervals_from_curve()` | `vulnerability.py:467-492` | 从 lp_curve 提取连续脆弱区间 |
| `_make_vulnerability_interval_from_curve()` | `vulnerability.py:495-519` | 构造区间对象 |
| `plot_vulnerability_ridge()` | `vulnerability.py:522-654` | 多池山脊图（Plotly go.Figure） |
| `plot_vulnerability()` | `vulnerability.py:657-749` | 单池 LLR 曲线 + 直方图（Plotly） |

## 四、GUI 架构

### RetreatPanel（`gui/retreat_panel.py`，~346 行）

| 组件 | 类型 | 角色 |
|------|------|------|
| `RetreatWorker` | `QThread` | 工作线程——调用 `compute_vulnerability_analysis()`，生成图表 |
| `RetreatPanel` | `QWidget` | 主面板——QSplitter 左右布局，配置区 + ChartWebView 图表区 |

**关键信号**：
- `RetreatWorker.progress(msg, pct)` → 进度更新
- `RetreatWorker.finished(dict)` → 分析完成（含 analysis + pool_specs + ridge_fig）
- `RetreatWorker.error(Exception)` → 错误处理
- `RetreatPanel.status_update(str)` → 状态栏消息
- `RetreatPanel.vulnerability_finished(VulnerabilityAnalysisResult)` → 向 PlanSearchPanel 传递结果

**配置项**：GDR 指标 + 成功阈值 + 脆弱比例 α + 直方图分箱数 + 回归曲线点数

**图表输出**：山脊总览图（多池对比）+ 单池详情（直方图上 + LLR 曲线下，脆弱区间红色着色）
