## 一、业务问题

本模块解决两大核心决策问题：

### 1. 脆弱性诊断

用户在某个池（截断点）停止抽卡时，按池结束时的剩余资源分组，回答：
- **在某池以资源 R 结束时，最终目标失败的概率有多大？**（P(失败 | 资源剩余=r)）
- **存在哪些资源水平区间，使得失败概率超过用户容忍阈值 α？**（脆弱区间检测）
- **各池的脆弱性分布模式如何影响总体规划？**

典型场景：用户计划抽满 5 个角色，当前在第 3 个池结束时还有 12000 抽，想知道从第 4 个池开始继续抽，有多大把握最终拿到全部目标。

### 2. 退路点方案优化

用户确定退路池（`from_pool_id`）后，回答三类搜索问题：
- **最少资源搜索**：「在该池截断并退路，额外投入多少资源能保证成功率 > 阈值？」（二分搜索 + SPRT）
- **最多目标卡搜索**：「给定固定资源，最多能追几张三张目标卡并成功？」（贪心后退法）
- **资源-目标权衡**：「前沿上各点的额外资源与可达目标组合？」（Pareto 前沿 / 贪心近似）

典型场景：用户在第 2 个池结束时资源告急，想放弃部分目标卡——保留哪几张能最大化成功率？

## 二、双面板架构

```
[批量模拟面板] → 产出 N 次 CompactResult 列表
                          ↓
              ┌───────────────────────┐
              │    RetreatPanel       │ ← 脆弱性诊断
              │  (retreat_panel.py)    │
              │                       │
              │  配置 α / bins / GDR  │
              │  → 脆弱区间 + 折线图   │
              │  → 用户选定退路池      │
              └───────────┬───────────┘
                          │ vulnerability_result
                          ↓
              ┌───────────────────────┐
              │  RetreatSearchPanel   │ ← 退路方案决策
              │ (retreat_search_panel.py) │
              │                       │
              │  搜索模式选择          │
              │  → 最少资源 / 最多目标 │
              │  → 资源-目标权衡曲线   │
              └───────────────────────┘
```

- **RetreatPanel** 是诊断端——先诊断再决策。「脆弱性诊断」→ 发现哪些池是高风险瓶颈 → 选定退路池。
- **RetreatSearchPanel** 是决策端——对选定的退路池做具体方案搜索，产出可操作的退路方案（保留 X 资源 + 放弃 Y 目标卡）。

## 三、技术实现：脆弱性分析管线

### 3.1 数据流

```
CompactResult 列表
      │
      ↓ compute_vulnerability_analysis()  [vulnerability.py]
      │
      ├─ 按 pool_end_resources 分组 —— 每池的 (资源剩余, 成功/失败) 点集
      ├─ 二元响应回归 —— LLR (当前默认) 或局部 logistic 回归 (阶段一新增)
      ├─ 方差估计 —— 核平滑残差方差
      ├─ 脆弱区间检测 —— lp_curve > α 的连续区域
      ├─ 保底统计快照 —— PityStatSnapshot (均值/中位数/p25/p75)
      │
      ↓
VulnerabilityAnalysisResult
  ├─ pool_results: List[PoolVulnerabilityResult]
  ├─ alpha, gdr_key, gdr_threshold, n_simulations
  └─ overall_failure_rate, global_bin_edges
```

### 3.2 关键数据结构

| 数据结构 | 位置 | 说明 |
|----------|------|------|
| `VulnerabilityInterval` | `core/vulnerability.py` | 单个脆弱区间（下界/上界/均值/最大比率） |
| `PoolVulnerabilityResult` | `core/vulnerability.py` | 单池脆弱性结果（分箱/LLR曲线/方差/区间列表/保底快照） |
| `VulnerabilityAnalysisResult` | `core/vulnerability.py` | 整体分析结果（池列表 + 元数据） |
| `PityStatSnapshot` | `core/vulnerability.py` | 保底计数器的汇总统计 |

### 3.3 核心函数

| 函数 | 说明 |
|------|------|
| `compute_vulnerability_analysis()` | 入口函数，处理结果列表 → 分箱 → LLR/Logistic → 区间检测 |
| `_fit_local_polynomial()` | 局部线性回归（LLR），当前默认方法 |
| `_logistic_ll_fit()` | 局部线性逻辑回归（IRLS），阶段一新增，替代 LLR |
| `_ll_fit()` | 单评估点局部线性拟合（被上面两个函数调用） |
| `_kernel_smooth_variance()` | 高斯核平滑方差估计 |
| `_find_contiguous_intervals_from_curve()` | 从 lp_curve 提取连续脆弱区间 |
| `_make_vulnerability_interval_from_curve()` | 构造区间对象（含数据范围裁剪） |
| `plot_vulnerability_ridge()` | 多池山脊图（Plotly） |
| `plot_vulnerability()` | 单池 LLR 曲线 + 直方图（Plotly） |

## 四、技术实现：退路搜索引擎

### 4.1 数据流

```
退路池 + 基础资源 + 保底初值 + 目标规格 + 权重
      │
      ↓ RetreatSearchEngine.search_xxx()  [retreat_search.py]
      │
      ├─ _build_truncated_env() → 截断配置 + SimulationEnv
      ├─ _simulate_with_resource() → 批量模拟 → CompactResult 列表
      ├─ _evaluate_success() → GDR 判定
      │
      ↓
RetreatSearchResult
  ├─ points: List[RetreatSearchPoint]  (额外资源 / 目标规格 / 成功率)
  ├─ resource_only_result: 纯资源搜索结果
  └─ target_only_result: 纯目标卡搜索结果
```

### 4.2 三个搜索算法

| 方法 | 算法 | 用途 | 当前缺陷 |
|------|------|------|---------|
| `search_min_resource()` | 二分搜索 + 固定 N 采样 | 寻找给定目标卡的最少所需资源 | R1/26: 噪声振荡 + 收敛无保证；R6: epsilon 缺乏统计依据 |
| `search_max_targets()` | 贪心后退法（按 miss_cost 升序逐一移除） | 给定资源下最大化可行目标卡 | R2: 仅探索 N+1 个点；R5: 权重缺乏基数解释 |
| `search_pareto()` | 调用 `search_max_targets` 后组合为前沿 | 资源-目标权衡曲线 | 24/R2: 非真正 Pareto，仅贪心近似；R4: 不共享模拟信息 |

### 4.3 关键数据结构

| 数据结构 | 位置 | 说明 |
|----------|------|------|
| `RetreatSearchPoint` | `core/retreat_search.py` | 单个搜索点（额外资源/目标规格/成功率） |
| `RetreatSearchResult` | `core/retreat_search.py` | 搜索结果（池/基础资源/保底初值/模式/点列表） |
| `RetreatSearchEngine` | `core/retreat_search.py` | 搜索引擎类（封装截断配置构建 + 模拟 + 搜索逻辑） |

### 4.4 相关外围代码

| 文件 | 角色 | 注意 |
|------|------|------|
| `core/retreat_config.py` | `RetreatConfigBuilder` 截断配置构建 | 从原始配置截断到退路池 |
| `core/forward_backward.py` | ForwardSearch/BackwardSearch 数据类 | 前进/后退法搜索方案（策略面板中使用） |
| `gui/resource_search_panel.py` | ResourceSearchPanel + ResourceSearchWorker | 独立的二分搜索实现（含 GUI）——与 `retreat_search.py` 功能重叠 |
| `gui/strategy_panel.py` | StrategyPanel | 前进/后退法搜索 GUI，使用 miss_cost/draw_target 权重 |

## 五、双面板 GUI 架构

### RetreatPanel（`gui/retreat_panel.py`）

- **RetreatWorker**（QThread）：工作线程，调用 `compute_vulnerability_analysis()`
- **RetreatPanel**（QWidget）：
  - 配置区：alpha（脆弱阈值）、num_curve_points、GDR 指标选择
  - 输出区：ChartWebView → `plot_vulnerability_ridge()` 山脊图 + 单池 `plot_vulnerability()` 曲线
  - 与 RetreatSearchPanel 联动：传递 `vulnerability_result` 和选定的退路池

### RetreatSearchPanel（`gui/retreat_search_panel.py`）

- **RetreatSearchWorker**（QThread）：工作线程，调用 `RetreatSearchEngine.search_xxx()`
- **RetreatSearchPanel**（QWidget）：
  - 配置区：退路池选择、搜索模式（最少资源/最多目标/Pareto）
  - 接收 `vulnerability_result` + 基础资源值 + 保底计数器初值
  - 权重配置：miss_cost_weights、desire_weights、card_value_weights
  - 输出区：ChartWebView → 搜索结果可视化 + 方案表格
