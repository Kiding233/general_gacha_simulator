| | `search_pareto()` — 调用 `search_max_targets()` 组合为 Pareto 前沿 |

### 三个搜索算法签名

```python
def search_min_resource(
    self, target_specs: Dict[str, int]
) -> RetreatSearchResult:
    """二分搜索：找到使成功率 ≥ success_threshold 的最少额外资源。
    每步固定运行 num_simulations 次模拟估计 p(r_mid)。
    当前使用翻倍法找上界（10 次翻倍，最大 50×2^10=51200 抽）。"""

def search_max_targets(
    self, base_target_specs: Dict[str, int]
) -> RetreatSearchResult:
    """贪心后退法：从全量目标卡开始，按 miss_cost_weight 升序逐一移除，
    直到资源足以成功。探索 N+1 个点（含全量 + 逐一移除后的子集）。"""

def search_pareto(
    self, target_specs: Dict[str, int]
) -> RetreatSearchResult:
    """资源-目标权衡曲线：调用 search_max_targets()，
    将其 N+1 个探索点作为 'Pareto 前沿' 返回。
    当前不执行支配性过滤——被支配点可能包含在内。"""
```

### 当前缺陷与待改进点

详见 P26 计划（`P26 退路搜索与资源目标搜索理论改进计划.md`），摘要：

- **R1/26**：二分搜索每步固定 N=500 → 改用 SPRT 序贯采样
- **24/R2**：Pareto 声称 → UI 降级为「贪心近似」+ 支配性过滤（阶段一）；引入 ε-Pareto 前沿（阶段三）
- **25/R3**：翻倍上界不一致 + 无不足检测 → 统一为 15 次翻倍 + 公共函数 + 上界不足报告
- **R4**：不共享模拟信息 → 随机克里金/贝叶斯优化（阶段三，依赖 SPRT 先完成）
- **R5**：错失代价权重缺乏基数解释 → UI 标注 + 替代排序方案（阶段一）
- **R6**：epsilon 精度缺乏统计依据 → 关联噪声水平（阶段一）

## 四、`gui/retreat_panel.py` 脆弱性分析面板

### 类结构

| 类 | 基类 | 角色 |
|----|------|------|
| `RetreatWorker` | `QThread` | 工作线程——调用 `compute_vulnerability_analysis()` |
| `RetreatPanel` | `QWidget` | 主面板——配置 + 触发 + 图表显示 |

### 关键信号

```python
class RetreatWorker(QThread):
    progress = pyqtSignal(str, int)   # 消息 + 百分比
    finished = pyqtSignal(object)     # VulnerabilityAnalysisResult
    error = pyqtSignal(object)        # Exception

class RetreatPanel(QWidget):
    status_update = pyqtSignal(str)   # 状态栏消息
```

### 关键方法

| 方法 | 说明 |
|------|------|
| `set_store(ConfigStore)` | 接收配置 |
| `set_config_panel(config_panel)` | 接收配置面板引用（读取策略类型等） |
| `set_vulnerability_result(result)` | 外部注入脆弱性分析结果（可从批量模拟面板传入） |
| `_on_analyze_clicked()` | 触发分析——创建 RetreatWorker → 启动 |
| `_on_analysis_finished(result)` | 分析完成——更新图表（Plotly WebView） |

### 与 RetreatSearchPanel 的联动

RetreatPanel 通过 `vulnerability_result` 属性（或信号）将分析结果传给 RetreatSearchPanel：
- 用户在 RetreatPanel 中查看脆弱区间 → 选定退路池
- RetreatSearchPanel 接收 `from_pool_id` + `base_resource` + `pity_init`，执行方案搜索

    finished = pyqtSignal(object)     # RetreatSearchResult

### 搜索模式

```python
self.search_mode: str  # 'resource' | 'target' | 'pareto'
```

### 权重配置

面板中可配置三种权重（均来自 ConfigStore 或用户手动输入）：
- `miss_cost_weights: Dict[str, float]` — 错失代价（影响目标卡移除顺序）
- `desire_weights: Dict[str, float]` — 需求权重（GDR weighted_satisfaction 使用）
- `card_value_weights: Dict[str, float]` — 卡牌价值权重（GDR total_card_value 使用）
