## 三、`gui/retreat_panel.py` 类结构

| 类 | 基类 | 行号 | 角色 |
|----|------|------|------|
| `RetreatWorker` | `QThread` | 28-110 | 工作线程——调用 `compute_vulnerability_analysis()` + 图表生成 |
| `RetreatPanel` | `QWidget` | 113-346 | 主面板——QSplitter 布局，配置区 + ChartWebView |

### RetreatWorker 参数

```python
class RetreatWorker(QThread):
    def __init__(
        self,
        simulation_results,         # List[CompactResult]
        target_specs,               # Dict[str, int]
        gdr_key,                    # str（默认 'all_targets'）
        gdr_threshold,              # float（默认 1.0）
        alpha,                      # float（默认 0.5）
        num_bins,                   # int（默认 20）
        num_curve_points,           # int（默认 200）
        pool_names,                 # Dict[str, str]
        desire_weights,             # Optional[Dict[str, float]]
        miss_cost_weights,          # Optional[Dict[str, float]]
        card_value_weights,         # Optional[Dict[str, float]]
        no_draw_resource,           # Optional[float]（不抽卡基线）
        no_draw_pool_resources,     # Dict[str, Dict[str, float]]
        cost_per_draw,              # float（默认 160）
        use_draw_units,             # bool
    ):
```

### RetreatPanel 关键方法

| 方法 | 说明 |
|------|------|
| `set_store(ConfigStore)` | 接收配置（目标卡、池名称、权重） |
| `set_config_panel(config_panel)` | 接收配置面板引用（读取策略类型等） |
| `set_simulation_results(results, target_specs, ...)` | 接收批量模拟结果 |
| `_run_analysis()` | 触发分析——创建 RetreatWorker → 启动 |
| `_on_finished(result)` | 分析完成——更新图表 + 发射 `vulnerability_finished` |
| `_on_error(err)` | 错误处理——恢复按钮 + 显示错误消息 |

### 信号流

```
RetreatWorker.progress  ──→  RetreatPanel._on_progress  ──→  status_update
RetreatWorker.finished  ──→  RetreatPanel._on_finished  ──→  chart_webview.set_charts()
                         │                                   │
                         │                                   └→  vulnerability_finished.emit(analysis)
                         └──→  status_update（状态栏消息）
RetreatWorker.error     ──→  RetreatPanel._on_error
```

## 四、外部依赖清单

| 导入路径 | 使用的符号 | 使用场景 |
|---------|-----------|---------|
| `gacha_simulator.core.vulnerability` | `compute_vulnerability_analysis`, `plot_vulnerability`, `plot_vulnerability_ridge` | RetreatWorker.run() |

### 上游数据提供者

| 数据 | 来源 | 说明 |
|------|------|------|
| `simulation_results` | 批量模拟面板 → `set_simulation_results()` | `List[CompactResult]` |
| `target_specs` | ConfigStore 或批量模拟面板传入 | 目标卡 ID → 数量 |
| `no_draw_resource` | SimulationEnvBuilder 计算 | 「不抽卡」基线的资源量 |
| `no_draw_pool_resources` | SimulationEnvBuilder 计算 | 每池不抽卡资源基线 |
| `cost_per_draw` | 面板内部 `_extract_cost_per_draw()` | 从池配置提取单抽成本 |

## 五、数据流总结

```
批量模拟面板
  ├── run_batch_parallel() → List[CompactResult]
  └── 不抽卡基线（no_draw_resource, no_draw_pool_resources）
         │
         ▼ set_simulation_results()
  RetreatPanel
         │
         ▼ 用户点击「运行脆弱性分析」
  RetreatWorker.run()
         │
         ├── compute_vulnerability_analysis(results, ...)
         ├── plot_vulnerability_ridge(analysis, ...)
         └── plot_vulnerability(pool_result, ...)
         │
         ▼ _on_finished({analysis, pool_specs, ridge_fig, ...})
         │
         ├── chart_webview.set_charts(charts)
         └── vulnerability_finished.emit(analysis) → PlanSearchPanel
```
