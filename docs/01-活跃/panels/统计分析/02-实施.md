## 二、三个核心子功能的代码入口

### 2.1 GDR 分布（`gdr_dist`）

**入口**：`AnalysisWorker._run_impl()` → `_prepare_gdr_dists` + `gdr_dist` 步骤（line 276-355）

**计算流程**：

```python
# 步骤0：预计算所有 GDR 指标的 EmpiricalDistribution
for gdr_def in UNIFIED_GDR_REGISTRY.values():
    values = [compute_gdr_from_compact(agg, target_specs, name) for agg in aggregate_data]
    gdr_dists[name] = EmpiricalDistribution(values)

# 步骤1：按用户选择绘制直方图/CDF
for metric_name, chart_types in gdr_dist_selections.items():
    dist = gdr_dists.get(metric_key)
    if 'hist' in chart_types:
        charts[key] = ChartSpec(chart_type="histogram", data=HistogramData(samples=...))
    if 'cdf' in chart_types:
        charts[key] = ChartSpec(chart_type="cdf", data=CDFData(samples=...))
```

**关键函数调用链**：
- `UNIFIED_GDR_REGISTRY`（`core/gdr.py`）——获取 13 种指标定义
- `compute_gdr_from_compact(agg, target_specs, name)`（`core/gdr.py`）——对单条模拟结果计算单个 GDR
- `EmpiricalDistribution(values)`（`core/distribution.py`）——构建经验分布，提供 `var`/`cvar`/`quantile`/`mean`/`median` 方法

### 2.2 转变分析（`transition_analysis`）

**入口**：`AnalysisWorker._run_impl()` → `transition_analysis` 步骤（line 1194-1293）

**计算流程**：

```python
# 数据源优先级：streaming预计算 > 实时计算
if self.transition_flags:
    success_flags_per_sim = self.transition_flags   # 流式预计算（推荐）
elif self.cumulative_snapshots:
    success_flags_per_sim = compute_transition_flags_from_gdr(...)  # 回退路径

# 核心计算
trans = compute_transition_matrices_from_flags(success_flags_per_sim, pool_ids_ordered)

# 可视化：成功率变化折线图 + 2×2 转移矩阵子图网格
charts['transition_analysis_rates']   # ScatterTrace: success_rate_before vs after
charts['transition_analysis_matrices'] # SubplotGridData: 每个转移对的 2×2 矩阵
```

**关键函数调用链**：
- `compute_transition_flags_from_gdr()`（`core/per_pool_analysis.py`）——从 cumulative_snapshots 实时计算成功/失败标记序列
- `compute_transition_matrices_from_flags()`（`core/per_pool_analysis.py`）——构造 `TransitionMatrix` 列表，包含 `success_to_success`/`success_to_fail`/`fail_to_success`/`fail_to_fail` 四个概率值

**数据依赖**：
- `transition_flags`：由 `TransitionFlagsExtractor`（`core/streaming.py`）流式预计算，格式为 `List[Dict[str, bool]]`
- `cumulative_snapshots`：回退数据源，格式为 `Dict[pool_id, List[dict]]`
- `pool_end_times`：用于确定池子时间顺序

### 2.3 每池 GDR（`per_pool_draws` / `cumulative_by_pool` / `per_pool_target_rate` / `per_pool_pity_rate`）

# 步骤1：从每条模拟中提取 PoolSnapshot
for agg in aggregate_data:

## 三、线程模型

### 3.1 AnalysisWorker（QThread）

```python
class AnalysisWorker(QThread):
    finished = pyqtSignal(dict)     # dict[str, ChartSpec]
    progress = pyqtSignal(str, int) # 状态消息, 百分比
    error = pyqtSignal(str)         # 错误消息

    def __init__(self, results, ctx, pool_end_times, selected, alpha, output_dir, ...):
        # 接收所有预处理数据（不自行遍历全量结果）

    def run(self):
        self._run_impl()  # 主计算逻辑，在子线程中执行
```

**关键设计**：
- Worker 在子线程中执行 GDR 计算 + ChartSpec 构造，不涉及 GUI 操作
- 进度报告通过 `progress` 信号异步发送
- `ChartSpec` 是纯数据类（dataclass），跨线程传递安全
- 实际 Plotly.js 渲染在主线程的 `_on_analysis_done()` 中进行

### 3.2 生命周期

```
update_results()                   # 接收新模拟结果，清空旧缓存
    ↓
用户勾选分析项 → 点击运行分析
    ↓
_run_analysis()                    # 主线程
    ├── 检查缓存（_needs_computation）
    ├── 创建 AnalysisWorker
    ├── worker.progress → _on_progress
    ├── worker.finished → _on_analysis_done
    └── worker.error → _on_analysis_error
```

### 3.3 条件缓存

`_computed_conditions` 字典的 key 为分析项标识符，value 为 `frozenset` 化的参数组合（α、指标选择、阈值等）。`_needs_computation()` 比较当前参数与缓存参数，判断是否需要重新计算。

## 五、配置入口

面板通过 `set_store(ConfigStore)` 接收配置，用于：
- `_get_pool_names()`：池 ID → 中文显示名映射（`store.pools[].name`）
- `_extract_cost_per_draw()`：从池子的 `cost` 字段提取单抽消耗（格式 `"资源类型:数量"`）

面板没有独立的配置持久化——所有用户选择（勾选、参数值）仅在内存中，切换 Tab 或关闭应用后丢失。
