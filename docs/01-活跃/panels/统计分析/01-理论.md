## 一、核心假设

### 1.1 GDR 指标独立性假设（当前隐式采用）

`compute_gdr_from_compact()` 对每条模拟结果独立计算 13 种 GDR 指标值，各指标间不做联合建模。相关性分析（`correlation`）使用 Pearson 相关系数矩阵——这隐含假设 GDR 指标间的联合分布可用线性相关结构近似。当某些 GDR 指标为二元（如 `all_targets` 取 0/1）而另一些为连续时，Pearson 相关系数可能低估或扭曲真实关系。

### 1.2 转变分析的一阶马尔可夫假设

转变分析将相邻池子间的成功/失败状态转移建模为 2×2 一阶马尔可夫转移矩阵。假设：

- **下一池的成功/失败仅取决于上一池的状态**（Markov property），不依赖更早池子的历史
- **转移概率对同批次所有模拟是平稳的**（同质性），不随模拟间的资源消耗差异而变化

这两个假设在抽卡游戏中可能不成立，详见下方 §2.2（缺陷 T3）。

### 1.3 条件分布的独立性假设

最差情形/最好情形/条件分布分析中，使用 `JointSamples.conditional_second()` 对主指标施加阈值过滤后，计算其他指标的条件分布。这隐含假设条件过滤后的子样本仍能代表条件分布——当极端分位数样本量很小时（如 n=1000 模拟、α=0.05 时尾部仅 50 个样本），条件均值和 VaR 的估计精度很低，但当前系统不报告条件样本量或条件估计的标准误。

### 1.4 Bootstrap CI 的正态近似假设（GDR 统计表格）

`_compute_statistics_unit()` 使用 `BootstrapEngine(B=1000, use_bca=True)` 计算均值、中位数、VaR 的 Bootstrap 置信区间。BCa（bias-corrected and accelerated）方法比百分位方法更准确，但仍假设 Bootstrap 分布能较好地逼近真实抽样分布——当 GDR 指标为二元（0/1）或高度离散时，此假设可能不满足。

### 1.5 每池分析的事件类型推断假设

`infer_events()` 根据抽卡序列和池 GDR 判定为每池分配 7 种事件类型之一（`pity_hit`/`early_hit`/`miss`/`skip`/`ignore`/`exchange`/`no_exchange`）。其中 skip/ignore 的判定依赖 `pool_target_map` 正确推断——若目标卡所属池子推断错误，后续 AA/BB/AB/BA 交叉分析的分类将系统性偏差。

|---|------|--------|---------|------------|
| T1 | 转移概率无置信区间/标准误 | 严重 | 用户看到 0.3→0.8 的转变，不知道是否显著——点估计无 SE、无 CI、无假设检验。转移矩阵每行构成多项分布（行内概率和为 1），独立比例估计忽略此约束 | P27 §2.1（Wilson 得分区间 + Bootstrap 同时置信域） |

| # | 缺陷 | 严重度 | 核心问题 | 改进计划章节 |
|---|------|--------|---------|------------|
| P1 | K 个池的多重比较无校正 | 严重 | K=8 时 FWER ≈ 34%。各池成功判定非独立（早期池消耗损害后续池），但系统不报告此约束 | P28 §2.1（FDR + Bootstrap 同时置信域） |

## 三、转变分析中的马尔可夫假设详解

### 3.1 假设内容

转变分析计算 `P(池_{i+1}=成功 | 池_i=成功)` 和 `P(池_{i+1}=成功 | 池_i=失败)`。这等价于假设池子间的成功/失败序列构成一阶马尔可夫链——下一池的状态仅依赖于上一池的状态，与更早的历史条件独立。

### 3.2 违反假设的两个维度

**保底状态记忆**（P27 §2.2）：保底系统维护一个跨池持续的计数器。A 池结束时计数器为 75（80 触发软保底）→ B 池的起始概率显著高于计数器为 0 的情况。但从转移矩阵视角看，A 池成功/失败这一二元状态丢失了「A 池结束时保底计数器为 75」这一关键信息。

**资源消耗路径依赖**：考虑两种「A 池成功」的模式：
- 模式 X：第 1 抽即出目标——资源消耗 1%，后续池空间充裕
- 模式 Y：第 80 抽（保底）出目标——资源消耗 100%，后续池资源紧张

两种模式均被编码为 `before=True`，但 B 池的真实成功率可能分别为 90%（模式 X）和 45%（模式 Y）。转移矩阵报告的 P(成功\|之前成功) 是这两种模式的混合平均——掩盖了实质性的子群差异。

### 3.3 不膨胀状态空间的缓解方案

P27 提出在 `TransitionMatrix` 中附加 `mean_resource_consumed_given_success` 和 `mean_resource_consumed_given_failure` 两个连续度量——仍保持 2×2 状态空间，但补充转移「强度」信息。资源消耗差异 <10% 表明马尔可夫解释可信；>50% 表明路径依赖显著。

## 四、每池GDR中的 Simpson 悖论风险详解

### 4.1 悖论结构

Simpson 悖论（Simpson, 1951）指聚合层面的统计关系与分组层面相反或显著不同。在每池分析的语境下：

```
池 A（角色池）：成功率 95%，消耗 60% 资源
池 B（武器池）：成功率 40%，消耗 40% 资源
全局成功率：85%（由池 A 的高成功率和高权重驱动）
```

用户看到全局 85% 可能满意，但池 B 的 40% 是需要关注的瓶颈——系统当前不自动提示此差异。

### 4.2 诊断方法

P28 提出两种自动诊断：
- **变异系数**：CV = std(各池成功率) / mean(各池成功率)，CV > 0.3 → 标注「高度异质」
- **极差**：max(池成功率) − min(池成功率) > 0.3 → 标注「池间成功率跨度大」

### 4.3 与全局 GDR 的关系

全局 GDR（`target_achievement`、`resource_remaining` 等）是从全模拟视角计算的聚合指标。每池 GDR（通过 `pseudo_compact` 构造的单池视角）是局部视图。两者不矛盾——但系统需要提醒用户：**全局成功可能由容易的池子驱动，掩盖高难度池子的低成功率**。这类似于医学文献中「聚集 AUC 高于各亚组 AUC」的 Simpson 悖论模式（Lerman, 2018）。

## 五、未建模的理论问题

### 5.1 模拟结果的空间相关性

同一批次模拟使用相同的调度（schedule）、相同的池子参数——模拟间的差异仅来自抽卡的随机种子。统计学上，模拟结果是独立同分布的（i.i.d.），前提是所有模拟从相同的初始状态出发。如果未来引入「模拟间共享全局状态」（如有限卡池/库存竞争），独立性假设将被打破。

### 5.2 双重随机性

每次模拟包含两层随机性：
1. **抽卡层**：从卡池中按受保底调整的概率分布抽取卡片
2. **策略层**：策略决策（何时切换池子、何时停止）取决于当前状态

GDR 分布是这两层随机性的卷积。当前分析不区分「运气的随机性」和「策略选择的随机性」——它们被不可分割地混合在 GDR 样本中。
