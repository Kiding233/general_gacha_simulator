## 一、问题概述

转变分析（transition analysis）对相邻卡池间的成功/失败状态转移进行 2×2 马尔可夫转移矩阵建模。补充审查发现 5 项缺陷，其中 2 项严重、2 项中等、1 项低等。

### 1.1 缺陷总览

| # | 缺陷 | 严重度 | 核心问题 |
|---|------|--------|---------|
| T1 | 转移概率无置信区间/标准误 | 🔴 严重 | 用户看到 0.3→0.8 的转变，不知道是否显著 |
| T2 | 零分母静默归零 | 🟡 中等 | `s_before=0` 时返回 0.0，被误解为「必然失败」 |
| T3 | 一阶马尔可夫假设可能不成立（保底/资源路径依赖） | 🔴 严重 | 转移概率估计可能有系统性偏差 |
| T4 | 多池转移的多重比较问题 | 🟡 中等 | K 个池 × 4 个概率值，FWER ≈ 1−(1−α)^(4K) |
| T5 | 事件-成败与池-池转变的 UI 信息冗余 | 🟢 低等 | 两种分析的关系未在 UI 中说明 |

### 1.2 涉及文件

| `core/streaming.py` | transition_flags 流式预计算 |

## 二、缺陷详解与改进方案

### 2.1 缺陷 T1：转移概率无不确定性量化（严重）

**现状**：

```python
success_to_success = ss / s_before if s_before > 0 else 0
```

将转移概率报告为单一数值，无标准误、无置信区间、无假设检验。

**理论背景**：2×2 转移矩阵每行是一个二项比例 MLE，SE = √(p(1−p)/n)。在典型规模下（n=1000）：
- P(成功|之前失败)=0.3 → SE≈0.015，95% CI≈[0.27, 0.33]
- P(成功|之前成功)=0.8 → SE≈0.013，95% CI≈[0.77, 0.83]
- 边界情况（P=0.95, n_before=200）：SE≈0.015，但正态近似在边界处失效

更关键的是，转移矩阵各行构成**多项分布**（行内概率和为 1）——一个单元格的方差必然伴随其他单元格的补偿变化。当前独立比例估计忽略此约束。

**推荐方案**：
1. **Wilson 得分区间**（二项比例 CI，边界处优于 Wald 区间）：替代简单比例报告，显示「0.3 [0.27, 0.33]」
2. **Bootstrap 同时置信域**：对每行多项比例使用 Bootstrap 重抽样 + 同时置信域方法（Sison-Glaz 或 Goodman），确保行内概率和 = 1
3. **UI 展示**：在转变矩阵热图中，单元格显示「估计值 ± CI」，非显著差异以灰色标记

**文献**：Craig & Sendi (2002, *Health Economics*); Möstel, Pfeuffer & Fischer (2020, *Computational Statistics*); Sison & Glaz (1995, *JASA*); Goodman (1965, *Technometrics*)

**依赖**：P3（Bootstrap 引擎，核心已完成）提供重抽样基础设施。

### 2.2 缺陷 T3：一阶马尔可夫假设可能不成立（严重）

**现状**：转变分析假设下一池的成功/失败仅依赖于上一池的状态。在抽卡游戏中，两个维度的路径依赖破坏此假设：

1. **保底状态记忆**：保底计数器积累跨越多池。A 池结束时的 70 抽计数器（80 触发软保底）→ B 池的转移概率不仅取决于 A 的成败，还取决于保底进度。
2. **资源消耗路径依赖**：A 池「险胜」（消耗 80% 资源）vs「轻松获胜」（消耗 30% 资源）→ B 池成功率截然不同，但当前模型将它们等同为 `before=True`。

**推荐方案（不膨胀状态空间的前提下）**：

**方案 A（低风险，推荐优先实施）——TransitionMatrix 附加连续度量**（见补充审查 §2.4）：

在 `TransitionMatrix` 数据结构中附加：
- `mean_resource_consumed_given_success`：之前成功组的平均资源消耗
- `mean_resource_consumed_given_failure`：之前失败组的平均资源消耗

这两个值的差异直接量化「成败的资源代价差异」：
- 差异 <10% → 成败主要由运气驱动，马尔可夫解释可信
- 差异 >50% → 资源消耗路径依赖严重，马尔可夫假设可能被违反

此方案不增加状态空间（仍为 2×2），计算成本可忽略（两次均值聚合），直接利用模拟中已有的资源消耗数据。

**方案 B（中风险，后续评估）——诊断性检验**：
1. 计算「给定之前状态」子群的资源分布均值/方差，若差异 >30%，在 UI 中标注「马尔可夫假设可能被违反：资源消耗路径依赖显著」
2. 若用户启用了多保底且保底跨池继承，在 UI 中标注：「保底状态记忆可能影响转移估计」
3. 未来版本可考虑 Billingsley (1961) 的马尔可夫链拟合优度检验

**文献**：Billingsley (1961), *Annals of Mathematical Statistics*; Anderson & Goodman (1957), *Annals of Mathematical Statistics*

### 2.3 缺陷 T2：零分母静默归零（中等）

**现状**：`ss / s_before if s_before > 0 else 0`

当 `s_before = 0`（之前无成功）时，代码返回 0.0。但这意味着「数据不足以估计」，而非「之前成功后必然失败」。静默返回 0.0 会误导用户相信 P(成功|之前成功) = 0%。

**推荐方案**：
1. 零分母时返回 `NaN` 并在 UI 中显示「数据不足」或「N/A」
2. 在 `TransitionMatrix` 中新增 `low_sample` 标记字段（类似 P2 中过程分析的 `low_sample` 标记）
3. 若分母 < 30（经验阈值），显示「低样本量（n=X）」警告

---

### 2.4 缺陷 T4：多池转移的多重比较问题（中等）

**现状**：对于 K 个有顺序的池，生成 K 个转移矩阵，每个含 4 个概率值。若用户检查所有 K×4 个值并关注「显著的」变化（如某处 P(成功|失败) 突然下降），FWER 约为 1−(1−α)^(4K)。对于 K=8：

- **未校正 FWER ≈ 1−(0.95)^32 ≈ 81%**

即使用户看到某个转移的「显著」变化，它有 81% 的概率只是随机波动。

**推荐方案**：
1. 在转变分析面板中标注：「此为探索性分析，非确认性检验——跨多池的转移变化需谨慎解读」
2. 在 UI 中提供可选的 **Benjamini-Hochberg FDR 校正**（Benjamini & Hochberg, 1995）：控制错误发现率而非族错误率，在多检验场景下比 Bonferroni 更不保守
3. 对相邻转移矩阵的差异使用 **Fisher 精确检验**（针对二项比例差异），而非仅眼球比较

**文献**：Benjamini & Hochberg (1995, *JRSS-B*); Benjamini & Yekutieli (2001, *Annals of Statistics*)

---

### 2.5 缺陷 T5：事件-成败与池-池转变的 UI 信息冗余（低等）

**现状**：`process_analysis.py` 的 AB 分析（事件→成败）计算 P(成功|事件模式)，与池-池转变分析概念上有重叠——「事件」维度（pity_hit/early_hit/miss）编码池内部过程，而池-池转变仅看最终成败。两种分析呈现不同粒度但 UI 未说明关系。

**推荐方案**：
1. 在转变分析面板添加一行说明文字：「此分析关注池间的状态转移（上一池成败 → 下一池成败）。如需分析池内部的成败驱动因素（事件→成败），请查看过程分析的 AB 交叉分析。」
2. 反之在过程分析面板也添加对称说明

## 三、改进方向：不膨胀状态空间的前提下补充转移强度信息

（详见补充审查 §2.4）

当前二元状态（成功/失败）的一个关键局限是丢失了转移「强度」——「险胜」和「轻松获胜」被等同对待。建议在 `TransitionMatrix` 中附加：

- **之前成功组的平均资源消耗**：`mean({资源消耗_i | before_i=True})`
- **之前失败组的平均资源消耗**：`mean({资源消耗_i | before_i=False})`

这量化了「成败的资源代价差异」——若差异小（<10%），马尔可夫解释可信；若差异大（>50%），为假设违反提供定量证据。此方案不增加状态空间（仍 2×2），计算成本可忽略。
