# 00-档案 — 统计分析面板

> 日期：2026-06-02 | 版本：v1（首次建档）
> 面板标识：`AnalysisPanel`（GUI Tab 名称："统计分析"）
> 关联面板：过程分析面板（共享 `process_trace`/`process_analysis` 分析路径）、策略比较面板（共享 GDR 指标定义）

## 一、业务问题

在对抽卡模拟系统进行批量模拟后，用户需要对多维度的模拟结果进行统计分析与可视化探索。核心需求包括：

1. **整体表现评估**：数千次模拟中，各类广义出率（GDR）指标的分布形态、集中趋势、尾部风险（VaR/CVaR）如何？
2. **条件探索**：当某项 GDR 指标处于极端值时（最差/最好情形），其他指标如何联动变化？
3. **时间演化追踪**：目标达成率、资源剩余、SSR 收集率等如何随抽卡次数/时间推移而变化？
4. **池间对比**：各卡池的抽卡数、目标卡出率、保底触发率有何差异？相邻池子间的成功/失败状态是否存在转移模式？
5. **指标间关系**：13 种 GDR 指标之间的相关性结构如何？

这些分析共同构成「从原始模拟结果到可操作洞察」的统计桥梁——玩家/设计者不仅看到「全局成功率 85%」，还能追问「哪些池子拖了后腿」、「池 A 失败后池 B 成功的概率是多少」、「最差 5% 情形下的资源消耗」。

## 二、核心子功能

统计分析面板通过 `ANALYSIS_CATEGORIES` 将分析项组织为 4 个大类、15 个子项：

### 2.1 总体广义出率分析

| 键 | 显示名 | 输出类型 | 说明 |
|----|--------|---------|------|
| `gdr_dist` | GDR分布 | 直方图 + CDF（多页） | 可自由选择任一组 GDR 指标，绘制分布直方图和累积分布曲线 |
| `gdr_statistics` | GDR指标统计 | 汇总表格 | 均值、中位数、标准差、VaR，Bootstrapped CI（B=1000, BCa） |
| `correlation` | GDR指标相关性 | 热力图 | 13 种 GDR 指标两两 Pearson 相关系数矩阵 |

### 2.2 风险分析

| 键 | 显示名 | 输出类型 | 说明 |
|----|--------|---------|------|
| `risk_var_cvar` | VaR/CVaR分析 | 汇总表格 | VaR(alpha)、CVaR、VaR-均值差、VaR-中位数差 |
| `risk_worst_case` | 最差情形分析 | 直方图 + 汇总表格 | 选中主指标在 α 分位数以下的条件分布 |
| `risk_best_case` | 最好情形分析 | 直方图 + 汇总表格 | 选中主指标在 1-α 分位数以上的条件分布 |
| `conditional_dist` | 条件分布 | 直方图 + 汇总表格 | 条件指标 ≥/＜ 阈值时的目标指标分布 |

### 2.3 时间演化

| 键 | 显示名 | 输出类型 | 说明 |
|----|--------|---------|------|
| `time_series` | GDR时间序列演化 | 折线图 | 随机抽取最多 20 条模拟的逐抽 GDR 演化路径 |
| `time_heatmap` | 时间-GDR热力图 | 热力图 | 目标达成率/资源剩余/SSR出数随抽数次数的分布密度 |
| `draws_vs_gdr` | 抽卡数-目标达成率散点图 | 散点图 | 总抽卡数 vs 目标达成率的二维分布 |
| `waterfall_3d` | 3D瀑布图 | 3D 瀑布图 | 时间-目标卡数-概率三维分布 |
| `waterfall_2d` | 2D压缩瀑布图 | 多折线 | 不同时间步的目标卡数概率分布（压缩为 2D） |

### 2.4 每池分析

| 键 | 显示名 | 输出类型 | 说明 |
|----|--------|---------|------|
| `per_pool_draws` | 每池抽卡数统计 | 水平柱状图 | 各池平均抽卡数 |
| `per_pool_target_rate` | 每池目标卡数 | 水平柱状图 | 各池获取目标卡的平均数量 |
| `per_pool_pity_rate` | 每池保底数 | 水平柱状图 | 各池触发保底的平均次数 |
| `cumulative_by_pool` | 截止每池的GDR分布 | 山脊图（多页） | 每池结束时刻的各 GDR 指标分布 |
| `transition_analysis` | 转变分析 | 折线图 + 子图网格 | 相邻池间成功率变化 + 各转移对 2×2 转移概率矩阵 |

## 三、技术实现

### 3.1 核心架构

```
AnalysisPanel (QWidget)
├── 左侧: 选项配置区 (QScrollArea + AnalysisItemWidget 列表)
│   ├── 全局参数: α风险水平、Bootstrap CI置信水平
│   ├── 图表单位: 以抽数为单位开关、单抽消耗
│   └── 按4大类别分组的可展开分析项（含参数配置）
├── 右侧: 结果展示区 (QScrollArea + ChartWebView)
│   └── ChartWebView: Plotly.js 渲染的 WebView
└── AnalysisWorker (QThread)
    └── 接收 results + ctx → 输出 dict[str, ChartSpec]
```

### 3.2 计算流程

1. `update_results()` 接收 `results`（`List[CompactResult]`）+ `gdr_context` + 各子功能所需数据（`draw_sequences`、`heatmap_data`、`cumulative_snapshots`、`transition_flags`）
2. 用户勾选分析项 → 点击「运行分析」→ `_run_analysis()`
3. 主线程创建 `AnalysisWorker`（QThread），传入所有预处理数据
4. Worker 的 `_run_impl()` 依次计算各项分析，生成 `ChartSpec` 对象
5. Worker 发送 `finished` 信号，携带 `dict[str, ChartSpec]`
6. 主线程将 `ChartSpec` 传入 `ChartWebView.set_charts()` 渲染为 Plotly.js HTML

### 3.3 渲染管线

- **图表规范层**：`visualization/chart_spec.py` — 定义 `ChartSpec`、`HistogramData`、`ScatterData`、`BarData`、`HeatmapData`、`RidgeData`、`SubplotGridData`、`TableData` 等数据结构
- **渲染层**：`gui/chart_webview.py` — `ChartWebView`（QWebEngineView 子类），将 `ChartSpec`字典序列化为 Plotly.js JS 代码，在 WebView 中渲染交互式图表
- **历史**：v1.9.0 前使用 matplotlib → 临时 PNG → QLabel 渲染；现已全部迁移至 Plotly

### 3.4 关键设计决策

1. **条件缓存机制**：`_computed_conditions` 字典记录每次计算的参数组合（α、指标选择、阈值等），参数未变时跳过重算
2. **拓展开关联动**：带配置的分析项（`_EXPANDABLE_KEYS`）使用 `AnalysisItemWidget`，勾选时自动展开配置区，取消勾选时自动收起
3. **流式预计算**：`transition_flags`、`cumulative_snapshots`、`heatmap_data`、`draw_sequences` 由上一层（批量模拟面板）通过 streaming extractor 预计算并传入，避免 AnalysisPanel 内重复遍历全量模拟结果
4. **抽数为单位换算**：资源类 GDR（`resource_remaining`、`resource_consumed`、`resource_per_card`）支持除以 `cost_per_draw` 转换为抽数等价量，由 `_DRAW_UNIT_GDR_KEYS` 控制

## 五、涉及文件一览

| 文件 | 角色 |
|------|------|
| `gui/analysis_panel.py` | 面板主文件：UI 构建 + Worker 调度 + 结果渲染 |
| `core/gdr.py` | 13 种 GDR 指标定义 + 计算函数（`compute_gdr_from_compact`） |
| `core/distribution.py` | `EmpiricalDistribution` + `JointSamples` + `freedman_diaconis_bins` |
| `core/per_pool_analysis.py` | 每池统计汇总 + 转变矩阵计算（`TransitionMatrix`、`compute_transition_matrices_from_flags`） |
| `core/process_trace.py` | 池事件推断 + 单池 GDR 计算（`compute_pool_gdr_single_pool`、`pseudo_compact`） |
| `core/process_analysis.py` | AA/BB/AB/BA 交叉分析（`compute_aa`/`bb`/`ab`/`ba`） |
| `core/bootstrap.py` | Bootstrap 引擎（`BootstrapEngine`，GDR 统计表格中用于均值/中位数/VaR 的 CI） |
| `visualization/chart_spec.py` | Plotly 图表数据规范（`ChartSpec`、各数据类型） |
| `gui/chart_webview.py` | Plotly WebView 渲染器（`ChartWebView`） |
| `core/streaming.py` | 流式预计算 `transition_flags` 等（`TransitionFlagsExtractor`） |
