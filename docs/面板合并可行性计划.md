### 1.3 最多目标卡的两种搜索方法

当前 `RetreatSearchEngine.search_max_targets()` 仅支持**后退法**（逐个移除不重要的卡）。还缺少**前进法**（逐个添加最想要的卡）。`core/forward_backward.py` 已定义了 `ForwardStep` / `BackwardStep` / `ForwardResult` / `BackwardResult` 四种 dataclass，但前进法的搜索逻辑尚未实现。

#### 1.3.1 后退法（Backward / 已有）

```
从全部目标卡开始 → 每步移除一个最不重要的卡 → 直到成功率≥阈值
```

- 权重类型：**错失代价权重**（miss_cost_weights，值越低越先被移除）
- 权重含义：未能抽出该卡时的"遗憾代价"，低代价的卡优先放弃
- 当前权重来源：`RetreatSearchPanel` 自有权重表 + `config_panel.get_miss_cost_weights()`
- 停止条件：成功率 ≥ 阈值
- 返回：一系列 `BackwardStep`（每步移除的卡ID、剩余目标集、成功率）

#### 1.3.2 前进法（Forward / 需实现）

```
从空目标集开始 → 每步添加一个最想要的卡 → 直到成功率＜阈值 → 回退一步
```

- 权重类型：**抽取意愿权重**（desire_weights，值越高越先被添加）
- 权重含义：对该卡的"渴望程度"，高渴望的卡优先加入目标集
- 权重来源：`config_panel.get_desire_weights()`
- 停止条件：成功率 ＜ 阈值（取上一个满足条件的集合）
- 返回：一系列 `ForwardStep`（每步添加的卡ID、当前目标集、成功率）

#### 1.3.3 两种方法权重对比

| 维度 | 后退法 | 前进法 |
|------|--------|--------|
| 起始集合 | 全部目标卡 | 空（或至少一张） |
| 每步操作 | 移除一张卡 | 添加一张卡 |
| 排序权重 | miss_cost_weights（升序） | desire_weights（降序） |
| 停止条件 | 成功率 ≥ 阈值 | 成功率 ＜ 阈值 |
| 权重语义 | "哪张卡最不重要" | "哪张卡最想要" |
| 典型场景 | "我资源不够，先放弃哪些" | "我逐步追加目标，能加到多少" |

## 二、合并方案

### 2.1 统一搜索引擎：`PlanSearchEngine`

将 `RetreatSearchEngine` 扩展为通用引擎，支持完整时间线 + 截断两种模式，以及前进法/后退法两种目标卡搜索：

```python
class PlanSearchEngine:
    def __init__(self, config_store, *,
                 # 完整时间线模式
                 from_pool_id: Optional[str] = None,
                 # 截断模式参数
                 base_resource: float = 0.0,
                 pity_counter_init: Optional[Dict] = None,
                 # 通用参数
                 miss_cost_weights=None, desire_weights=None,
                 card_value_weights=None,
                 success_threshold=0.95, gdr_key='all_targets',
                 gdr_threshold=1.0, num_simulations=500,
                 max_workers=4, strategy_name='smart',
                 strategy_params=None,
                 progress_callback=None):
        ...

    def _build_env(self, target_specs, resource_value):
        if self.from_pool_id is None:
            # 完整时间线模式
            env = SimulationEnvBuilder.from_config_store(self.config_store)
            ir = dict(env.initial_resources)
            ir['draw_resource'] = resource_value
            env.initial_resources = ir
            return env
        else:
            # 截断模式（原逻辑）
            ...

    # --- 三种搜索模式 ---

    def search_min_resource(self, target_specs) -> ResourceSearchResult:
        """最少资源：给定目标集，二分搜索最少资源"""

    def search_max_targets_backward(self, target_specs) -> BackwardResult:
        """最多目标卡 — 后退法：从全部目标开始，逐个移除"""

    def search_max_targets_forward(self, candidate_specs) -> ForwardResult:
        """最多目标卡 — 前进法：从空开始，逐个添加"""

    def search_pareto(self, target_specs) -> ParetoResult:
        """Pareto前沿：完整目标集 + 逐卡移除后的最少资源"""
```

**关键改动**：`from_pool_id=None` 表示完整时间线模式，非 None 表示截断模式。

### 2.2 搜索方法选择逻辑

在面板中，"最多目标卡"模式下提供两个子选项：

```
搜索类型: ○ 最少资源（固定目标集）
          ○ 最多目标卡（固定资源）  ┬ ○ 后退法（逐步移除不重要卡）
           │                        └ ○ 前进法（逐步添加想要卡）
          ○ Pareto前沿（自动双维度搜索）
```

切换前进法/后退法时：
- 前进法 → 显示 desire_weights 权重预览（从配置页读取，"高渴望优先添加"）
- 后退法 → 显示 miss_cost_weights 权重预览（"低代价优先移除"）

### 2.3 权重配置方案

三种权重的使用场景和配置入口：

| 权重 | 使用场景 | 语义 | 排序方向 | 配置入口 |
|------|---------|------|----------|---------|
| `desire_weights` | 前进法、GDR加权满意度 | 渴望程度 | 降序（高→低） | 配置页「权重」标签页 |
| `miss_cost_weights` | 后退法、GDR加权满意度 | 遗憾代价 | 升序（低→高） | 配置页「权重」标签页 |
| `card_value_weights` | GDR总卡牌价值 | 单卡价值 | — | 配置页「权重」标签页 |

**结论**：三种权重已全部在配置页的权重表中可编辑，合并面板不需要新增权重配置入口。合并面板仅需**展示**当前权重排序预览，让用户确认前进/后退法的排序顺序是否符合预期。

#### 2.3.1 权重预览 UI

在"最多目标卡"模式下，根据选中的前进法/后退法动态显示一张只读预览表：

```
┌─ 权重排序预览（后退法：低代价优先移除）─────────────┐

│ 1     │ card_a  │ 0.3     │ ⬅ 最先移除（最不重要） │
```

#### 2.5.3 实现方式

使用 `QStackedWidget` 在结果区域按搜索模式切换子面板，而非在一个面板中用 if/else 控制可见性。三种结果子面板：

- `ResourceResultWidget` — 二分步骤表 + 趋势图（从 ResourceSearchPanel 迁移）
- `TargetResultWidget` — 卡增删步骤表 + 成功率阶梯图（新写，约150行）
- `ParetoResultWidget` — 方案表 + Pareto图（从 RetreatSearchPanel 迁移）

`TargetResultWidget` 内部根据前进法/后退法切换列的语义标签（"移除"/"添加"、排序方向）。

### 2.6 优势

- **消除约 60% 代码重复**：二分搜索逻辑、模拟调用、进度报告统一
- **前进法填补功能空缺**：当前只有后退法，前进法提供互补的决策视角
- **资源搜索面板获得 Pareto 能力**：当前仅支持最少资源，合并后可切换模式
- **统一的进度报告**：条目2修复后，所有模式进度条从 0→100 不回退
- **单一代码路径**：新增 GDR 指标或策略时只需在一处适配

### 2.7 风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|----------|

- [ ] **A.2: 在 `RetreatSearchEngine` 中实现 `search_max_targets_forward()`**

```python
def search_max_targets_forward(self, candidate_specs: Dict[str, int],
                                desire_weights: Dict[str, float],
                                base_resource: float) -> ForwardResult:
    """前进法：从空开始，按渴望度降序逐个添加卡
    
    candidate_specs: 候选目标卡 {card_id: quantity}
    desire_weights: 渴望度权重（降序=越想要越先加）
    base_resource: 固定资源量
    """
    sorted_cards = sorted(candidate_specs, 
                         key=lambda cid: desire_weights.get(cid, 0), reverse=True)
    current_targets = {}
    steps = []
    
    for card_id in sorted_cards:
        candidate = {**current_targets, card_id: candidate_specs[card_id]}
        prob = self._simulate_with_resource(candidate, base_resource)
        if prob < self.success_threshold:
            break  # 添加此卡后成功率不足，停止
        current_targets = candidate
        steps.append(ForwardStep(
            added_card_id=card_id,
            current_targets=dict(current_targets),
            success_prob=prob,
        ))
    
    return ForwardResult(
        steps=steps,
        final_targets=current_targets,
        final_success_prob=steps[-1].success_prob if steps else 1.0,
        base_resource=base_resource,
    )
```
