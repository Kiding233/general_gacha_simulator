## 业务问题

在资源有限的情况下，玩家需要确定「最多能抽到多少张不同的目标卡」——即找到一个目标卡子集，使得成功率不低于某个阈值（如 80%）。这是一个组合优化问题：在 N 张候选目标卡中搜索最大可行子集。

最多目标卡面板提供两种搜索策略（前进法/后退法），通过迭代模拟评估每个子集的成功率，最终输出最大可行目标集的卡片清单和预期成功率。

## 设计意图

### 核心抽象

- **StrategyPanel**（`gui/strategy_panel.py`，672 行）：QWidget，提供搜索方法选择、GDR 阈值配置、权重表编辑（desire/miss_cost/card_value）、目标卡列表等控件。通过 `StrategyWorker` QThread 执行搜索。
- **StrategyWorker**（内联 QThread）：调用 `_forward_method()` 或 `_backward_method()`，每轮迭代触发一次并行模拟，通过 `progress` 信号更新进度。
- **ForwardResult / BackwardResult**（`core/forward_backward.py`）：封装搜索过程的步骤历史和最终结果。
- **ForwardStep / BackwardStep**：记录每步添加/移除的卡片、当前目标集和对应的成功率。

### 两种搜索算法

| 方法 | 方向 | 排序依据 | 停止条件 |
|------|------|---------|---------|
| 前进法 | 空集 → 逐步添加 | 渴望度（desire）降序 | 成功率跌至阈值以下 |
| 后退法 | 全集 → 逐步移除 | 未命中成本（miss_cost）升序 | 成功率升至阈值以上 |

### 数据流

```
用户配置（目标卡列表 + 权重 + GDR 阈值）
  → StrategyWorker._forward_method() / _backward_method()
    → 每轮迭代:
      → SimulationEnvBuilder.from_config_store(config_store)
      → run_batch_parallel() (每次 N 次模拟)
      → compute_success_probability(histories, specs, gdr_key, threshold)
      → 记录 ForwardStep / BackwardStep
  → ForwardResult / BackwardResult
  → 图表渲染（Plotly 条形图 / 搜索过程阶梯图）
```

## 技术实现

### 前进法（_forward_method）

1. 按 desire 权重降序排列候选目标卡
2. 初始当前集为空
3. 按顺序添加卡片，每步运行 N 次模拟评估成功率
4. 若成功率 >= 阈值，记录为新的有效集
5. 若成功率 < 阈值，停止——上一次有效集即为最大可行集

### 后退法（_backward_method）

1. 按 miss_cost 权重升序排列候选目标卡
2. 初始当前集为全部候选卡
3. 先评估初始全集的成功率，若已 >= 阈值则直接返回
4. 按顺序移除卡片，每步运行 N 次模拟评估成功率
5. 若成功率 >= 阈值，停止——当前集即为最小可放弃集（等价于最大可行集）
