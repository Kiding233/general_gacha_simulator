# 最多目标卡面板 -- 理论

> 创建日期：2026-06-02（首次建档）
> 关联模块：`gui/strategy_panel.py`、`core/forward_backward.py`

---

## 一、核心假设

### 1.1 单调性假设（Monotonicity）

前进法和后退法均依赖目标集大小的单调性假设：

- **前进法**：添加目标卡不应降低成功率（更多目标 → 满足更严格的标准 → 成功率单调不减）。但实际上，添加一张低权重、高竞争的目标卡可能分散策略资源，导致原本可达成的目标卡反而未能获得——成功率可能不升反降。
- **后退法**：移除目标卡不应降低成功率（更少目标 → 满足更宽松的标准 → 成功率单调不减）。这一假设更强——在实际的策略模拟中，当全体目标卡均与策略搜索结果关联时可能是成立的（因为减少约束不会使问题变难），但不能保证100%成立。

**违反单调性的场景**：
- 策略的「按需追卡」（smart）模式：添加新目标卡可能改变策略在不同池之间的资源分配，导致每个池的抽卡次数减少，反而降低所有目标卡的达成概率
- 保底计数器跨池共享：添加目标卡导致的额外抽卡可能意外触发共享保底计数器，使其他池的概率上升（前进法提升成功率）或下降（占用保底触发机会）

### 1.2 渴望度/未命中成本排序的合理性

前进法按 desire 降序添加、后退法按 miss_cost 升序移除。这两个排序假设：

1. **desire 反映优先级**：desire 越高的卡越应优先保留
2. **miss_cost 反映不可放弃性**：miss_cost 越低的卡越应优先移除

此假设源自消费理论（Lancaster, 1966）中的属性偏好排序——在没有预算约束下的最优选择。但在有预算约束（有限的抽卡资源）下，最优排序可能还需要考虑各卡池的成本、池子开启时段、保底概率等因素。

### 1.3 模拟评估的无偏性

每轮搜索迭代运行 N 次独立模拟并计算成功率 `p̂ = successes / N`。假设：

1. **N 次模拟对当前目标集的无偏估计**：`E[p̂] = p_true`
2. **迭代间独立**：不同轮次的模拟使用独立随机种子（seed=0 + counter）

但由于每轮迭代评估不同的目标集，而目标集之间是嵌套关系（前进法的当前集是上一轮的严格超集），迭代间存在序列相关性——如果一轮碰巧高估了成功率（由于采样误差），可能导致过早停止（前进法过早停止于过小的目标集）。

## 二、算法分析

### 2.1 前进法复杂度

设候选卡数为 M，每步迭代次数为 N：

- 时间复杂度：O(M × N)（每步运行一次 N 次模拟的批量）
- 最坏情况迭代次数：M（每张卡都使成功率保持 >= 阈值）
- 最好情况迭代次数：1（第一张卡就使成功率 < 阈值）

### 2.2 后退法复杂度

与前进法相同：O(M × N)。但后退法的初始迭代运行在全集上（目标数量最多），该次模拟的计算成本（每轮模拟中的策略决策）可能高于后续迭代。策略决策复杂度与目标卡数量正相关。

### 2.3 错误源

| 错误类型 | 前进法 | 后退法 |
|----------|--------|--------|
| 采样误差导致过早停止 | 高（可能因一次低估而停止） | 低（必须在全集成功率高估时才停止） |
| 单调性违反导致次优解 | 中（可能跳过可行卡） | 低（移除卡后成功率升高是期望行为） |
| 排序偏差 | 高——依赖 desire 排序质量 | 中——依赖 miss_cost 排序质量 |
| 局部最优陷阱 | 无（贪婪算法，非全局搜索） | 无 |

### 2.4 与全局搜索的比较

前进/后退是贪婪启发式，非全局最优搜索（如穷举所有 2^M 个子集）。在单调性假设成立且排序合理时，贪婪解与全局最优解一致。在单调性违反时，可能存在更大的可行目标集但被贪婪策略错过。

## 三、已知局限性

### 3.1 无全局搜索模式

当前仅支持贪婪前进/后退，无分支定界、动态规划或整数规划等全局优化方法。对于 M>10 的场景，穷举不现实，但中等规模（M=6~10）可受益于有限的剪枝搜索。

### 3.2 速度-准确性权衡硬编码

每轮迭代的模拟次数 N 由用户在面板中设置（默认 500），对全部迭代使用相同值。早期迭代（小目标集）可能不需要与后期迭代相同的样本量，因为成功率的变化幅度更大。自适应样本量分配（如多臂老虎机算法）可更高效利用计算资源。

### 3.3 与 P8 面板合并的设计变迁

当前两个独立面板（最多目标卡 / 最少资源）共享核心搜索思路（迭代模拟 → 成功率评估），但 UI 和 Worker 代码独立维护。P8 面板合并计划将二者整合为统一的多目标搜索面板，消除重复代码。
