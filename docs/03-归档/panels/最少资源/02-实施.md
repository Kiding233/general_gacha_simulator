## 三、ResourceSearchWorker 两阶段搜索

### 阶段一：翻倍搜索上界

```
1. r_hi = initial_resource_hint (或 cost_per_draw × 100 如果 hint ≤ 0)
2. prob = _simulate_with_resource(r_hi)
3. while prob < threshold:
     a. r_hi *= 2
     b. prob = _simulate_with_resource(r_hi)
     c. 记录 ResourceSearchStep(phase='搜索上界')
     d. if 翻倍次数 > 15: 终止(MaxIterations)
4. 阶段一完成，r_hi 确立为可行上界
```

### 阶段二：二分搜索

```
1. r_lo = resource_lo (用户指定的下界，默认 0)
2. epsilon = cost_per_draw × precision_draws
3. while r_hi - r_lo > epsilon:
     a. r_mid = (r_lo + r_hi) / 2.0
     b. prob = _simulate_with_resource(r_mid)
     c. if prob >= threshold:
          r_hi = r_mid (phase='二分(满足)')
        else:
          r_lo = r_mid (phase='二分(不足)')
     d. 记录 ResourceSearchStep
     e. if 二分迭代数 > max_iterations: break
4. 返回 ResourceSearchResult(min_resource=r_hi, ...)
```

## 四、关键细节

### _simulate_with_resource(resource_value)

```python
# 核心：覆盖 draw_resource，其余资源值保持不变
ir = dict(self._initial_resources_backup)
ir['draw_resource'] = resource_value
histories = run_batch_parallel(
    env=self._sim_env,
    target_specs=self.target_specs,
    initial_resources=ir,
    ...
)
return compute_success_probability(histories, ...)
```

每次调用 `_simulate_with_resource()` 运行完整的 N 次并行模拟。这导致每步搜索迭代的开销 = N × T（T 为单次模拟的期望抽数）。

### cost_per_draw 提取

两个静态方法 `_extract_cost_per_draw(pools)` 和 `_extract_cost_per_draw_from_entries(pool_entries)` 提供了从两种不同数据结构提取单抽消耗的路径：
- `pools`：模拟环境中的 `Pool` 对象列表（`p.cost` 可能是 list/dict/int）
- `pool_entries`：`ConfigStore` 中的 `PoolEntry` 列表（`pe.cost` 为字符串如 `draw_resource:160`）

两个方法均只返回第一个匹配值（已知问题，CLAUDE.md #3）。
