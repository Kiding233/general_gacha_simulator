## 业务问题

在给定目标卡清单的情况下，玩家需要知道「最少需要多少初始资源才能达到指定的成功率阈值」。这与最多目标卡面板是对偶问题：给定资源找最多目标 vs 给定目标找最少资源。

最少资源面板使用两阶段搜索（上界翻倍 + 二分搜索），在每个候选资源值上运行 N 次模拟评估成功率，最终返回满足成功率阈值的最小初始资源量。

## 设计意图

### 核心抽象

- **ResourceSearchPanel**（`gui/resource_search_panel.py`，647 行）：QWidget，提供搜索参数控件（精度、初始上界、下界、最大迭代次数）、GDR 选择、目标卡权重配置。通过 `ResourceSearchWorker` QThread 执行搜索。
- **ResourceSearchWorker**（内联 QThread）：实现两阶段搜索——阶段一翻倍搜索上界、阶段二二分搜索最小资源。
- **ResourceSearchStep / ResourceSearchResult**（`core/forward_backward.py`）：封装每步搜索的状态和最终结果。与 `ForwardStep`/`ForwardResult` 共享同一文件。
- **`_extract_cost_per_draw()`**：从模拟环境的池子列表中提取单抽消耗（资源/抽），用于计算 epsilon（搜索精度以抽数为单位）。

### 两阶段搜索算法

| 阶段 | 算法 | 目的 | 停止条件 |
|------|------|------|---------|
| 阶段一：搜索上界 | 翻倍法（exponential doubling） | 找到使成功率 >= 阈值的资源上界 | prob >= threshold 或翻倍超过 15 次 |
| 阶段二：二分搜索 | 经典二分法 | 在 [lo, hi] 区间内找到满足阈值的最小资源值 | hi - lo <= epsilon（精度） 或迭代超过 max_iterations |

### 数据流

```
用户配置（目标卡 + GDR 阈值 + 精度 + 初始上界）
  → ResourceSearchWorker.run()
    → 阶段一（翻倍）:
      → _build_simulation_env()
      → _simulate_with_resource(r) × n   (r *= 2 直到 prob >= threshold)
      → 上界 r_hi 确立
    → 阶段二（二分）:
      → while r_hi - r_lo > epsilon:
        → r_mid = (lo + hi) / 2
        → _simulate_with_resource(r_mid)
        → if prob >= threshold: hi = mid else: lo = mid
    → ResourceSearchResult(min_resource=r_hi, steps, ...)
  → 图表渲染（Plotly 搜索过程图 + 成功率 vs 资源曲线）
```

## 技术实现

### _simulate_with_resource(r) 核心方法

```
1. 复制初始资源字典: ir = dict(self._initial_resources_backup)
2. 覆盖 draw_resource: ir['draw_resource'] = r
3. run_batch_parallel(env, target_specs, initial_resources=ir, ...)
4. compute_success_probability(histories, ...)
5. 返回成功率
```

### 精度控制

`epsilon = actual_cost_per_draw * precision_draws`

- `precision_draws` 默认 1.0 → epsilon = 单抽消耗（如 160 资源）
- `precision_draws` = 0.5 → epsilon = 半抽消耗（如 80 资源）
- 二分搜索在 `r_hi - r_lo <= epsilon` 时终止

### 已知问题

`_extract_cost_per_draw()` 和 `_extract_cost_per_draw_from_entries()` 均遍历池列表但只返回第一个有效成本值（break on first match）。多池且各池成本不同时，使用的 epsilon 可能不准确。此问题在 CLAUDE.md 已知问题 #3 中记录。
