# 统计方法外部库替代实施方案

> 日期：2026-05-28 | 状态：已完成

## Context

项目中大量统计方法从零手写（Bootstrap 引擎 415 行、Wilson CI、以及计划 P19 中的 11 种假设检验），而 Python 生态中已有久经检验的成熟替代方案。当前依赖仅 `PyQt6`、`numpy`、`matplotlib` 三个，连 `scipy` 都未声明——但 `bootstrap.py` 内部实际已在 import `scipy.stats`（隐藏的未声明依赖）。

核心发现：
- `BootstrapEngine` 已完整实现但**无任何生产代码调用**（P3 状态：核心完成，UI 未接入）
- `wilson_ci()` 已正确实现且被 GUI 使用，Laplace 平滑已删除
- `scipy.stats.bootstrap`（SciPy 1.7+）默认 BCa 方法，可直接替代 ~60% 的 bootstrap.py
- `bootstrap.py` 中的厚尾专用方法（m-out-of-n、GPD 尾部、Hill 估计量、TVD）是 scipy 不具备的独有功能，需保留

## 目标

1. 将 `scipy` 声明为正式依赖，消除隐藏依赖
2. Bootstrap 标准路径改用 `scipy.stats.bootstrap`，削减手写代码
3. 保留 scipy 不具备的厚尾专用方法
4. 为 BootstrapEngine 补充测试
5. Wilson CI 保持现状（14 行、正确、已测试，不值得为它引入 statsmodels）
6. 为 P19 假设检验 + P4 EVT 提供外部库使用指南

## 不改的部分

| 模块 | 原因 |
|------|------|
| `wilson_ci` | 14 行公式，正确且已测试，statsmodels 替代收益为零 |
| m-out-of-n / GPD 尾部 / Hill / TVD | scipy 无等价功能 |
| `EmpiricalDistribution` | 对 numpy 的合理封装，非造轮子 |
| 脆弱性分析核回归 | P12 已优化为纯 numpy 闭式解，外部库无更好替代 |
| 抽卡领域专有逻辑 | GDR、事件推断、保底、策略——无外部替代 |

## 实施方案

### 步骤 1：添加 scipy 依赖

**文件**：`pyproject.toml`

```diff
 dependencies = [
     "PyQt6>=6.0",
     "numpy>=1.20",
     "matplotlib>=3.5",
+    "scipy>=1.10",
 ]
```

选择 `>=1.10` 因为 `scipy.stats.bootstrap` 在 1.7 引入、1.10 后 BCa 稳定。

### 步骤 2：重构 `bootstrap.py` —— 标准路径委托给 scipy

**文件**：`gacha_simulator/core/bootstrap.py`

**`bootstrap_probability` 重构**：
- 用 `scipy.stats.bootstrap(data, np.mean, method='BCa')` 替代手写的 `_resample_indices` + `_compute_bca` + `_percentile_ci` 流程
- 空数据守卫保持不变（scipy 空数据会抛异常，需自行处理）
- 边界情况（point_est = 0 或 1）scipy BCa 内部会自行回退

**`bootstrap_mean` 重构**：
- 非厚尾路径同上，委托给 `scipy.stats.bootstrap(data, np.mean, method='BCa')`
- `auto_heavy_tail=True` 路径保持不变——这是 scipy 不具备的独有逻辑

**`bootstrap_quantile` 重构**：
- 非 GPD 路径用 `scipy.stats.bootstrap(data, lambda x: np.quantile(x, q), method='percentile')`
- GPD 路径保持不变

**可删除的私有方法**：
- `_resample_indices` —— scipy 内部处理
- `_percentile_ci` —— scipy 内部处理
- `_compute_bca` —— scipy 内部处理（删除约 60 行）

**保留的方法**：
- `_bootstrap_tail_gpd` —— scipy 无等价
- `_bootstrap_mean_m_out_of_n` —— scipy 无等价
- `_select_m` —— 配套 m-out-of-n
- `hill_estimator` / `_select_hill_k` —— scipy 无等价
- `detect_heavy_tail` —— scipy 无等价
- `total_variation_distance` —— 独立工具

**导入变更**：
- 模块顶部 `from scipy.stats import bootstrap as _scipy_bootstrap`
- 移除内部延迟 import（`from scipy.stats import norm`、`from scipy.stats import genpareto`）改为顶部导入

### 步骤 3：BootstrapResult 适配

scipy 返回的 `BootstrapResult` 包含 `confidence_interval`（tuple）、`standard_error`、`bootstrap_distribution`。需写一个简单的适配层，将 scipy 结果转换为项目的 `BootstrapResult`：

```python
@staticmethod
def _from_scipy_result(scipy_res, point_est, method, n, B) -> BootstrapResult:
    ci_low, ci_high = scipy_res.confidence_interval
    return BootstrapResult(
        point_est, ci_low, ci_high,
        scipy_res.standard_error, method, n, B
    )
```

### 步骤 4：添加 BootstrapEngine 测试

**新文件**：`tests/core/test_bootstrap.py`

测试覆盖：
- `bootstrap_probability`：正常二分类数据 / 全成功 / 全失败 / 空数据 / BCa vs 百分位
- `bootstrap_mean`：正常连续数据 / 空数据 / auto_heavy_tail 触发 m-out-of-n
- `bootstrap_quantile`：中位数 / 低分位数 + GPD 路径 / 空数据
- `hill_estimator`：正态数据（α 应较大）/ 厚尾数据（α < 2）
- `detect_heavy_tail`：正常 vs 厚尾判定
- `total_variation_distance`：相同分布 / 不同分布
- `BootstrapResult.format_ci`：格式字符串验证

### 步骤 5：文档 —— P19 假设检验 + P4 EVT 外部库使用指南

在 `docs/比较分析面板重构与结果数据管理层设计.md` 中补充一节「外部库替代手写」，列出：

**P19 假设检验与比较分析**：

| 计划手写 | 成熟替代 | 所属库 |
|---------|---------|--------|
| KS 检验 | `scipy.stats.ks_2samp` | scipy |
| Mann-Whitney U | `scipy.stats.mannwhitneyu` | scipy |
| 置换检验 | `scipy.stats.permutation_test` | scipy (1.8+) |
| 配对 t 检验 | `scipy.stats.ttest_rel` | scipy |
| Wilcoxon 符号秩 | `scipy.stats.wilcoxon` | scipy |
| Cohen's d | `pingouin.compute_effsize(efsize='cohen-d')` | pingouin |
| 多重比较校正 (Holm/BH) | `statsmodels.stats.multitest.multipletests` | statsmodels |
| Wilson CI | `statsmodels.stats.proportion.proportion_confint(method='wilson')` | statsmodels |
| 贝叶斯因子 | `pingouin.bayesfactor_ttest` | pingouin |

**P4 EVT 极值理论与尾部拟合**：

| 计划手写 | 成熟替代 | 对比 |
|---------|---------|------|
| GPD 分布拟合 (MLE) | `pyextremes` 或 `QuantLite` | 见下方详细对比 |
| GEV 分布拟合 | `pyextremes`（BM+POT，MLE+MCMC+L-moments）或 `QuantLite`（GPD+GEV+Hill+POT） | |
| VaR / CVaR 解析公式 | `QuantLite.risk.metrics`（历史/Para/Cornish-Fisher 三种 VaR + CVaR + 尾比） | pyextremes 仅返回水平，QuantLite 直接给 VaR/CVaR |
| Hill 尾部指数 | 已有 DIY（`bootstrap.py`），`QuantLite.risk.evt` 也内置 | DIY 已可用，外部库作为替代选项 |
| 厚尾分布（Student-t, Lévy stable） | `QuantLite.distributions.fat_tails` | 项目当前无此需求 |

**pyextremes vs QuantLite 对比**：

| 维度 | pyextremes | QuantLite |
|------|-----------|-----------|
| 成熟度 | 252 stars，经典教材（Coles 2001）实现 | 2025 年新包，较新 |
| 拟合方法 | MLE + MCMC(Emcee) + 矩法 + L-moments | MLE |
| VaR/CVaR | ❌ 不直接提供（需手动从拟合分布计算） | ✅ 内置 `tail_risk_summary()` |
| Hill 估计量 | ❌ | ✅ |
| 厚尾分布族 | ❌ | ✅ Student-t, Lévy stable, Kou jump-diffusion |
| 非平稳 EVT | ❌ | ❌（nsEVDx 支持，但项目不需要） |
| 推荐场景 | 经典平稳极值分析，学术严谨性优先 | EVT+风险度量一体化，工程便利性优先 |

**建议**：如果 P4 EVT 部分推进，优先评估 **pyextremes**（成熟稳定，但 VaR/CVaR 需从拟合分布手动计算）。当前项目的 Hill 估计量和 GPD 尾部 Bootstrap 已可覆盖基本需求，外部库作为未来可选升级路径记录。
