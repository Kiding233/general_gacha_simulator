> 这些计划的文档还不够完整，直接写代码会踩坑。先把设计讨论清楚再动手。

### 1.1 阻塞下游的关键决策

#### [讨论] P8 面板合并 —— 方案 A vs 方案 B

**要决定什么**：大三合一（方案A：StrategyPanel + ResourceSearchPanel + RetreatSearchPanel）还是小二合一（方案B：仅前两者，RetreatSearchPanel 保持独立）。

**背景**：方案A净减~1000行但回归风险高，方案B净减~700行更保守。`docs/面板合并可行性计划.md` v6 推荐方案A，同时将方案搜索和脆弱性分析均改为一级Tab消除子Tab嵌套。

**阻塞**：无。自身是独立计划。
**涉及文件**：`docs/面板合并可行性计划.md`

---

#### [讨论] P19 比较分析面板 —— 统计方法与可视化形式

**要决定什么**：
- **Phase2 统计方法**：置换检验 / KS / MWU / Cohen's d / 随机占优(FSD/SSD) —— 哪些纳入 MVP？默认行为是什么？
- **Phase3 可视化形式**：帕累托散点图 / 效应量热力图 / 雷达图 —— 哪些是 MVP？哪些后续？
- **Phase1 UI 形态**：数据管理视图是「独立Tab」还是「合入批量模拟面板」？

**背景**：架构方向（ResultStore、增量缓存、模拟与分析分离）已基本确定，但 Phase2/Phase3 的具体方法均为提案性质。`docs/比较分析面板重构与结果数据管理层设计.md` 第3版标注「待进一步讨论」。

**阻塞**：P19 Phase2/Phase3 的设计依赖这些决策。
**涉及文件**：`docs/比较分析面板重构与结果数据管理层设计.md`
