## 1. 设计目标

将当前 `strategy_comparison_panel.py`（331行，单 Worker 跑多策略 → 5 个朴素指标 → QTableWidget）重构为通用的「比较分析面板」：

1. **数据管理先行**：ResultStore 集中管理所有模拟结果，支持命名、持久化、导入/导出
2. **比较前透明检查**：可比性指纹系统展示数据集差异，用户决定比什么
3. **多层递进分析**：EDA → 占优 → 检验 → 前沿 → 偏好，每层独立可折叠
4. **GDR 联动**：切换 GDR 指标时全部矩阵和图表即时刷新
5. **跨面板数据整合**：比较不仅限于 GDR，还纳入过程分析、脆弱性等的产出

## 2. 可比性指纹系统

### 2.1 数据模型

```python
@dataclass
class ComparabilityFingerprint:
    """模拟完成时自动提取，随 StoredDataset 持久化"""
    config_hash: str           # pools + pity + schedules 的 hash
    strategy_name: str
    target_cards: dict         # {card_id: quantity}
    initial_resources: dict    # {resource_id: amount}
    stop_condition: str        # 停止条件 display_name
    seed_start: int
    seed_end: int
    num_simulations: int
    pool_ids: tuple            # 池子 ID 列表（有序）
    created_at: str            # ISO 时间戳
```

### 2.2 差异矩阵（数据集作为行）

用户在数据管理面板勾选 2+ 数据集后，左侧可比性面板展示：

```
                        策略       目标卡      初始资源     停止条件      种子       N      池子
smart_0602              smart     {1001:1,...}  抽卡:1000  all_pools_end  1-1000   1000   pool_1,pool_2
pool_quota_0602         pool_quota {1001:1,...} 抽卡:1000  all_pools_end  1-1000   1000   pool_1,pool_2
                         ⚠不同      ✓            ✓          ✓             ✓        ✓      ✓
```

### 2.3 差异语义标注

| 差异维度 | 标注 |
|---------|------|
| 策略不同 | 「策略比较——同一环境下策略的选择差异」 |
| 目标卡不同 | 「各策略在各自目标集上评估，非同一目标下的优劣」 |
| 配置不同 | 「不同配置对比——同一策略在不同环境下的鲁棒性」 |
| 种子不同 | 「无法配对比较，使用分布比较」 |
| N 不同 | 「样本量不同影响检验功效」 |
| 全部相同 | 「比较的是蒙特卡洛噪声幅度，适合评估 N 是否足够及结果可复现性」 |

### 2.4 可比性判定

- **仅策略不同**：策略比较模式
- **仅配置不同 + 策略相同**：不同配置对比模式
- **配置不同 + 策略不同**：标注「多重差异，结论需谨慎归因」
- **全部相同**：变异分析模式

## 4. ComparisonAnalyzer 五层递进

### 概览

```
L1 描述    →  描述统计 + ECDF/PMF 叠加（必定执行，轻量）
L2 占优    →  FSD/SSD/TSD 三矩阵（DD Bootstrap，默认执行）
L3 检验    →  KS/MWU/t-test + 三 p 值矩阵（默认执行）
L4 前沿    →  均值-CVaR 或 双 GDR 帕累托前沿（用户触发）
L5 偏好    →  均值-λ·CVaR 排序 + λ 滑块（暂缓实现）
```

**核心控件**：GDR 选择器位于面板顶部，切换时 L1/L3/L4 即时刷新（< 0.1s）。L2（DD Bootstrap）后台计算后更新（~2s/对，可并行），更新前显示旧 GDR 结果并标注「重新计算中」。

### 4.1 L1：探索性分析

#### 描述统计表

| 数据集 | 均值 | 中位数 | 标准差 | 偏度 | CVaR₀.₀₅ | 成功率 | min | max |
|--------|------|--------|--------|------|-----------|--------|-----|-----|
| A smart | 0.87 | 0.91 | 0.18 | −0.42 | 0.41 | 72% | 0.12 | 1.00 |
| B pool_quota | 0.79 | 0.83 | 0.22 | −0.18 | 0.35 | 65% | 0.08 | 1.00 |
| ... | | | | | | | | |

统计量来源：
- **GDR 统计**（来自 analysis_panel）：mean / median / std / skewness / VaR / CVaR
- **成功率**（来自 process_analysis BB）：整体成功率
- **脆弱性摘要**（来自 retreat）：per-pool failure_rate

#### 可视化

- **ECDF 叠加图**：所有数据集的 CDF 曲线叠加，交叉点高亮 → 直观展示随机占优关系
- **PMF 叠加直方图**：离散数据的概率质量函数叠加 → 补充 ECDF 的分布形态信息

删去原计划的箱线图和小提琴图——离散抽卡数据不适合密度估计，且分位数已在描述统计表中展示。

### 4.2 L2：随机占优（DD Bootstrap）

三阶并排显示，每阶输出一个 n×n 矩阵（上半三角）：

```
         FSD (j=1)              SSD (j=2)              TSD (j=3)
      A    B    C    D       A    B    C    D       A    B    C    D
A     —  .004  .67  .001   A  —  .002 .031 .000   A  —  .001 .018 .000
B     —    —  .89  .022    B  —    —  .41 .008    B  —    —  .28 .005
C     —    —    —  .55     C  —    —    — .019    C  —    —    — .014
D     —    —    —    —     D  —    —    —    —    D  —    —    —    —
```

- 绿色 p < .05 → 行策略 j 阶占优于列策略
- 解读：行策略在 j 阶随机优势意义上优于列（p 值来自 DD Bootstrap B=2000）
- FSD ⇒ SSD ⇒ TSD（逐渐宽松）
- 网格点数自适应样本量（`min(200, m+n)`），Bonferroni 校正点态 CI

### 4.3 L3：假设检验

#### 检验方法

| 方法 | 输入 | 输出 | 实现 |
|------|------|------|------|
| KS 检验 | 两个独立样本 | D 统计量 + p 值 | `scipy.stats.ks_2samp` |
| Mann-Whitney U | 两个独立样本 | U 统计量 + p 值 | `scipy.stats.mannwhitneyu` |
| Welch t-test | 两个独立样本 | t 值 + p 值 | `scipy.stats.ttest_ind` |

#### 方向箭头

每个格子加方向指示：`↑` 行优于列、`↓` 行劣于列、`—` 不显著。方向根据 GDR 极性自动判断（`lower_is_better` 取反）。

```
        A          B          C          D
A       —      ↑ .032      — .451     ↑ .003
B       —         —        — .124     ↑ .016
C       —         —          —        ↑ .041
D       —         —          —          —
```

#### 三重 p 值矩阵

- 原始 p 值（不校正）
- Holm-Bonferroni（控制 FWER）
- Benjamini-Hochberg（控制 FDR，推荐默认显示）

用户通过顶部下拉框切换校正方法，矩阵即时更新。

#### 置换检验

当前不替代 KS/MWU/t-test。通过置换 + 不同统计量（mean/median/CVaR/自定义 λ）可统一替代上述检验，但保持现有 scipy 方法以降低实现风险。将来可作为统一入口引入。

### 4.4 L4：帕累托前沿

#### 双 GDR 模式

用户选择 2 个 GDR 指标（X 轴和 Y 轴），每个点 = 一个策略在 (GDR_X_均值, GDR_Y_均值) 的坐标。前沿 = 不被任何其他点同时在两维上支配的点集（右上边界）。

```
X轴: [resource_efficiency ▼]
Y轴: [target_achievement ▼]
```

特殊用法：选同一个 GDR，X=CVaR₀.₀₅，Y=均值 → 传统 mean-CVaR 前沿。

#### 输出

- **散点图**：蓝点 = 前沿策略（非支配），灰点 = 被支配策略，虚线 = 前沿边界
- **非支配集列表**：前沿策略名称 + 坐标值
- **支配关系说明**：「A SSD B：均值 (.87 > .62) + CVaR (.41 > .08)」

### 4.5 L5：偏好排序（暂缓实现）

- 均值-λ·CVaR 排序 + λ 滑块交互
- 确定性等价（CARA/CRRA 效用函数）
- Davidson-Duclos Bootstrap SD 检验的正式 CI（当前用 Bonferroni 点态 CI 替代）

## 5. 跨策略可比较数据矩阵

### 5.1 直接可比的标量指标

来源各面板，可直接填入 L1 描述统计表或作为独立的比较维度：

| 来源面板 | 统计量 | 说明 |
|---------|--------|------|
| analysis_panel | 17 个 GDR 的 mean / median / std / VaR / CVaR | 主要数据源 |
| analysis_panel | GDR × GDR 相关系数矩阵 | 策略间相关结构差异 |
| analysis_panel | 每池 mean_draws / target_rate / pity_rate | 需池对齐 |
| process_analysis (BB) | all_success_prob / all_fail_prob | 整体成败率 |
| process_analysis (BB) | per-pool success_rate | 需池对齐 |
| retreat | overall_failure_rate | 整体脆弱性 |
| retreat | per-pool failure_rate + pity_mean/median | 池结束时状态 |
| resource_search | min_draws = min_resource / cost_per_draw | 最少需要多少抽 |

### 5.2 需转换后可比的指标（暂缓）

| 来源 | 原始数据 | 可提取的标量 | 暂缓原因 |
|------|---------|-------------|---------|
| process AB | P(success \| event) | 条件概率 | 需事件特征标准化 |
| process BA | P(pattern \| success) / P(pattern \| failure) | 诊断优势比 | 同上 |
| retreat | lp_curve | 固定资源点的失败概率 / AUC | 需统一资源网格 |
| retreat | vulnerability_intervals | max_ratio | 依赖 α 和分箱参数 |
| analysis | transition_matrix | SS / FS 概率 | 需池对齐 + success_criterion 统一 |
| analysis | draws_vs_gdr | Spearman ρ | 需决定使用哪个 GDR |
| retreat_search | Pareto 前沿点 | (extra_resource, target_count) | 依赖脆弱性分析结果 |

### 5.3 不可比

- worst_impact 的后续池子模拟（强假设依赖用户自定义配置）
- strategy_panel 的具体 card_id 集合（配置特定）
- 个体轨迹 / 瀑布图（纯可视化产物，非聚合统计）

### 5.4 per-pool 对齐策略

per-pool 数据需池对齐才能比较。对齐优先级：
1. 按 `pool_id` 匹配（同配置下直接匹配）
2. 按位置索引（第 1 池 / 第 2 池 ...）
3. 无法对齐时标注「池结构不同，per-pool 数据不可比」

## 7. 暂缓实现

| 项目 | 理由 | 标记 |
|------|------|------|
| L5 偏好排序（均值-λ·CVaR + λ 滑块） | 需效用参数 UI，用户群体不熟悉 | Phase 3 |

| BA 诊断优势比 | 需先标准化事件特征 | P34/P35 之后 |
| per-pool 跨策略比较 | 需池对齐基础设施 | Phase 2 |
| Q-Q 图 | 非核心需求 | 待评估 |
| 效应量热力图（Cohen's d 矩阵） | 低优先级可视化 | Phase 3 |
| 配对 t 检验 / 反事实遗憾 | 由可比性矩阵驱动，非默认路径 | Phase 3 |
