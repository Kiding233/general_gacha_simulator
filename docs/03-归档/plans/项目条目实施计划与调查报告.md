转变分析（Transition Analysis）的理论定义来自 `理论参考.svg`：

> 根据每个池子后成功、失败的比例，计算每个时间点（抽完池子后）之间成功与失败的转移矩阵。能看何时更容易失败，但是家底剩多少翻身概率不一样。

**现行三种成功判定模式与 GDR 等价性分析：**

| 现行模式 | 判定逻辑 | 计数范围 | 等价 GDR | 范围函数 | 阈值 | 是否等价 |
|---------|---------|---------|---------|---------|------|---------|
| `all_targets` | 池结束时目标卡总数 ≥ 总需求 | 累积 | `all_targets`（已有） | `compute_pool_gdr_cumulative()` | 1.0 | ✅ 完全等价（且 GDR 可修正现有多卡种类 bug） |

| `per_pool_target` | 该池产出 ≥ 1 张目标卡 | 仅该池 | `target_card_draws`（**新建**） | `compute_pool_gdr_single_pool()` | 1 | ✅ 新建后等价 |

**现行 `all_targets` 模式的 bug：**

```python
# analysis_panel.py:299-306
obtained = 0
for i, cid in enumerate(card_ids):
    if times[i] > end_time: break
    if cid in target_ids_trans:
        obtained += 1              # ← 所有目标卡的计数加总
flags.append(obtained >= total_needed)  # ← total_needed = sum(target_specs.values())
```

问题：`{card_A: 2, card_B: 1}` 的需求被「抽到 3 张 card_A、0 张 card_B」满足时，`obtained=3 >= 3` 判成功，但实际 card_B 未获得。GDR `all_targets` 逐卡检查 `card_counts.get(cid, 0) >= qty`，每种卡独立判定，可正确判失败。

**现行实现不显示图片的原因分析：**

`transition_flags` 来自 `DrawSequenceExtractor._update_transition()`（`streaming.py:301`），首行 `if not self._pool_end_times: return`。若初始化时 `pool_end_times` 为空，所有 flags 不会被收集。在 `AnalysisWorker.run()` 中，四层守卫条件（`pool_end_times` 非空 → `transition_flags` 非空 → `n_sims > 0` → `trans` 非空）任何一层失败都静默跳过，用户看不到任何错误提示。

```python
def _gdr_target_card_draws(compact, target_specs, **kw):
    """不去重计数：所有目标卡的出现次数之和。用于 per_pool_target 判定。"""
    return sum(
        compact.get('card_counts', {}).get(cid, 0)
        for cid in target_specs
    )
```

```python
'target_card_draws': GDRDefinition(

    category=GDRCategory.TARGET,
```

```python
def compute_transition_flags_from_gdr(
    cumulative_snapshots: Dict[str, List[Dict]],
    pool_ids_ordered: List[str],
    target_specs: Dict[str, int],
    gdr_key: str = 'all_targets',
    threshold: float = 1.0,
    scope: str = 'cumulative',
    aggregates: List[Dict] = None,
    ssr_ids: Set[str] = None,
    **gdr_kwargs,
) -> List[List[bool]]:
    """通过 GDR 框架逐池判定成功/失败，替代硬编码的 _compute_transition_flags()。

    Args:
        cumulative_snapshots: 池ID → 累积快照列表（scope='cumulative' 时使用）
            结构: {pool_id: [snap_sim0, snap_sim1, ...]}
        pool_ids_ordered: 按结束时间排序的池ID列表
        target_specs: 目标卡需求
        gdr_key: GDR 指标 key
        threshold: 成功阈值
        scope: 'cumulative' → compute_pool_gdr_cumulative()
               'single_pool' → compute_pool_gdr_single_pool()
        aggregates: scope='single_pool' 时需要，每条模拟的聚合数据列表
            每个元素需含 pool_card_counts 字段
        ssr_ids: SSR 卡ID集合（ssr_collection GDR 需要）
    """
    from .process_trace import compute_pool_gdr_cumulative, compute_pool_gdr_single_pool

    if scope == 'cumulative':
        n_sims = len(next(iter(cumulative_snapshots.values()), [])) if cumulative_snapshots else 0
    else:
        n_sims = len(aggregates) if aggregates else 0
    
    if n_sims == 0:
        return []

    flags_per_sim = [[] for _ in range(n_sims)]
    for pool_id in pool_ids_ordered:
        for sim_idx in range(n_sims):
            if scope == 'cumulative':
                snaps = cumulative_snapshots.get(pool_id, [])
                snap = snaps[sim_idx] if sim_idx < len(snaps) else {}
                val = compute_pool_gdr_cumulative(
                    snap, pool_id, target_specs, gdr_key,
                    ssr_ids=ssr_ids, **gdr_kwargs,
                )
            else:  # single_pool
                agg = aggregates[sim_idx] if sim_idx < len(aggregates) else {}
                val = compute_pool_gdr_single_pool(
                    agg, pool_id, target_specs, gdr_key,
                    ssr_ids=ssr_ids, **gdr_kwargs,
                )
            flags_per_sim[sim_idx].append(
                val is not None and val >= threshold
            )
    return flags_per_sim
```

## P13 — 新增GDR：抽数转化效率

**指标名称：** `draw_conversion_efficiency`（显示名：`抽数转化效率`）

**公式：** `实际抽数 / (实际资源消耗 / 每抽资源量)`

即：`draw_pool_draws × cost_per_draw / resource_consumed`

**含义：** 消耗的资源等价为抽数后，与实际抽数的比值。值为 1.0 表示「花多少资源就抽多少次」；**>1.0 表示有送抽/免费抽**（实际抽数多于资源等价抽数，送 20 抽时可达 1.2）；<1.0 表示部分资源消耗在非抽卡行为上（如兑换池）。

**典型场景：**
- 花 16000 资源，每抽 160，抽卡池 100 抽 → `100 × 160 / 16000 = 1.0`（标准）
- 花 16000 资源，但活动送了 20 抽，抽卡池 120 抽 → `120 × 160 / 16000 = 1.2`（送抽高于 1.0）
- 花 16000 资源，50 抽花在兑换池，抽卡池只抽了 50 抽 → `50 × 160 / 16000 = 0.5`

**关键语义：**
- 「实际抽数」仅统计抽卡池（`pool_type='draw'`）的抽数，不含兑换池
- 「实际资源消耗」来自 `total_consumed['draw_resource']`
- 「每抽资源量」从池配置解析（默认 160），作为参数传入
- `lower_is_better=False`（越高越好）

**依赖：** P4（`lower_is_better` 字段必须先实现）。

```python
def _gdr_draw_conversion_efficiency(compact, target_specs, cost_per_draw=160, draw_pool_ids=None, **kw):
    """抽数转化效率 = 实际抽数 / (资源消耗 / 每抽成本)
    
    实际抽数仅统计抽卡池（pool_type='draw'），不含兑换池。
    draw_pool_ids 为抽卡类型的池ID集合，不传时回退到 total_draws。
    """
    resource_consumed = compact.get('total_consumed', {}).get('draw_resource', 0)
    if resource_consumed <= 0 or cost_per_draw <= 0:
        return 0.0
    
    if draw_pool_ids:
        pool_draw_counts = compact.get('pool_draw_counts', {})
        draw_pool_draws = sum(
            cnt for pid, cnt in pool_draw_counts.items() if pid in draw_pool_ids
        )
    else:
        draw_pool_draws = compact.get('total_draws', 0)
    
    return (draw_pool_draws * cost_per_draw) / resource_consumed
```

```python
acquired_before = stats.acquired_counts.get(reward.id, 0)
acquired_after = acquired_before + 1  # stats.on_draw 之后
rg = dict(reward.resources_gained or {})

# 第1次获得此卡 → 额外资源
if acquired_before == 0 and reward.first_time_bonus:
    for k, v in reward.first_time_bonus.items():
        rg[k] = rg.get(k, 0) + v

# 第N次获得此卡 → 额外资源
for nth, bonus in (reward.nth_time_bonus or {}).items():
    if acquired_after == nth:
        for k, v in bonus.items():
            rg[k] = rg.get(k, 0) + v

# 超出N张后再获得 → 额外资源
if reward.excess_bonus:
    threshold = reward.excess_bonus.get('threshold', 999999)
    if acquired_before >= threshold:
        for k, v in reward.excess_bonus.get('resources', {}).items():
            rg[k] = rg.get(k, 0) + v
```
