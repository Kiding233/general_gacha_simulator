# 最差影响分析修复方案

> 日期：2026-05-25 | 修订：v2（2026-05-26：修复 3 搁置、修复 4 实施完成）| 状态：已完成（修复 1、2、4 已实施，修复 3 已搁置）

## 背景

对 `WorstImpactAnalyzer.analyze()` 第二轮模拟路径的审查发现了四个问题，均位于 `gacha_simulator/core/worst_impact.py`。

## 修复清单

### 修复 1：SmartStrategy → DrawTargetStrategy

**位置**：`worst_impact.py` 第 176 行

**当前**：`strategy_name='smart'`

**问题**：SmartStrategy 会做多池比较、资源预留、兑换优先等决策，其行为随版本演进可能变化，导致最差影响分析结果不稳定。

**改为**：使用已实现的 `DrawTargetStrategy`（第 72-96 行），该策略行为确定——"目标池子可用就抽，不可用就等"，不做任何多池决策。

**注意**：`DrawTargetStrategy.__init__` 需要 `target_card_ids: Set[str]` 和 `pool_id: str` 两个参数。需确认 `run_batch_parallel` 的 `strategy_params` 传递机制能否正确注入这两个参数；如不能，需在 `create_strategy()` 中为 `draw_target` 添加参数处理分支。

### 修复 2：max_workers=1 → 动态

**位置**：`worst_impact.py` 第 171 行

**当前**：`max_workers=1`

**问题**：强制串行，20 核机器浪费 95% 算力。

**改为**：`max_workers=os.cpu_count() or 4`

**附加**：需在文件顶部添加 `import os`。

### 修复 3：MAX_POOLS=99 → 动态上限 — ⬜ 已搁置

**搁置日期**：2026-05-26。当前模拟速度已足够快，99 池的构造开销不可感知。详见 `docs/搁置计划记录.md` §7。

<summary>原方案（点击展开）</summary>

**位置**：`worst_impact.py` 第 22 行（常量定义）、第 204 行（使用处）

**当前**：`MAX_POOLS = 99`

**问题**：绝大多数场景实际仅需 0-10 个池子，99 浪费构造开销。且如果运气极好（每池 1 抽出货），99 池天花板会截断真实结果。

**改为**：`MAX_POOLS = ceil(worst_resource / cost_per_draw) + 5`

- `cost_per_draw` 从解析后的池子成本中获取
- `+ 5` 为安全边际（单抽成本估算下，提前出货带来的额外池子数）
- 注意 `MAX_POOLS` 现在是运行时常量（依赖 `worst_resource`），而非模块级常量，需从 `analyze()` 传入 `prepare_simulation_config()`

</details>

### 修复 4：draw_resource 硬编码 — ✅ 已完成（2026-05-26）

**涉及位置**：`worst_impact.py` 5 处

**方案**：新增 `self._resource_name` 属性，在 `_prepare_pool_info()` 和 `_apply_custom_pool_config()` 中从解析后的成本（`self._parsed_cost`）提取资源类型名。`prepare_simulation_config()`、`ConditionalResourceDistribution` 调用处改为引用该属性。

**已修改**：
- `__init__` 新增 `self._resource_name = 'draw_resource'`（默认值）
- `_prepare_pool_info()` 解析成本后提取 `self._resource_name`
- `_apply_custom_pool_config()` 解析成本后提取 `self._resource_name`
- `ConditionalResourceDistribution(..., resource=self._resource_name)`
- `ConsecutivePoolTargetCondition(resource_name=self._resource_name, ...)`
- `'initial_resources': {self._resource_name: worst_resource}`
