# 比较分析面板重构与结果数据管理层设计

> 日期：2026-05-26 | 修订：第3版 | 状态：**待进一步讨论**
> 触发：风险决策理论报告 + 玩家社区策略比较方法论 研究报告启发
>
> **⚠ 注意**：本计划中的具体分析方法（统计检验选择、可视化形式、默认参数等）均为提案性质，尚未最终决定。仅架构层面的方向（ResultStore、增量缓存、模拟与分析分离）已基本确定。

---

## 背景

两份外部研究报告对当前项目提出了系统性的改进方向。本计划聚焦其中**可立即落地的架构改进**——建立通用的结果数据管理层（`ResultStore`），将模拟与分析分离，并重构策略比较面板为「比较分析面板」。

## 问题诊断

1. **数据孤岛**：`MainWindow.simulation_results` 只存单次结果，新模拟覆盖旧数据，无法跨批次比较
2. **模拟与分析紧耦合**：模拟完成后立即全量分发给所有面板，无法事后回顾历史模拟；所有面板始终分析同一批数据，无法切换视角
3. **策略比较面板独立王国**：`StrategyComparisonPanel` 自己跑模拟，与其他面板零交互，只比较 5 个朴素指标
4. **分析结果重复计算**：每次切换或重新打开都要从头计算 GDR、分布、轨迹等，无缓存

## 关键设计决策（第3版修订）

1. **集中式数据管理**：新增「数据管理」视图，选择一个数据集加载到分析面板。**备选方案**：合入批量模拟面板（用折叠面板或内部 Tab 区分「运行模拟」和「已存数据集」），减少顶层 Tab 数量。两种方案均可，实施时评估
2. **模拟与分析分离**：模拟 → 存入 ResultStore → 用户选择加载哪个数据集进行分析
3. **增量计算 + 参数绑定**：不预先计算所有分析。用户在各面板操作时按需触发，计算结果以 `(analysis_type, params_hash)` 为 key 缓存。同参数不重复计算
4. **比较范围不限策略**：可比较不同策略、不同配置、不同初始资源的任意数据集，面板改名为「比较分析面板」
5. **配对/分布双模式自动检测**：系统检测数据集间是否满足同种子+同N，自动在配对模式（反事实遗憾、配对t检验）和分布模式（KS、随机占优）间切换
6. **多重比较校正**：所有假设检验报告原始 p 值 + Holm-Bonferroni（FWER）+ Benjamini-Hochberg（FDR）三组矩阵
7. **废弃灵敏度独立 Tab**：鲁棒性/灵敏度分析作为比较分析的后续可选模式

### StoredDataset

```python
@dataclass
class StoredDataset:

```

### 核心设计变更：增量计算 + 参数绑定

**原设计问题**：「首次加载数据集时计算所有分析」不现实——每个分析有大量参数组合（GDR 指标 × 阈值 × α × 条件 × …），不可能穷举。

**新设计**：计算多少、缓存多少。每次分析计算附带其参数签名（cache key），同一参数组合不重复计算。

```python
@dataclass
class CachedAnalysis:
    """随分析面板使用而渐进填充的缓存"""

    # —— 基础数据（加载时一次性提取，极低成本 O(N) 单次遍历） ——
    per_sim: PerSimValues              # 见下方（<200 KB）

    # —— 按需计算的缓存字典，key = f"{analysis_type}:{params_hash}" ——
    _store: dict[str, Any] = field(default_factory=dict)
```

### 基础数据 `PerSimValues`（始终计算，成本极低）

直接从 aggregate_data 中一次性提取，是所有后续分析的共同输入：

```python
@dataclass
class PerSimValues:
    """从 raw aggregate_data 一次性提取的逐模拟值（O(N) 单次遍历）"""
    total_draws: list[int]                # 每次模拟的总抽数
    pity_triggers: list[int]              # 每次模拟的保底触发次数
    card_counts: dict[str, list[int]]     # card_id → 每次模拟的获得数
    final_resources: dict[str, list[float]]   # resource_name → 每次模拟的最终值
    total_consumed: dict[str, list[float]]    # resource_name → 每次模拟的消耗
    pool_draw_counts: dict[str, list[int]]    # pool_id → 每次模拟的抽数
    target_acquisitions: dict[str, list[int]]  # target_card_id → 每次模拟的获得数
```

### 各分析方法与缓存 Key 的对应关系

每个分析方法被调用时，以 `(analysis_type, params_hash)` 为 key 查找缓存，命中则直接返回，未命中则计算后存入。

#### 统计分析（AnalysisPanel）

| 分析方法 | 参数 | Cache Key 示例 | 计算成本 | 预先计算？ |
|----------|------|---------------|---------|-----------|
| GDR 值分布 | gdr_key | `"gdr:resource_efficiency"` | 轻量（N 次浮点计算） | 按需 |
| 成功率 | gdr_key + threshold + lower_is_better | `"success:target_achievement:1.0:False"` | 极轻 | 按需 |
| VaR/CVaR | metric + alpha | `"var:draw_resource:0.05"` | 极轻（分位数计算） | 按需 |
| 最差条件分析 | cond + gdr_key + threshold + alpha | `"worst_case:failure:target_achievement:1.0:0.05"` | 中等（条件筛选+排序） | 按需 |
| 资源分布 CDF | resource_name | `"cdf:draw_resource"` | 极轻 | 按需 |
| 不抽卡基线 | — | `"no_draw_baseline"` | 极轻 | 首次加载 |

#### 过程分析（ProcessAnalysisPanel）

| 分析方法 | 参数 | Cache Key 示例 | 计算成本 | 预先计算？ |
|----------|------|---------------|---------|-----------|
| 事件轨迹 | success_criterion | `"trace:all_targets"` | 中等（逐池推断） | 按需 |
| 转变矩阵 | success_criterion | `"transition:all_targets"` | 轻量 | 按需 |
| AA/AB/BA/BB | success_criterion + pattern | `"aa:all_targets:sequence"` | 中等 | 按需 |
| 每池 GDR | success_criterion | `"per_pool_gdr:all_targets"` | 中等 | 按需 |

#### 脆弱性分析（RetreatPanel）

| 分析方法 | 参数 | Cache Key 示例 | 计算成本 | 预先计算？ |
|----------|------|---------------|---------|-----------|
| 脆弱区间 | gdr_key + threshold + alpha + num_bins + num_curve + weights | `"vuln:target_achievement:1.0:0.05:50:200"` | **昂贵**（局部回归+同时置信带） | 否——仅在用户触发时 |
| 条件分布 | gdr_key + threshold + lower_is_better | `"cond_dist:resource_efficiency:0.5:True"` | 中等 | 按需 |

#### 最差影响分析（WorstImpactPanel）

| 分析方法 | 参数 | Cache Key 示例 | 计算成本 | 预先计算？ |
|----------|------|---------------|---------|-----------|
| 最差影响 | condition + gdr_key + threshold + alpha + n_sims + pool_duration + distribution | `"worst_impact:failure:target_achievement:1.0:0.05:1000:21:dist_hash"` | **极昂贵**（内部运行独立模拟） | 否——仅在用户触发时 |

### 缓存生命周期

```
加载数据集
  ↓
提取 PerSimValues（一次性，~200KB，始终保留）
  ↓
用户在各分析面板操作 → 触发具体分析方法
  ↓                           ↓
检查缓存（cache_key）   未命中 → 计算 → 存入 _store
  ↓ 命中
直接返回（<1ms）
  ↓
切换数据集时：PerSimValues 替换，_store 清空（或保留在内存等待下次切回）
  ↓
持久化到 JSON：PerSimValues + _store 中所有已计算的分析结果
```

#### 5c. 比较分析引擎（`ComparisonAnalyzer`，在 `core/` 中独立实现）

从各数据集的 `PerSimValues`（及已缓存的 GDR 分布）中提取指标，进行成对比较。

**两种输入模式（自动检测）**：

| 模式 | 检测条件 | 可用方法 |
|------|---------|---------|
| **配对模式** | 所有选中数据集 seed 相同 + N 相同 | 反事实遗憾、配对 t 检验、Wilcoxon 符号秩 |
| **分布模式** | 否则（默认） | KS、MWU、置换检验、随机占优 |

系统自动检测并标注可用的分析模式，用户无需手动选择。

**比较方法清单**：

| 方法 | 输入 | 输出 | 多重比较校正 |
|------|------|------|-------------|
| **一阶随机占优 (FSD)** | A 和 B 的经验 CDF | FSD 矩阵（✓/—） | 不适用（非假设检验） |
| **二阶随机占优 (SSD)** | A 和 B 的经验 CDF 积分 | SSD 矩阵（✓/—） | 不适用 |
| **Kolmogorov-Smirnov** | 两个独立样本 | D 统计量 + 原始 p 值 + **校正 p 值** | ✓ |
| **Mann-Whitney U** | 两个独立样本 | U 统计量 + 原始 p 值 + **校正 p 值** | ✓ |
| **Cohen's d** | 两个独立样本 | d 值 + 置信区间 + 解释标签 | 不适用 |
| **置换检验** | 两个独立样本 + 可选统计量 λ | 置换 p 值 + **校正 p 值** + λ 的置换分布 | ✓ |
| **配对 t 检验** | 配对的差值列表（仅配对模式） | t 值 + p 值 | ✓ |
| **反事实遗憾** | 配对结果（仅配对模式） | 遗憾分布 + E[Regret] | 不适用 |

**多重比较校正**（对所有假设检验方法统一应用）：

m = C(n,2) 对比较。输出三个 p 值矩阵：

| 矩阵 | 含义 | 适用场景 |
|------|------|---------|
| 原始 p 值 | 不做校正 | 用户自行判断 |
| Holm-Bonferroni 校正 | 控制族系错误率 (FWER) | 需要严格确认差异时 |
| Benjamini-Hochberg 校正 | 控制假发现率 (FDR) | 探索性分析（推荐默认显示） |

**置换检验应作为默认方法**：

理由：置换检验可检验任意统计量（均值差/CVaR差/中位数差/成功率差/自定义），仅需可交换性假设，计算量极小（PerSimValues 已有 N=1000 个值，10000 次置换 < 0.1s）。KS/MWU 作为补充参考。

> ⚠ **以下所有具体方法、默认统计量、可视化形式均为提案，待进一步讨论后确定。**

```python
def permutation_test(values_a, values_b, n_perm=10000, statistic=np.mean):
    """
    H0: 两个策略在该统计量上无差异
    返回：观察到的差异、置换 p 值、置换分布
    """
    observed = statistic(values_a) - statistic(values_b)
    pooled = np.concatenate([values_a, values_b])
    n_a = len(values_a)
    permuted_diffs = np.empty(n_perm)
    rng = np.random.default_rng(42)
    for i in range(n_perm):
        rng.shuffle(pooled)
        permuted_diffs[i] = statistic(pooled[:n_a]) - statistic(pooled[n_a:])
    p_value = np.mean(np.abs(permuted_diffs) >= np.abs(observed))
    return observed, p_value, permuted_diffs
```

### P19 假设检验与比较分析

以下 P19 计划中的假设检验方法已有成熟的外部实现，不应从零手写：

| 计划手写 | 成熟替代 | 所属库 |
|---------|---------|--------|
| KS 检验 | `scipy.stats.ks_2samp` | scipy（已引入） |
| Mann-Whitney U | `scipy.stats.mannwhitneyu` | scipy（已引入） |
| 置换检验 | `scipy.stats.permutation_test` | scipy（1.8+，已引入） |
| 配对 t 检验 | `scipy.stats.ttest_rel` | scipy（已引入） |
| Wilcoxon 符号秩 | `scipy.stats.wilcoxon` | scipy（已引入） |
| Cohen's d | `pingouin.compute_effsize(efsize='cohen-d')` | pingouin |
| 多重比较校正 (Holm/BH) | `statsmodels.stats.multitest.multipletests` | statsmodels |
| Wilson CI | `statsmodels.stats.proportion.proportion_confint(method='wilson')` | statsmodels |
| 贝叶斯因子 | `pingouin.bayesfactor_ttest` | pingouin |

**实施建议**：P19 阶段引入 `pingouin` 和 `statsmodels` 作为可选依赖，直接使用上述外部方法，无需手写检验逻辑。Bootstrap CI 已委托给 `scipy.stats.bootstrap`（P23 已完成）。
