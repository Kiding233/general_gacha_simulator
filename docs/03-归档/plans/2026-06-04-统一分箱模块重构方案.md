> 创建日期：2026-06-04
> 状态：就绪（2026-06-04 修订）

## 背景

当前代码库中 GDR 绘图的直方图分箱策略分散在 7 个位置，使用 5 种不同策略，互不知晓：

| 位置 | 策略 | 问题 |
|------|------|------|
| `analysis_panel._hist_params` | 整数检测 + FD | 未用于GDR分布绑图(死代码)；逐整数分箱导致~16001空bin |
| `analysis_panel` GDR分布 | Plotly默认(auto) | 无GDR类型感知，A类有限格点走连续直方图 |
| `plotly_charts._build_histogram` | 无默认nbins | 不感知数据特性 |
| `plotly_charts._build_ridge` | nbins=50 | 硬编码 |
| `vulnerability.py` | 步长检测→对齐→FD | 最完善，但仅用于退路面板 |
| `comparison_analysis_panel` | sqrt(N)*2 | 绕过ChartSpec，直接plotly |
| `process_analysis_panel` | Plotly默认 | 无分箱控制 |

目标：构建 `core/gdr_binning.py`，使所有已有和将要添加的 GDR 绘图自动使用正确的分箱策略。

## 一、模块接口

### 1.1 模块位置与职责

`gacha_simulator/core/gdr_binning.py` —— 唯一职责：给定 GDR key + 样本数据，返回正确的分箱策略。

```
面板层 (gui/*.py)  ──调用──→  compute_bins(gdr_key, samples, ...)
                                    ↓
渲染层 (plotly_charts.py)  ←──  BinningResult.to_layout_hints() 传入 ChartSpec
```

### 1.2 核心数据结构

```python
@dataclass(frozen=True)
class BinningResult:
    """不可变分箱结果——由 compute_bins() 返回。"""

    # ── 分箱参数 ──
    bin_edges: np.ndarray | None       # None = 委托渲染器默认
    bar_mode: bool = False             # True → 柱状图 (go.Bar)
    density: bool = True               # True → 概率密度; False → 频数

    # ── 柱状图专用（bar_mode=True 时） ──
    bar_x: np.ndarray | None = None    # 柱的 x 坐标（= 格点值）
    bar_y: np.ndarray | None = None    # 柱的高度（= 计数）

    # ── 内部使用 ──
    _extra: dict = field(default_factory=dict)  # 额外 layout_hints（nbins 等）

    # ── 元数据 ──
    inf_fraction: float | None = None  # resource_per_card 的 inf 占比
    inf_label: str | None = None       # 图表标注文本，调用方自主决定渲染方式

    def to_layout_hints(self) -> dict:
        """转为可直接合并到 ChartSpec.layout_hints 的字典。"""
        hints = dict(self._extra)
        if self.bin_edges is not None:
            hints["bin_edges"] = self.bin_edges
        if self.bar_mode:
            hints["bar_mode"] = True
            if self.bar_x is not None:
                hints["bar_x"] = self.bar_x
            if self.bar_y is not None:
                hints["bar_y"] = self.bar_y
        hints["density"] = self.density
        return hints
```

### 1.3 唯一入口函数

```python
def compute_bins(
    gdr_key: str,
    samples: np.ndarray,
    *,
    target_specs: dict[str, int] | None = None,
    cost_per_draw: float | None = None,
    use_draw_units: bool = False,
    fixed_edges: np.ndarray | None = None,
) -> BinningResult:
    """为指定的 GDR 指标计算正确的分箱策略。

    自动检测值域类别，返回包含 bin_edges / density / bar_mode 等信息的 BinningResult。
    所有面板的 GDR 直方图绑图都应调用此函数。

    参数：
        gdr_key: GDR 指标 key（UNIFIED_GDR_REGISTRY 中的键）
        samples: 样本值数组
        target_specs: 目标卡规格 {card_id: quantity}
        cost_per_draw: 单抽成本（用于 resource_consumed/remaining 步长检测）
        use_draw_units: 是否将资源值转为抽数单位
        fixed_edges: 外部预计算的统一分箱边界。非 None 时跳过检测，直接使用。
                    用于脆弱性分析保证多个池子使用相同 bin 边界。

    多数据集比较场景：调用方将各组样本合并后传入，bar_mode 时各组共享
    bar_x（合并后格点并集），各自用 bin_edges 计数得到 bar_y。合并后唯一值
    >20 则自然进入连续路径，无需调用方显式指定。
    """
```

## 二、判定逻辑：两层检测 + IQR集中分布保护

不做"按 GDR 类型预设分箱策略"。只看数据本身——数据长什么样，就怎么画。

```
样本数据
  │
  ├── fixed_edges 非 None？
  │     └── 是 → 跳过检测，直接使用传入边界
  │
  ▼
第一层：有多少个不同值？（全量样本）
  │
  ├── ≤ 20 个 → 柱状图（一格点一柱，bar_mode=True）
  │     └── 边界 = 相邻值中点 → 计数 → go.Bar(x=格点值, y=计数)
  │
  └── > 20 个 → 进入第二层
        │
        ▼
第二层：B₁类步长检测（detect_step_size）
  │
  ├── 命中 → 对齐分箱（compute_aligned_bins）
  │     ├── resource_remaining / resource_consumed 的默认路径
  │     └── use_draw_units=True → 值 ÷ cost → round → Δ=1 整数分箱
  │
  └── 未命中 → 第三层
        │
        ▼
第三层：连续分布分箱
  │
  ├── IQR / span < 0.05（高度集中 + 少量离群值）
  │     → nbins = max(50, min(200, n / 20))
  │
  └── 正常 → freedman_diaconis_bins(samples, min_bins=10, max_bins=200)
```

### 2.1 柱状图边界计算（间隔不均一也能正确计数）

```python
def _bins_finite_grid(samples):
    uniq = np.sort(np.unique(samples))
    n = len(uniq)

    if n == 1:
        edges = np.array([uniq[0] - 0.5, uniq[0] + 0.5])
    else:
        edges = np.empty(n + 1)
        edges[0] = uniq[0] - (uniq[1] - uniq[0]) / 2
        for i in range(1, n):
            edges[i] = (uniq[i-1] + uniq[i]) / 2
        edges[n] = uniq[-1] + (uniq[-1] - uniq[-2]) / 2

    counts, _ = np.histogram(samples, bins=edges)
    return BinningResult(
        bin_edges=edges,
        bar_mode=True,
        density=False,
        bar_x=uniq,
        bar_y=counts,
    )
```

例：取值 `[0, 0.3, 0.7, 1.0]` （间距 [0.3, 0.4, 0.3]）→ 每个实际取值一根柱子，x 坐标精确在值上。

### 2.2 步长分箱（B₁类）

```python
def _bins_stepped(samples, cost_per_draw, use_draw_units):
    if use_draw_units and cost_per_draw and cost_per_draw > 0:
        # 转为抽数单位 → round 取整 → 退化为 Δ=1 整数分箱
        scaled = np.round(np.asarray(samples, dtype=np.float64) / cost_per_draw).astype(int)
        lo, hi = int(np.min(scaled)), int(np.max(scaled))
        edges = np.arange(lo - 0.5, hi + 1.5, 1.0)  # 经典整数分箱
        return BinningResult(bin_edges=edges, density=False)

    step = detect_step_size(samples)
    if step is None:
        return _bins_continuous(samples)  # 回退

    edges = compute_aligned_bins(samples, step)
    return BinningResult(bin_edges=edges, density=False)
```

### 2.3 连续分箱（含IQR集中分布保护）

```python
def _bins_continuous(samples):
    n = len(samples)
    if n < 5:
        return BinningResult(bin_edges=None, density=True)

    vals = np.asarray(samples, dtype=np.float64)
    q1, q3 = np.percentile(vals, [25, 75])
    iqr = max(q3 - q1, 1e-9)
    span = vals.max() - vals.min()

    if span > 0 and iqr / span < 0.05:
        # 高度集中但有离群值——FD 会因 IQR≈0 产生过少 bin
        nbins = max(50, min(200, int(n / 20)))
    else:
        from gacha_simulator.core.distribution import freedman_diaconis_bins
        nbins = freedman_diaconis_bins(samples, min_bins=10, max_bins=200)

    return BinningResult(bin_edges=None, density=True,
                         _extra={'nbins': nbins})
```

### 2.4 特殊处理：resource_per_card 的 inf

```python
def _bins_with_inf(samples):
    is_inf = ~np.isfinite(samples)
    inf_count = np.sum(is_inf)
    inf_frac = inf_count / len(samples)

    finite = samples[~is_inf]
    if len(finite) < 5:
        return BinningResult(
            density=False, bar_mode=False,
            inf_fraction=inf_frac,
            inf_label=f"非有限值: {inf_frac:.1%} ({inf_count}/{len(samples)})",
        )

    base = _bins_continuous(finite)
    return BinningResult(
        bin_edges=base.bin_edges,
        density=base.density,
        inf_fraction=inf_frac,
        inf_label=f"inf占比: {inf_frac:.1%} ({inf_count}/{len(samples)})"
                   if inf_frac > 0 else None,
    )
```

### 2.5 compute_bins 主调度

```python
def compute_bins(gdr_key, samples, *, target_specs=None,
                 cost_per_draw=None, use_draw_units=False,
                 fixed_edges=None) -> BinningResult:

    # 0. 外部预计算边界 → 直接使用，跳过全部检测
    if fixed_edges is not None:
        return BinningResult(bin_edges=fixed_edges, density=False)

    vals = np.asarray(samples, dtype=np.float64)

    # 0.5 resource_per_card 的 inf 处理
    if gdr_key == "resource_per_card" and np.any(~np.isfinite(vals)):
        return _bins_with_inf(vals)

    # 1. 第一层：有限格点检测
    uniq = np.unique(vals)
    if len(uniq) <= 20:
        return _bins_finite_grid(vals)

    # 2. 第二层：步长检测（仅对 B₁ 类有意义，但不预设类型——
    #    对任意 >20 唯一值的数据尝试步长检测，命中则对齐分箱）
    if cost_per_draw is not None or use_draw_units:
        stepped = _bins_stepped(vals, cost_per_draw, use_draw_units)
        if stepped.bin_edges is not None:   # 步长命中
            return stepped

    # 3. 第三层：连续分箱
    return _bins_continuous(vals)
```

`gdr_key` 仅用于触发 `resource_per_card` 的 inf 分离路径（以及 `_bins_with_inf` 内部的 inf_label 文案），其余判定全部数据驱动——不按 GDR 类型预设策略。

### 2.6 极端配置的自适应

| 场景 | 数据特征 | 判定路径 | 结果 |
|------|---------|---------|------|
| `resource_efficiency` 10池正常 | 347个唯一值 | >20 → 步长未命中 → FD | 连续直方图 |
| `resource_efficiency` 2池极端 | 6个唯一值 | ≤20 → 柱状图 | 事实离散，画柱状图 |
| `all_targets` | 2个唯一值 {0,1} | ≤20 → 柱状图 | 2柱 |
| `target_achievement` T=200 | 150个唯一值 | >20 → FD | 连续直方图 |
| `target_achievement` T=3 | 4个唯一值 | ≤20 → 柱状图 | 4柱 |
| `resource_remaining` cost=160 | 500个唯一值 | >20 → 步长命中 → 对齐 | 对齐分箱 |
| 高度集中 + 离群值 | IQR/span=0.01 | 步长未命中 → IQR/span<0.05 → n/20 bins | 密集区高分辨率 |
| 多数据集比较（合并后判定） | 各组格点可能不同 | 合并唯一值 >20 → 步长/FD | 若合并后 ≤20 则同格点 bar_mode |

## 四、渲染层改动

### 4.1 `_build_histogram` 增加 bar_mode 分支

面板不判断图表类型。所有面板永远传 `chart_type="histogram"` + `HistogramData`。
渲染层看 `layout_hints["bar_mode"]` 决定用 `go.Bar` 还是 `go.Histogram`。

```python
# plotly_charts.py _build_histogram()

def _build_histogram(self, s: ChartSpec) -> go.Figure:
    d: HistogramData = s.data
    bar_mode = s.layout_hints.get("bar_mode", False)

    if bar_mode:
        return self._build_bar_from_grid(s)

    # 现有 go.Histogram 逻辑不变
    ...

def _build_bar_from_grid(self, s: ChartSpec) -> go.Figure:
    """有限格点柱状图：每个实际取值一根柱子。"""
    bar_x = s.layout_hints.get("bar_x")
    bar_y = s.layout_hints.get("bar_y")
    color = s.layout_hints.get("color", "#1f77b4")

    fig = go.Figure()
    fig.add_trace(go.Bar(
        x=bar_x, y=bar_y,
        marker_color=color,
        hovertemplate="值: %{x:.4f}<br>频数: %{y}<extra></extra>",
    ))

    # 复用现有注解系统（均值线、分位数线等）
    self._apply_annotations(fig, s)

    # 不渲染 HistogramData.overlays——柱状图的 x 轴是格点值，
    # go.Histogram 叠加会错位。若确有叠加需求，调用方应将
    # overlays 数据也转为 bar_x/bar_y 并通过多个 go.Bar trace 实现。
    return fig
```

#### bar_mode 的 overlays 限制

bar_mode 柱状图的 x 轴是格点值（间距不均一），`go.Histogram` 叠加轨迹无法对齐。`_build_bar_from_grid` 忽略 `HistogramData.overlays`。若面板确实需要在柱状图上叠加条件分布，应将条件分布也通过 `compute_bins` 预分箱后作为第二个 `go.Bar` trace 传入。当前最差/最好情形主要关注连续型 GDR，此限制无实际影响。

### 4.2 `_build_ridge` 不改变

山脊线图继续使用 `go.Histogram` + 共享 `xbins`。分箱边界由调用方通过 `layout_hints["bin_edges"]` 传入。

## 五、面板调用模式

### 5.1 统一范式

```python
from gacha_simulator.core.gdr_binning import compute_bins

# 改造前
spec = histogram(samples=samples, title="...", xlabel="...")

# 改造后
bin_result = compute_bins(gdr_key, samples,
                          target_specs=target_specs,
                          cost_per_draw=cost_per_draw,
                          use_draw_units=use_draw_units)
spec = histogram(
    samples=samples,
    title="...",
    xlabel="...",
    density=bin_result.density,
    **bin_result.to_layout_hints(),
)
# inf_label 由调用方自行决定是否展示（直方图标题、ChartAnnotation 或独立标签）
```

#### inf_label 的使用约定

`inf_label` 是元数据，不强制渲染方式。面板侧可根据上下文选择：
- 直方图：拼入 `ChartSpec.title`
- ECDF：不展示（ECDF 在 inf 处自然截断）
- 或作为 `ChartAnnotation` 显示在图表顶部

### 5.2 comparison_analysis_panel 的特殊处理

该面板使用 `make_subplots` 并排 PMF+ECDF，多数据集共享分箱。将各组样本合并后调用 `compute_bins`——合并数据的唯一值是各组唯一值的并集：

- **合并后 ≤20 个唯一值 → bar_mode**：各组格点自然一致（如 `all_targets` 永远是 {0,1}，`target_achievement` 同 T 同格点），共享 `bar_x`，每组各自用 `bin_edges` 计数得到 `bar_y`，叠加为分组柱状图
- **合并后 >20 个唯一值 → 连续路径**：各组格点不一致时自动发生，无需调用方干预。共享 `bin_edges`，各组各自 `go.Histogram`

```python
# 合并后统一判定，而非各组独立判定后强制覆盖
all_vals = np.concatenate([group_samples for group_samples in all_groups])
bin_result = compute_bins(gdr_key, all_vals,
                          target_specs=target_specs,
                          cost_per_draw=cost_per_draw)

if bin_result.bar_mode:
    # 各组用共享 bin_edges 各自 np.histogram → 各得 bar_y
    for i, group_samples in enumerate(all_groups):
        counts, _ = np.histogram(group_samples, bins=bin_result.bin_edges)
        fig.add_trace(go.Bar(x=bin_result.bar_x, y=counts, name=group_names[i]))
else:
    # 连续路径：共享 bin_edges 或 nbins，各组 go.Histogram
    for i, group_samples in enumerate(all_groups):
        fig.add_trace(go.Histogram(x=group_samples, name=group_names[i], ...))
```

## 六、脆弱性分析统一分箱的对齐保证

当前 `vulnerability.py` 的流程（重构后不变）：

```
1. 收集全部池子的所有样本 → _global_all
2. detect_step_size(_global_all) → step
3. compute_aligned_bins(_global_all, step) → global_bin_edges
4. 每个池子 + 山脊线图：
   compute_bins(gdr_key, pool_samples, fixed_edges=global_bin_edges)
   → 所有池子得到完全相同的 bin_edges
```

`detect_step_size()` 和 `compute_aligned_bins()` 从 `vulnerability.py` 移到了 `gdr_binning.py` 成为公开函数。`vulnerability.py` 改为 `import`。底层逻辑一条不变。

`fixed_edges` 参数是保证对齐的关键——非 None 时 `compute_bins` 跳过内部检测，直接使用传入的边界。
