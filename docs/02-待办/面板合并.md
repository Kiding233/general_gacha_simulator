# 面板合并——方案搜索三合一

> 提炼自 `03-归档/plans/面板合并可行性计划.md` | 2026-05-28 v6 | 设计阶段，未实施

## 问题

三个搜索面板（最多目标卡/最少资源/退路分析→方案搜索）共 1,867 行代码做同一件事——迭代运行模拟搜索最优方案。另有三个分散问题：「最多目标卡」Tab名误导、`RetreatPanel` 子Tab嵌套是GUI中唯一子Tab、权重表重复维护。

## 方案

**推荐方案A（大三合一）**：StrategyPanel + ResourceSearchPanel + RetreatSearchPanel → `PlanSearchPanel`

- 统一引擎 `PlanSearchEngine`：`from_pool_id=None`（完整时间线）vs `from_pool_id=str`（退路点）
- 双起点 × 四模式（最少资源/前进法/后退法/Pareto）→ 8种组合
- 步骤引导式UI + 条件可见逻辑
- `RetreatPanel`（脆弱性分析）不合并，升级为一级Tab
- 结果：3个旧Tab → 2个一级Tab（方案搜索 + 脆弱性分析），消除子Tab嵌套

**方案B（小二合一）**：仅合并 StrategyPanel + ResourceSearchPanel，风险更低
