# 全局约定

> 最后更新：2026-06-02

## 编码规范

- PR 代码变更限制在 `gacha_simulator/` 目录，不得修改 `tests/`、`output/`、`pyproject.toml`、`README.md`
- 版本号在 `gacha_simulator/_version.py`，遵循 Pride Versioning（PATCH=SHAME, MINOR=DEFAULT, MAJOR=PROUD）
- 所有分析面板从 `CompactResult` 列表提取数据，不再使用旧的 `InfoVector` 路径
- 策略不维护可变运行时状态（`acquired`、`pool_draw_counts` 等由 `StrategyContext` 传入）

## 计划文件约定

- 计划文件放入对应模块文件夹（`docs/01-活跃/panels/xxx/` 或 `docs/01-活跃/subsystems/xxx/`），不新建独立文件在顶层
- 搁置条目在 `docs/00-meta/搁置计划记录.md` 中登记，标注归属模块
- 计划文件增量修改——标注/替换/更正/补充，不整段删除
- 将计划文件归档前，如有搁置条目先补充到搁置记录
- 所有大型计划应在 `docs/00-meta/模块状态矩阵.md` 有对应条目

## 文档维护工作流

- **每次会话结束时**：将关键结论写入当前模块的 `05-笔记.md`（带日期）；如有代码变更则更新 `02-实施.md`
- **新建模块时**：创建六文件骨架 → 读实际代码填写 `02-实施.md` → 从审查报告拆入 `01-理论.md`
- **计划完成时**：主题计划 → `99-历史/`；更新 `02-实施.md` + `04-问题.md` + `模块状态矩阵.md`
- **每周清理时**：清理 `05-笔记.md` + `04-收件箱/`；更新 `00-本周聚焦.md` + `模块状态矩阵.md`

## P0/P1/P2 定义

| 级别 | 含义 | 示例 |
|------|------|------|
| P0 | 阻塞——影响正确性或阻止关键功能使用 | 保底概率未归一化、Pareto 声称不严格 |
| P1 | 风险——影响可靠性但不阻塞 | 缺失置信区间、无多重比较校正 |
| P2 | 债务——可延期但不忽视 | 文档化不足、硬编码值、缺少 UI 说明 |

## 搁置规范

- 搁置触发条件：外部依赖未就绪、方案决策未定、优先级明确低于当前工作
- 恢复条件：在 `搁置计划记录.md` 中必须有明确的解除条件
- 搁置条目必须标注归属模块

## 文档命名约定

- 所有中文文档使用 `.md` 扩展名
- 六文件固定命名：`00-档案.md` ~ `05-笔记.md`
- 主题计划文件保留原标题，不改名
- 目录名使用中文，无空格
