## 仍需人遵守的约定

以下约定机器无法判断——仍需人自行遵守。

### Pride Versioning 语义

版本号在 `gacha_simulator/_version.py`，格式 `MAJOR.MINOR.PATCH`：

| 级别 | 含义 | 触发 |
|------|------|------|
| MAJOR (PROUD) | 里程碑达成 | 新版面板上线、核心架构重构完成、重大性能突破 |
| MINOR (DEFAULT) | 功能累积 | 任何功能性变更（新面板、新 GDR、新策略、新分析） |
| PATCH (SHAME) | 缺陷修复 | 仅 bug 修复、文档修正——无功能性变更 |

### GDR 调用规范（强制）

- 使用 `make_gdr_calculator(store, target_specs, gdr_key, ...)` 构造 `GDRCalculator`——权重从 `ConfigStore` 自动提取
- **禁止**绕过直接调 `compute_gdr_from_compact` / `compute_success_probability`（权重易漏传、静默退化 1.0）
- 例外：`process_trace.py` / `per_pool_analysis.py` 中间层通过 `**kwargs` 透传权重，调用方需在 kwargs 注入 `desire_weights`/`miss_cost_weights`/`card_value_weights`

### 计划文件约定

- 计划文件放入对应模块文件夹，不新建独立文件在顶层
- **计划文件增量修改**——标注/替换/更正/补充，不整段删除
- 搁置条目在 `docs/00-meta/搁置计划记录.md` 中登记，标注归属模块 + 解除条件
- 所有计划文件首行含 META 元数据头（Planner skill 自动写入）

### 扩展指南（接口约束）

- **添加新 GDR 指标**：在 `core/gdr.py` 实现 + 在 `UNIFIED_GDR_REGISTRY` 注册
- **添加新策略**：在 `core/strategy.py` 实现 `Strategy` 子类 + `STRATEGY_REGISTRY` 注册
- **添加新停止条件**：在 `core/stop_condition.py` + `STOP_CONDITION_REGISTRY` 注册
- **添加新面板**：在 `gui/` 创建 → `MainWindow._setup_ui()` 注册 Tab

## P0/P1/P2 定义

| 级别 | 含义 | 示例 |
|------|------|------|
| P0 | 阻塞——影响正确性或阻止关键功能使用 | 保底概率未归一化、Pareto 声称不严格 |
| P1 | 风险——影响可靠性但不阻塞 | 缺失置信区间、无多重比较校正 |
| P2 | 债务——可延期但不忽视 | 文档化不足、硬编码值、缺少 UI 说明 |

## 搁置规范

- 触发条件：外部依赖未就绪、方案决策未定、优先级明确低于当前工作
- 恢复条件：在 `搁置计划记录.md` 中必须有明确的解除条件
- 搁置条目必须标注归属模块
