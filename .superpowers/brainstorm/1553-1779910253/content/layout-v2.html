两方案对比

A · 鸣潮式
✓ 左侧卡池列，多池间切换直观
✓ 池子信息紧凑，右侧空间宽裕
✓ 适合池子数量多、频繁切换的场景
✗ 左侧列占用宽度，中央展示区稍窄
✗ 单池沉浸感稍弱
B · 原神式
✓ 中央大幅立绘区，沉浸感最强
✓ 顶部标签切换池子，符合主流习惯
✓ 全幅展示区有"抽卡游戏"的味道
✗ 标签栏只能放有限数量（4-5个池子）
✗ 池子多时需要横向滚动标签

两个方案共享相同的底部调试区设计，仅上半区的布局哲学不同。
调试区可在两方案间通用——等待控制、模拟状态、GDR 全表的结构不变。
