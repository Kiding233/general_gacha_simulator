原始数据展示的细节约定

展示方式

• 每个数据结构一张可折叠卡片（QGroupBox 或自定义 collapsible widget）
• 展开后以 monospace 字体显示 Python repr() 或 dataclass 字段展开
• 颜色标记：键名(绿色)、数值(黄色)、字符串(蓝色)、布尔(橙色)
• 默认全部折叠，按需点开——因为 raw dump 可能很长

哪些数据需要 dump

GachaState
resources / pity_counters / real_time / total_actions / extra_state
来源: core/state.py
Pool (×N)
id / name / cost / rewards / available_from/until / is_exchange / pool_type / _cum_weights / _adjusted_*
来源: core/pool.py
CompactResult
draw_card_ids[] / draw_pool_ids[] / draw_pity[] / card_counts / final_resources / total_consumed / ...
来源: core/result_types.py .to_dict()
PityEngine
每条保底规则: 计数器当前值 / 触发阈值 / 概率分布 / 重置条件
来源: core/pity.py
StrategyContext
state / current_pools[] / all_pools[] / target_cards / acquired / pool_draw_counts / total_draws / ssr_ids
来源: core/strategy.py
StopCondition
类型名 / 参数 / .check() 当前判定 / .description()
来源: core/stop_condition.py
ResourceGain / SimulationStats
ResourceGain: 各收益函数定义 · SimulationStats: total_consumed / total_gained / card_counts / pity_triggers / ...
来源: core/resource_gain.py · service/gacha_service.py

注意点

• Pool 的 rewards 列表可能非常长（几百张卡），应限制显示数量（如前 10 条 + "…共 N 条"）
• CompactResult 的序列数组随步数增长，建议只显示最后 K 步的序列片段
• 所有 raw dump 是只读的，不可编辑
• 每次操作（抽卡/等待）后自动刷新所有展开中的卡片
