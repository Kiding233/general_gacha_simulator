调试面板 — 需要展示哪些信息？

调试模式下用户需要观察的核心维度

🔍 逐步状态追踪

每执行一次操作（抽卡/等待），记录并展示：
• 步数 · 时间戳 · 操作类型（单抽/十连/等待）· 操作对象（哪个池）
• 抽到的卡 ID + 稀有度 + 保底触发情况
• 资源消耗 + 资源获得 + 各类资源余额变化
• 保底计数器变化（软保底 + 硬保底 + 大保底）
• 策略决策痕迹：当前策略名称、为什么选这个操作

📊 GDR 实时追踪

17 种 GDR 指标 —— 每次操作后重新计算，展示变化趋势：
• 当前值 + 与上一步的差值（↑/↓）
• 与阈值的对比（达标 ✓ / 未达标 ✗）
• 可选：选定一个主 GDR，画实时走势迷你图

🧩 池子快照

每个池子的当前状态一目了然：
• 名称 · 类型 · 开启/关闭时间 · 已抽次数
• 保底状态：软保底计数 / 硬保底计数 / 大保底标记
• UP 卡列表 · 目标卡标记 · 当前 SSR 概率分布
• 该池的逐池 GDR 值

📋 操作历史日志

类似控制台的滚动日志，按时间倒序或顺序：
• [步号] 时间 | 操作 | 池子 | 结果卡 | 稀有度 | 保底? | 资源变化
• 支持筛选：按池子 / 操作类型 / 稀有度过滤
• 新操作自动追加，可暂停滚动

⚙️ 模拟引擎内部状态

暴露引擎内部不可见变量：
• GachaState 全量 dump（资源、时间、各计数器）
• PityEngine 内部状态（各保底机制的当前概率分布）
• StrategyContext 最后一次构建时的内容
• 可用池列表 vs 全部池列表对比
• StopCondition 当前判定结果
