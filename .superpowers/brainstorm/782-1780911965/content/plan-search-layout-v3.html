三种模式对各控件的复用关系

模式 权重表 前进/后退选项 二分参数 引擎实际调用
最少资源 ✗ 隐藏 ✗ 隐藏 ✓ 显示 二分搜索最少资源
最多目标卡 ✓ 显示 ✓ 显示 ✗ 隐藏 前进法或后退法（由子选项决定）
Pareto前沿 ✓ 显示 ✗ 隐藏 ✓ 显示 按错失代价升序逐张移除 × 每步二分搜索最少资源
= 后退法排序逻辑 + 最少资源二分逻辑的组合

关键洞察：Pareto 不是独立算法 —— 它复用了「后退法的 miss_cost 排序」+「最少资源的二分搜索」。 因此 UI 上权重表（提供排序依据）和二分参数（提供搜索精度）都要显示，唯独前进/后退子选项不需要（Pareto 固定按代价升序 = 后退方向）。
